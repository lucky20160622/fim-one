## [未发布]

### 已更改

- **集中式工具截断** — 所有工具类型（WebFetch、HttpRequest、ShellExec、PythonExec、NodeExec）现在将截断委托给共享的 `truncation.py` 模块，而不是内联逻辑。两个入口点：`truncate_tool_output()`（JSON 感知、字符级）用于网络/API 工具，`truncate_bytes()`（字节级）用于代码执行工具。默认值可通过 `TOOL_OUTPUT_MAX_CHARS`、`TOOL_OUTPUT_MAX_ITEMS`、`TOOL_OUTPUT_MAX_BYTES` 环境变量配置。
- **领域检测解耦为独立中间件** — 领域分类（法律、医疗、金融等）现在是一个独立模块（`domain.py`），在每个端点（ReAct 和 DAG）内独立运行，而不是与自动路由分类捆绑在一起。两个端点都直接调用 `classify_domain()`，消除了对自动路由传播 `domain_hint` 的依赖。
- **领域 SOP 指令已软化** — 专家领域指令现在指导而不是强制网络搜索。当用户的查询和上下文已包含充分信息时，智能体直接回答，而不是强制进行不必要的搜索。
- **`web_search` 始终在 ReAct 中固定** — `web_search` 工具现在始终包含在 ReAct 工具选择中，无论是否检测到专家领域，确保它不会被工具选择器删除。
- **`AUTO_ROUTING` 环境变量已移除** — auto 端点现在始终对查询进行分类；禁用分类的切换已被移除。
- **引文验证器接受上游 `domain_hint`** — `should_verify_citations()` 现在接受来自查询级 LLM 分类的 `domain_hint` 参数，在领域已知时绕过逐步关键字扫描。

### 已添加

- **按模型原生函数调用切换** — 新增 `tool_choice_enabled` 设置（ENV 全局 + 管理员按模型）控制是否尝试强制 `tool_choice` 用于结构化输出提取。拒绝强制工具选择的模型（例如思考模式模型）可以跳过第 1 级，直接进入 JSON 模式，消除每次结构化调用约 10 秒的性能损耗。可在设置 → 模型 → 高级中配置。
- **结构化输出兼容性表** 在模型文档中 — 显示哪些提供商支持原生函数调用、JSON 模式和规划可靠性评级。
- **文件附件下载** — 聊天消息中的文件附件卡片现在可点击以通过身份验证获取下载原始文件。添加了下载图标指示器和悬停状态以提高可发现性。
- **智能文档处理（视觉感知）路线图项** — 添加了计划功能，用于根据模型视觉能力进行自适应文档处理（三个层级：视觉优先、混合、纯文本）。
- **DAG 质量改进（P0）：默认模型升级** — 没有显式 `model_hint` 的 DAG 步骤现在使用注册表的通用模型（例如 Sonnet）而不是快速模型。只有带有 `model_hint="fast"` 的步骤使用轻量级模型。
- **DAG 质量改进（P1）：技能自动发现** — `DAGPlanner.plan()` 接受 `skill_descriptions` 参数。可用技能（名称 + 描述）被注入到规划提示中，以便规划器可以为域匹配的步骤建议 `tool_hint="read_skill"`。
- **DAG 质量改进（P2）：引用验证器** — 新增 `citation_verifier.py` 模块通过正则表达式提取法律/医疗/财务引用，通过 LLM 判断验证准确性，失败时使用更正反馈重试。集成到执行器步骤后流程中。由 `DAG_CITATION_VERIFICATION` 环境变量控制（默认：`true`）。
- **DAG 质量改进（P3）：结构化内容上下文保留** — `_build_step_context()` 检测依赖结果中的结构化内容（法律引用、markdown 表格、代码块），并应用可配置的截断乘数（`DAG_STRUCTURED_CONTEXT_MULTIPLIER`，默认 `3.0`），防止 DAG 步骤之间关键引用的丢失。
- **DAG 质量改进（P4）：路由器中的域分类** — `RouteDecision` 现在包括 `domain_hint`（legal/medical/financial/null）。紧密耦合的域分析优先使用 ReAct 模式；选择 DAG 时，域上下文被注入到规划器中以指导 `model_hint` 选择。
- **ReAct 中的域模型升级** — 当路由器检测到专家域（法律/医疗/财务）时，ReAct 端点通过 `registry.get_by_role("reasoning")` 从通用模型升级到推理模型。注入强制指令要求在写入前进行网络搜索和通过搜索进行引用验证。`read_skill` 工具在工具选择中被固定。

### 已更改

- **温度提示移至工具提示** — 模型表单对话框中的温度推理提示现在显示为滑块旁标签旁的紧凑工具提示图标 (HelpCircle)，而不是滑块下方的静态段落，减少了视觉混乱。
- **RAG 基础管道** — 三阶段管道：检索、引文提取和置信度评分。置信度评分是前 3 个证据单位的平均检索分数。
- **API 密钥横幅更新为"活跃"** — 管理员和用户设置 API 密钥页面现在显示绿色"API 密钥活跃"横幅（替换之前的"即将推出"通知），并附有 API 身份验证文档的链接。
- **多工作进程文档已澄清** — `REDIS_URL` 和 `WORKERS` 环境变量描述已更新，包含涵盖停止、注入、PostgreSQL 要求和本地开发指导的警告检查清单。
- **DAG 步骤默认模型** — 没有 `model_hint` 的步骤现在解析为通用模型（之前为快速模型）。快速模型仅在显式设置 `model_hint="fast"` 时使用。
- **路由器分类输出** — `RouteDecision` 现在输出 `domain_hint`（legal/medical/financial/null）以及现有的 `mode` 字段，支持下游域感知模型选择。

### 已修复

- **区域文档中重复的 OpenAPI 端点** — 从 `docs.json` 中所有非英文区域 API 选项卡中删除了冗余的 `Endpoints` OpenAPI 组。英文部分保留单一来源；区域部分不再重复。
- **结构化输出降级链** — 3 级回退（原生 FC → JSON 模式 → 纯文本提取）现在在模型生成格式错误的 JSON 时正确地遍历所有级别。之前在第 2 级因 `ValueError` 而短路。
- **`json_mode_enabled` 数据库值被忽略** — 通过管理员配置的模型（模型组）现在正确使用其按模型的 `json_mode_enabled` 设置，而不是总是回退到 `LLM_JSON_MODE_ENABLED` 环境变量。
- **DAG 规划失败消息** — 规划失败现在显示用户友好的双语消息，建议切换模型，而不是原始管道错误。
- **DAG 步骤卡布局** — 重新设计了 `StepProgressCard`，使任务文本在步骤徽章下方换行，而不是在单行上截断。步骤节点中的工具徽章现在仅在不存在 `tools_used` 但存在 `tool_hint` 时显示，避免冗余显示。
- **MCP 服务器所有者绕过 allow_fallback** — 服务器所有者不再被 `allow_fallback=False` 限制阻止。之前，即使设置了 `allow_fallback=False`，MCP 服务器的创建者也可能被跳过；现在仅排除没有凭证的非所有者用户。
- **模型提供商 API 密钥验证** — 创建非 openai 兼容的提供商现在需要 API 密钥，包括内联验证错误、必需的星号指示符和正确的 `aria-invalid` 可访问性属性。

## [v0.8] - 2026-03-20

### 新增

- **ReAct 合成中的智能体指令** — `ReActAgent` 上的新 `agent_directive` 参数将智能体的核心目的（例如"将中文翻译成英文"）注入合成系统提示中，确保最终答案遵循智能体的身份而不是默认为对话回复。在 ReAct 和 DAG 聊天端点中都进行了连接。

- **市场重新设计第 1 阶段 — 解决方案 + 组件两层模型** — 将平面 6 类型市场选项卡替换为结构化的两级导航：顶级类别切换（解决方案：智能体/技能/工作流；组件：连接器/MCP 服务器）和类型子选项卡。添加了范围选择器下拉菜单以在全球市场和用户组织之间切换。从市场范围中移除了知识库（仅限组织）。统一了组织和市场共享：两者现在都需要明确订阅（不再自动显示组织）。

- **智能文件内容注入** — 小型上传文件（`<32K` 字符）会自动获取并内联到 LLM 上下文中；大型文件会收到带有按需读取工具提示的元数据摘要。
- **`read_uploaded_file` 智能体工具** — 双模式文件读取工具：分页读取（偏移/限制）用于顺序访问，类似 grep 的搜索（支持正则表达式的查询）用于目标内容提取。
- **`GET /api/files/{file_id}/content` 端点** — 分页文本内容 API，用于访问从上传文件中提取的完整文本。
- **`.content` 辅助文件** — 完整提取的文本与上传文件一起存储为辅助文件，无需重新提取即可实现持久内容访问。
- **文件 API 响应中的 `content_length` 字段** — 文件上传和列表端点现在包括 `content_length` 以显示文本内容大小。
- **`fileApi.getContent()` 前端客户端方法** — 用于从前端获取分页文件内容的新 API 客户端方法。

- **工作流引擎节点重试** — 每个节点的重试，具有指数退避。节点可以在其数据配置中指定 `retry_count` 和 `retry_delay_ms`。失败的节点最多重试 N 次，延迟上限为 60 秒。发出带有尝试次数和先前错误的 `node_retrying` SSE 事件。在重试等待期间尊重取消事件。
- **工作流变量存储表达式评估** — 通过 simpleeval 进行安全的 Python 表达式评估。`evaluate_expression()` 方法公开存储变量（不包括环境变量）以及常见内置函数（len、str、int、float、bool、abs、min、max、sum、round、sorted）。加上 `has()`、`delete()`、`keys()` 辅助方法。
- **工作流第 2 阶段节点：HumanIntervention** — 暂停以获得批准的节点（自动批准存根）。读取 prompt_message（带有 `{{variable}}` 插值）、受理人、timeout_hours。Sky-500 主题的前端节点，带有提示预览和 @受理人徽章。配置面板包含提示文本区域、受理人输入和超时字段。
- **工作流模板扩展到 12 个** — 四个新模板：人工批准管道（LLM → HumanIntervention → ConditionBranch → 双端点）、多步数据处理（CodeExecution → ConditionBranch → Transform → ListOperation）、知识增强智能体（QuestionUnderstanding → KnowledgeRetrieval → LLM）、迭代审查循环（Loop → LLM → VariableAssign）。
- **工作流窗格上下文菜单** — 右键单击空画布以获取粘贴（剪贴板为空时禁用）、全选、适应视图和自动布局操作。判别联合状态将节点与窗格上下文菜单分开。
- **工作流节点上下文菜单** — 右键单击任何节点以访问配置、复制、复制和删除操作。对于开始/结束单例节点，删除操作被隐藏。菜单在外部单击或滚动时自动关闭。
- **工作流第 2 阶段模板** — 两个新的内置模板（共 8 个）：列表处理管道（开始 → ListOperation → Transform → 结束）和增强问答（开始 → QuestionUnderstanding → LLM → 结束）。
- **工作流第 2 阶段验证** — 为 ListOperation（缺少输入/表达式）、Transform（缺少输入/空操作）、Iterator（缺少列表变量）、Loop（缺少条件）、DocumentExtractor、QuestionUnderstanding（缺少输入）、ParameterExtractor（缺少输入文本/空参数）扩展蓝图验证警告。修复：排序操作不再错误地要求表达式。
- **工作流节点注释** — 每个节点的注释功能。单击任何节点配置面板中的"注释"以添加在画布上节点下方显示为斜体文本的自由格式注释。自动扩展文本区域，带有 `data.note` 存储。
- **工作流对齐网格** — 16px 对齐网格，与点背景图案匹配，用于精确节点对齐。
- **工作流连接验证** — 除了现有的开始/结束限制外，还防止自连接和重复边。
- **工作流缩放感知放置** — 节点调色板拖放使用 `screenToFlowPosition()` 在任何缩放级别下正确放置。
- **工作流 SSE 多行数据** — SSE 解析器现在处理多行 `data:` 字段并忽略保活注释行。
- **工作流键盘快捷键** — 复制 (Cmd+C)、粘贴 (Cmd+V)、复制 (Cmd+D)、全选 (Cmd+A)、Escape 取消选择。复制/粘贴生成新 ID 并偏移位置；开始/结束节点被排除在复制之外以维持单例约束。
- **工作流内联描述编辑** — 单击工具栏中的描述行以打开弹出文本区域，用于编辑工作流描述而无需导航离开。
- **工作流 ConditionBranch LLM 模式** — 适当的基于 LLM 的条件评估，替换了天真的"匹配第一个"存根。从条件标签构建分类提示，调用快速 LLM，并使用不区分大小写 + 模糊匹配。
- **反应式边标签** — 条件/分类器分支的边标签现在使用 `useNodesData()` 在配置面板中编辑条件或类时进行实时更新。
- **工作流统计面板** — 基于工作表的统计显示，显示总运行次数、已完成/失败/已取消计数、带有视觉进度条的成功率、平均持续时间和相对格式的最后运行时间。从工具栏中的 BarChart3 图标触发。
- **第 2 阶段工作流节点：DocumentExtractor 和 QuestionUnderstanding** — 两个更多节点类型：
  - **DocumentExtractor**：文档文本处理的存根执行器 — 支持文本/base64 输入类型和 4 种提取模式（full_text、带有 page_range 的页面、带有字数/字符数/行数/页数的元数据、带有 markdown 表格检测的表格）。URL 输入推迟到后续实现。
  - **QuestionUnderstanding**：具有 4 种模式的基于 LLM 的问题预处理 — 重写（清晰度）、扩展（上下文 + 子问题）、分类（意图/主题/置信度 JSON）、分解（子问题数组）。使用快速 LLM 和可选的自定义系统提示覆盖。
  - 完整前端：节点组件、配置面板、调色板条目（QU 的 AI 类别、DE 的数据类别）、变量选择器支持。
- **第 2 阶段工作流节点：ListOperation 和 Transform** — 两个数据处理节点类型：
  - **ListOperation**：8 个列表操作（filter、map、sort、slice、flatten、unique、reverse、length），对 filter/map/sort 使用 simpleeval 表达式。支持 JSON 字符串输入自动解析。
  - **Transform**：基于管道的数据转换，具有 6 种操作类型（json_path、type_cast、format、regex_extract、string_op、math_op）。操作按顺序应用 — 一个的输出馈入下一个。
  - 完整前端：节点组件、带有特定于操作的编辑器的配置面板、调色板条目、变量选择器支持。
- **第 2 阶段工作流节点：Loop 和 ParameterExtractor** — 两个更多高级节点类型：
  - **Loop**：带有 simpleeval 表达式评估的 while 条件循环、可配置的循环变量名称和 max_iterations 安全限制。前端包含条件输入、变量名称和迭代限制配置。
  - **ParameterExtractor**：从文本进行基于 LLM 的结构化参数提取。定义类型参数（名称/类型/描述/必需），提供输入文本引用，并可选地添加提取说明。返回解析的 JSON。前端包含参数列表编辑器和提取提示文本区域。
- **第 2 阶段工作流节点：Iterator 和 VariableAggregator** — 两个新的高级节点类型：
  - **Iterator**：使用可配置的项/索引变量名称和 max_iterations 安全限制迭代列表。支持本机列表、JSON 字符串解析和模板变量引用。
  - **VariableAggregator**：通过四种模式合并来自多个上游分支的输出 — 收集到列表、连接字符串（自定义分隔符）、合并字典或取第一个非空值。
  - 完整前端集成：节点组件、调色板条目、配置面板表单、小地图颜色、变量选择器支持。后端执行器在引擎中注册。
- **工作流模板扩展** — 添加了两个新的内置模板（共 6 个）：数据处理管道（开始 → CodeExecution → TemplateTransform → 结束）和带知识检索的智能体（开始 → KnowledgeRetrieval → 智能体 → 结束）。修复了新模板上缺少的 `icon` 字段。为"intermediate"和"advanced"模板类别添加了缺少的 i18n 键。
- **工作流 ENV 变量对话框** — 每个工作流的加密环境变量管理 UI。键值编辑器，带有掩蔽显示、重复键检测和内联验证。连接到 `GET/PUT /api/workflows/{id}/env` 端点。可通过工具栏下拉菜单访问。
- **工作流自动保存** — 5 秒去抖动自动保存计时器，带有立即 Cmd+S 覆盖。保存状态指示器显示"保存中..."、"未保存的更改"或"保存于 Xs 前"，30 秒自动刷新。
- **工作流列表搜索和筛选** — 按名称/描述的客户端搜索、状态筛选按钮（全部/草稿/活跃）、排序下拉菜单（最近更新/创建/名称 A-Z）和无结果的空状态。
- **工作流彩色编码小地图** — 小地图节点颜色现在与每个节点类型的类别颜色匹配（绿色=开始、红色=结束、蓝色=LLM、橙色=条件等）。
- **智能体技能系统** — 智能体指令的按需技能加载：`Skill` 模型（名称、描述、内容、脚本）具有完整 CRUD、发布审查和可见性生命周期；`read_skill(name)` 内置工具用于渐进式披露（~80% 令牌减少）；技能存根注入系统提示（~15 令牌/技能）；每个智能体的 `compact_instructions` 字段用于自定义 ContextGuard 压缩策略。
- **技能 UI** — 技能列表页面、全页编辑器（内容/脚本/设置选项卡）、SkillCard 组件、侧边栏导航、智能体编辑器技能选择器，带有紧凑指令文本区域。
- **工作流发布流程** — 工作流的发布/取消发布/重新提交端点，与现有智能体发布模式匹配。组织模型上的新 `review_workflows` 标志（带有 Alembic 迁移）以启用组织级工作流审查门控。发布操作连接到工作流卡和工具栏 UI。
- **工作流蓝图系统** — 用于设计和执行多步自动化蓝图的可视工作流编辑器：

### 已更改

- **评估中心** — 门户中新增 `/eval` 部分，用于定量智能体质量基准测试。功能包括：测试数据集管理（创建/编辑提示词 + 预期行为 + 断言测试用例的集合）；评估运行（选择智能体 + 数据集 → 通过 `asyncio.Semaphore(5)` 并行运行所有用例 → 使用快速 LLM 评分器对每个答案进行评分 → 保存每个用例的通过/失败 + 延迟 + token 结果）；结果查看器，具有可展开行、通过率进度条和自动轮询（3秒）功能。适合企业采购审查。迁移 `r8t0v2x4z567`。
- **管理员：每个模型的 `json_mode_enabled` 标志** — 每个模型配置现在在管理员 → 模型 → 高级设置中都有明确的 JSON 模式切换（默认开启）。禁用它可防止 LiteLLM 注入助手预填充消息（JSON `role:assistant` 预填充），该消息会导致 AWS Bedrock 在第 2+ 次智能体迭代时以 `ValidationException` 拒绝对话。ENV 配置的模型由新的 `LLM_JSON_MODE_ENABLED` 环境变量控制。所有 URL 模式启发式方法（`is_bedrock`、`is_anthropic_relay`）已被移除；功能现在是显式的。包括 Alembic 迁移（`q7s9u1w3y456`）。
- **管理员模型表单 UX** — 重组了添加/编辑模型对话框：默认仅显示名称、模型名称和 API 密钥。所有其他设置（基础 URL、提供商、温度、最大输出 token、上下文大小、JSON 模式）折叠在"高级"切换下，减少常见情况下的视觉混乱。
- **设置：组织选项卡** — 用户现在可以直接从设置页面创建、加入和管理组织。功能包括创建组织对话框（名称、自动生成的 slug、描述、emoji 图标）、编辑对话框（仅限所有者）、用于管理成员角色的成员表单（所有者/管理员/成员，带有彩色徽章）以及内联操作（非所有者可离开，所有者可删除）。所有组织操作调用面向用户的 `/api/orgs` 端点。完整的 i18n 支持（EN + ZH）。
- **SSE 协议 v2 — 实时答案流式传输**：后端 `StreamChunk` 获得 `delta_reasoning` 和 `usage` 字段；LLM 请求包括使用情况跟踪；ReAct `_run_native()` 现在通过 `stream_chat` 流式传输。SSE 协议在答案后立即发出 `done`，然后将 `suggestions`、`title` 和 `end` 作为单独事件发送。LLM 执行期间的 DB 会话早期释放。SQLite 池大小从 5 增加到 20。
- **AI Builder 扩展 — 7 个新工具 + `is_builder` 标志**：三个新连接器构建器工具（`GetSettings`、`TestConnection`、`ImportOpenAPI`）和四个新智能体构建器工具（`ListConnectors`、`AddConnector`、`RemoveConnector`、`SetModel`）。智能体上新增 `is_builder` 布尔列，带有 Alembic 迁移。会话创建时构建器提示自动刷新。OpenAPI 导入的 SSRF 防护。
- **SSE v2 前端 — 流式光标、DAG 轮快照、UI 优化**：前端钩子解析 `end` / `suggestions` / `title` 事件。DAG 重新规划将之前失败的轮显示为可折叠卡片。CSS 点脉冲流式光标。DAG 布局与步骤状态解耦。折叠侧边栏新聊天按钮。连接器编辑器溢出修复。`font-mono` 替换为 `tabular-nums`。
- **AI Builder 概念页面**：新增 `docs/concepts/ai-builder.mdx`，包含连接器和智能体构建器指南，以及中文翻译。
- **构建器对话框 — 用于连接器管理的 ReAct 智能体**：将 AI 助手面板从单次 LLM 调用升级为完整的 ReAct 智能体循环。"高级构建器"按钮打开全屏对话框，其中包含嵌入式 PlaygroundPage（左侧聊天，右侧实时操作预览）。六个连接器构建器工具（`connector_list_actions`、`connector_create_action`、`connector_update_action`、`connector_delete_action`、`connector_update_settings`、`connector_test_action`）直接在 DB 上操作，并在构建器智能体处于活动状态时自动注入。`POST /api/builder/session` 创建或重用每个连接器的构建器智能体，该智能体保持对话历史。
- **双数据库支持（SQLite + PostgreSQL）**：`DATABASE_URL` 现在接受 SQLite（`sqlite+aiosqlite:///...`）和 PostgreSQL（`postgresql+asyncpg://...`）连接字符串。SQLite 保持零配置默认值；PostgreSQL 推荐用于生产和多工作进程部署。Docker Compose 自动配置 PostgreSQL 服务及健康检查。Alembic 迁移是方言感知的（json_extract vs `::json->>'key'`）。管理员健康仪表板显示实际数据库引擎和版本。
- **模型配置文档**：新增 `docs/configuration/models.mdx` 页面，记录模型配置参考，包括每个提供商的扩展思考/推理设置。
- **扩展思考/推理支持**：为具有扩展思考的模型（OpenAI o 系列、Gemini 2.5+、Claude）添加 `LLM_REASONING_EFFORT`（低/中/高）和 `LLM_REASONING_BUDGET_TOKENS` 环境变量。为 OpenAI/Gemini 发送 `reasoning_effort`，为 Anthropic 发送 `thinking`（自动检测）。从多个提供商字段名称（DeepSeek R1、Claude 代理）提取 `reasoning_content` 并在 UI"思考"步骤中显示。
- **管理员：工具禁用** — 新的管理员工具面板，具有每个工具的启用/禁用切换（存储为 `disabled_builtin_tools` 设置）。禁用的工具在运行时从聊天工具解析中过滤。`/api/tools/catalog` 端点返回 `disabled` 标志，工具页面 UI 显示禁用状态。
- **管理员：连接器页面上的 MCP 服务器** — MCP 服务器管理从工具页面移至连接器页面作为新选项卡。`MCPServersSection` 提取为具有命令式操作引用的可重用组件。
- **管理员：健康仪表板中的推理工作量** — 推理工作量标签现在显示在管理员健康检查仪表板中。
- **导出：PDF 格式** — 对话现在可以导出为 PDF 文档（需要 `reportlab` + `Markdown` 附加功能）。
- **start.sh：WORKERS 环境变量** — 设置 `WORKERS=N` 以在门户/api 模式下使用多个工作进程运行 uvicorn。

### 已更改
- **测试：NativeToolFakeLLM 针对基于流的 ReAct 更新**：测试假 LLM 现在生成与新的基于流的 `_run_native()` 循环相匹配的 `StreamChunk` 序列
- **LLM 层**：用 [LiteLLM](https://github.com/BerriAI/litellm) 替换直接的 `AsyncOpenAI` 客户端以支持通用提供商。所有特定于提供商的条件判断（Anthropic 思考、`max_completion_tokens`、GPT-5 工具解决方案）已被移除。Anthropic 模型现在通过 LiteLLM 的原生 API 路由，启用适当的 `reasoning_content`（扩展思考）支持
- **工具页面简化**：工具页面现在仅显示内置工具（不再有选项卡）；MCP 管理已移至连接器页面
- **Redis 中断代理**：通过 Redis Pub/Sub 进行跨工作进程中断/注入中继。当设置 `REDIS_URL` 时自动启用；否则回退到进程内队列。Docker Compose 现在包含 Redis 服务（自动配置，无需用户操作）
- **导出：默认详细程度**：导出对话框现在默认为摘要（无执行详情）而不是完整
- **工件：按文件 mtime 排序**：列出所有工件现在使用每个文件的 mtime 以实现准确排序（恢复对话时间解决方案）
- **聊天：异步图像加载**：`_load_image_data_urls` 现在是异步的，通过 `asyncio.to_thread` 将文件读取卸载到线程以解除事件循环阻塞
- **多工作进程就绪**：SSE 票证、OAuth 状态和绑定票证现在使用 JWT 签名的令牌而不是内存中的字典；文件索引锁使用 `fcntl.flock()` 以实现跨进程安全；维护缓存 TTL 从 30 秒减少到 5 秒以加快跨工作进程传播
- **健康仪表板：基础设施检查**：向管理员健康仪表板添加了数据库类型（SQLite/PostgreSQL）和 Redis 状态
- **智能中继路由**：自动从 URL 路径模式检测 API 协议（例如 `/claude` 或 `/anthropic` → Anthropic 原生，`/gemini` → Google 原生），为第三方中继平台启用适当的身份验证标头和参数转换，无需手动提供商配置
- **结构化输出降级**：统一的 `structured_llm_call()` 实用程序具有 3 级提取（原生 FC → JSON 模式 → 纯文本 + 正则表达式回退）。重构了 DAG 规划器、计划分析器和工具选择以使用共享实用程序，用一致的重试和降级替换了所有结构化 LLM 调用站点中的临时 JSON 解析
