  ## [未发布]

<div id="added">
  ### 新增
</div>

* **SSE 协议 v2 — 实时回答流式传输**：后端 `StreamChunk` 新增 `delta_reasoning` 和 `usage` 字段；LLM 请求包含用量追踪；ReAct `_run_native()` 现在通过 `stream_chat` 进行流式传输。SSE 协议在回答后立即发送 `done`，随后将 `suggestions`、`title` 和 `end` 作为独立事件发送。LLM 执行期间提前释放 DB 会话。SQLite 连接池大小从 5 提升至 20
* **AI Builder 扩展 — 7 个新工具 + `is_builder` 标志**：三个新连接器构建器工具（`GetSettings`、`TestConnection`、`ImportOpenAPI`）和四个新智能体构建器工具（`ListConnectors`、`AddConnector`、`RemoveConnector`、`SetModel`）。智能体新增 `is_builder` 布尔字段及 Alembic 迁移。构建器提示词在会话创建时自动刷新。OpenAPI 导入新增 SSRF 防护
* **SSE v2 前端 — 流式光标、DAG 轮次快照、UI 优化**：前端 hooks 解析 `end` / `suggestions` / `title` 事件。DAG 重新规划显示之前失败轮次为可折叠卡片。CSS 点脉冲流式光标。DAG 布局与步骤状态解耦。侧边栏折叠时新增新建对话按钮。连接器编辑器溢出修复。`font-mono` 替换为 `tabular-nums`
* **AI Builder 概念文档页面**：新增 `docs/concepts/ai-builder.mdx`，含连接器和智能体构建器指南，附中文翻译
* **Builder Dialog — Connector 管理 ReAct 智能体**：将 AI 助手面板从单次 LLM 调用升级为完整 ReAct 循环。"高级构建器"按钮打开全屏对话框，左侧为嵌入式 PlaygroundPage，右侧为实时 Action 预览（3 秒轮询）。六个 Connector Builder 工具（`connector_list_actions`、`connector_create_action`、`connector_update_action`、`connector_delete_action`、`connector_update_settings`、`connector_test_action`）直接操作数据库，Builder Agent 激活时自动注入。`POST /api/builder/session` 按 Connector 缓存复用 Builder Agent，保留对话历史
* **双数据库支持（SQLite + PostgreSQL）**：`DATABASE_URL` 现在同时支持 SQLite（`sqlite+aiosqlite:///...`）和 PostgreSQL（`postgresql+asyncpg://...`）连接字符串。SQLite 仍为零配置默认值；PostgreSQL 推荐用于生产环境和多进程部署。Docker Compose 自动配置 PostgreSQL 服务并附带健康检查。Alembic 迁移可感知数据库方言（json_extract vs `::json->>'key'`）。管理员健康看板显示实际数据库引擎和版本
* **模型配置文档**：新增 `docs/configuration/models.mdx` 页面，记录模型配置参考，包括各提供商的扩展思考/推理设置
* **扩展思考 / 推理支持**：新增 `LLM_REASONING_EFFORT`（low/medium/high）和 `LLM_REASONING_BUDGET_TOKENS` 环境变量，支持具备扩展思考能力的模型（OpenAI o 系列、Gemini 2.5+、Claude）。OpenAI/Gemini 发送 `reasoning_effort` 参数，Anthropic（自动检测）发送 `thinking`。从多个提供商字段名（DeepSeek R1、Claude 代理）中提取 `reasoning_content`，并在 UI 的"思考中"步骤中展示
* **管理后台：工具禁用**：新增管理后台工具面板，支持逐工具启用/禁用切换（存储为 `disabled_builtin_tools` 设置）。禁用的工具在运行时会从聊天工具解析中过滤掉。`/api/tools/catalog` 端点返回 `disabled` 标志，工具页面 UI 显示禁用状态
* **管理后台：MCP 服务器移至连接器页面**：MCP 服务器管理从工具页面移至连接器页面，作为新标签页。`MCPServersSection` 抽取为可复用组件，支持命令式操作引用
* **管理后台：健康看板显示推理强度**：管理员健康检查看板现在显示推理强度标签
* **导出：PDF 格式**：对话现在可以导出为 PDF 文档（需要 `reportlab` + `Markdown` 额外依赖）
* **start.sh：WORKERS 环境变量**：设置 `WORKERS=N` 以在 portal/api 模式下以多 worker 运行 uvicorn

* **Docker Compose 部署**：单镜像 (API + Frontend) 、用于数据持久化的命名卷，以及独立模式的 Next.js 输出模式
* **管理后台：登录历史记录**：认证端点现在会将每次登录尝试 (IP 地址、User-Agent、成功/失败及原因) 记录到 `LoginHistory` 表中，用于安全审计
* **管理后台：文件浏览器**：新增管理后台文件浏览器端点，用于列出和下载用户上传的文件，并支持分页
* **管理后台：增强的智能体/知识库视图**：管理后台智能体列表和知识库视图现已提供 `model_name`、`tools`、`kb_ids` 和 `embedding_model` 字段，提升可见性
* **聊天：敏感词拦截**：ReAct 和 `inject_message` 聊天端点现在都会执行敏感词过滤，拦截包含已配置敏感词的消息
* **SSE：结构化错误处理**：`useSSE` hook 中的 `onError` 回调已从普通字符串升级为 `ApiError` 对象，以提供更丰富的错误上下文 (错误代码、参数)
* **UI：`formatTokens` 工具**：新增用于显示令牌计数的格式化辅助工具，支持基于区域设置的数字格式化
* **UI：`Checkbox` 和 `FadeIn` 组件**：新增 shadcn 风格的复选框组件和淡入动画包装组件

<div id="changed">
  ### 变更
</div>

* **测试：NativeToolFakeLLM 更新为流式 ReAct**：测试用假 LLM 现在生成 `StreamChunk` 序列，以匹配新的基于流式的 `_run_native()` 循环
* **LLM 层**：使用 [LiteLLM](https://github.com/BerriAI/litellm) 替换了直连 `AsyncOpenAI` 客户端，实现通用提供商支持。所有提供商特定的条件逻辑（Anthropic thinking、`max_completion_tokens`、GPT-5 tools 变通方案）已全部移除。Anthropic 模型现在通过 LiteLLM 的原生 API 路由，正确支持 `reasoning_content`（扩展思考）
* **工具页面简化**：工具页面现在仅显示内置工具（不再有标签页切换）；MCP 管理已移至连接器页面
* **Redis 中断代理**：通过 Redis Pub/Sub 实现跨进程中断/注入中继。设置 `REDIS_URL` 时自动启用，否则回退到进程内队列。Docker Compose 现已包含 Redis 服务（自动配置，无需用户操作）
* **导出：默认详情级别**：导出对话框现在默认为摘要模式（不含执行详情），而非完整模式
* **工件：按文件 mtime 排序**：list-all-artifacts 现在使用每个文件的 mtime 进行精确排序（还原了之前使用会话创建时间的权宜做法）
* **聊天：异步图片加载**：`_load_image_data_urls` 现在是异步函数，通过 `asyncio.to_thread` 将文件读取卸载到线程，避免阻塞事件循环
* **多进程就绪**：SSE 票据、OAuth 状态和绑定票据改用 JWT 签名令牌替代内存字典；文件索引锁使用 `fcntl.flock()` 实现跨进程安全；维护模式缓存 TTL 从 30s 降低至 5s 加速跨进程传播
* **健康看板：基础设施检查**：管理员健康看板新增数据库类型（SQLite/PostgreSQL）和 Redis 状态检查
* **智能中转路由**：根据 URL 路径模式自动检测 API 协议（如 `/claude` 或 `/anthropic` → Anthropic 原生，`/gemini` → Google 原生），为第三方中转平台自动启用正确的认证头和参数转换，无需手动配置提供商

* **文档：ASCII → Mermaid 图表**：所有使用框线字符绘制的 ASCII 图均已转换为原生 Mermaid 图表 (共 12 张，分布在 6 个页面) ，以获得更好的渲染效果和交互性
* **文档：深色模式主题**：优化了 Mintlify 的颜色配置——深色模式下采用暖琥珀色强调色、渐变背景装饰，以及与项目主题一致的背景颜色

<div id="fixed">
  ### 已修复
</div>

* **DAG：AnalysisCard Markdown 渲染**：分析推理内容现在使用 `MarkdownContent` 渲染，修复了多步骤分析中粗体/列表无法显示的问题
* **DAG：折叠步骤时的滚动定位**：点击侧边栏中的步骤节点时，现在会先自动展开折叠的步骤区域，再滚动定位
* **DAG：历史记录显示步骤卡片**：历史轮次现在在展开的折叠区域中渲染步骤卡片（之前被 `hideStepCards` 隐藏）
* **DAG：重新规划时清除旧分析**：重新规划开始时，第一轮"目标未达成"的分析会立即清除，且第二轮的"正在分析结果..."加载提示可正确显示
* **UI：聊天搜索框焦点环**：为搜索输入框添加 `focus-visible:outline-none`，防止焦点环溢出
* **导出：PDF 可读性**：增加了所有 PDF 样式的行距 (leading) 和单元格内边距，提升文本可读性
* **导出：移除表情符号过滤**：移除了 DOCX 和 PDF 导出中对最终回答内容不必要的 `_strip_emoji()` 调用，保留原始内容

* **导出：RFC 5987 文件名支持**：优先使用 Content-Disposition 头中 UTF-8 编码的 `filename*`，而非普通 `filename`，修复导出对话时中日韩文件名乱码问题
* **Docker i18n 失效**：将 `messages/` 目录复制到独立输出中，并添加 `outputFileTracingIncludes`，以便 next-intl 在运行时发现 locale JSON 文件
* **Docker 启动竞态条件**：在启动 Next.js 之前，于 entrypoint 中添加 API 就绪等待循环 (Python socket 轮询) ，避免出现 `e.map is not a function` SSR 错误
* **工件：按会话 created&#95;at 排序**：按会话创建时间而不是文件 mtime 对工件进行排序 (mtime 在 Docker 卷上不可靠) 
* **OAuth：根据 window.location 端口自动检测生产环境 API URL**：浏览器自动检测现在可在生产环境中正确处理自定义端口 (不再仅限于 3000)
