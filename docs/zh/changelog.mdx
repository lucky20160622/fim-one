<div id="added">
  ### 新增
</div>

* **管理后台：审核日志 Tab** — 管理员审计页面新增"系统日志 / 审核日志"分段切换。审核日志面板调用新接口 `GET /api/admin/review-log`（支持按组织、资源类型、操作筛选，limit/offset 分页），展示时间、组织名、资源类型、资源名、操作状态（颜色编码：绿=已通过、红=已拒绝、琥珀=已提交/已重提、灰=已撤销）及操作人。全面支持国际化（中英文）。
* **平台组织**：内置组织 `platform`（ID：`00000000-0000-0000-0000-000000000001`）— 所有新用户自动加入。该组织取代了"全局"可见性概念，为所有用户提供跨组织资源共享的中央枢纽。
* **资源订阅**：用户现可订阅/取消订阅组织市场中的共享资源（智能体、连接器、知识库、MCP 服务器）。新增 `resource_subscriptions` 数据库表追踪订阅关系。
* **市场 API**：新增端点用于浏览和管理订阅：
  - `GET /api/market` — 浏览当前组织中的可用资源
  - `POST /api/market/subscribe` — 订阅资源
  - `DELETE /api/market/unsubscribe` — 取消订阅资源
  - `GET /api/market/subscriptions` — 列出用户当前订阅
* **市场页面**：门户中新增 `/market` 页面，用于浏览和管理对组织资源的订阅。
* **智能体自动发现**：智能体新增 `discoverable` 布尔字段。`discoverable=true` 的智能体可通过 `call_agent` 工具被 LLM 自动发现并调用（仅限单级委派，无递归）。
* **子智能体绑定**：智能体新增 `sub_agent_ids` 字段 — 可调用子智能体的显式白名单，实现细粒度访问控制。
* **CallAgentTool**：内置工具，用于将任务委派给专家智能体，支持结构化输入验证和结果处理。
* **MCP 服务器凭据**：新增 `mcp_server_credentials` 表，允许用户为 MCP 服务器设置环境变量和 HTTP 请求头的个人覆盖。
* **MCP 凭据端点**：`PUT /api/mcp-servers/{id}/my-credentials` 允许用户为共享 MCP 服务器存储个人凭据覆盖。
* **MCPServer 上的 `allow_fallback`**：布尔字段控制凭据回退行为 — 当为 `true` 时，缺失的用户凭据会回退到所有者默认值。
* **连接器/知识库切换**：新增资源状态管理端点：
  - `POST /api/connectors/{id}/toggle` — 暂停/恢复连接器
  - `POST /api/knowledge-bases/{id}/toggle` — 暂停/恢复知识库
* **独立知识库对话**：`ChatStreamRequest` 和 `Conversation` 新增 `kb_ids` 字段 — 用户现可直接与知识库聊天，无需将其绑定到智能体。
* **平台组织保护**：平台组织无法删除，用户也无法自行离开。确保所有用户保持与组织枢纽的连接。
* **发布审核 UI（前端）**：组织级发布审核系统前端 — 组织编辑对话框中的审核开关、带类型/状态筛选和通过/拒绝工作流的 ReviewsSheet、智能体卡片上的发布状态徽章（待审核/已通过/已拒绝，带拒绝原因 tooltip）、发布对话框中的审核提示（当目标组织需要审核时）、已拒绝资源的重新提交操作、组织管理员/所有者的"审核管理"下拉菜单。所有资源类型（智能体、连接器、知识库、MCP 服务器）新增 `publish_status`、`reviewed_by`、`reviewed_at`、`review_note` 类型字段。完整国际化支持（中英文）。

<div id="changed">
  ### 变更
</div>

* **可见性简化**：资源可见性从 `personal | org | global` 简化为 `personal | org`。"全局"资源现在发布到平台组织，保持向后兼容的同时简化了可见性模型。
* **发布接口统一**：全部 4 种资源类型（智能体、连接器、知识库、MCP 服务器）现在仅支持 `scope="org"` 用于发布。不再有 `scope="global"` 或管理员专属全局发布端点。
* **自动平台组织注册**：新用户在注册时自动添加到平台组织，确保立即获得共享组织资源的访问权限。
* **管理后台清理**：移除"全局智能体"和"全局 MCP"管理标签页，因为全局资源现在通过平台组织进行管理。

<div id="removed">
  ### 移除
</div>

* 从智能体和 MCPServer 模型及所有 API 响应中移除 `is_global` 字段
* 从智能体和 MCPServer 中移除 `cloned_from_agent_id` 和 `cloned_from_mcp_server_id` 字段
* 移除全局智能体管理端点（`/api/admin/global-agents/*`）
* 移除全局 MCP 服务器端点（`/api/admin/mcp-servers/*`）
* 从资源、API 契约和前端逻辑中移除所有 `visibility="global"` 概念引用

<div id="v075-2026-03-12">
  ## [v0.7.5] - 2026-03-12
</div>

<div id="added">
  ### 新增
</div>

* **自由模式切换 + 逐条模式追踪**：用户现可在会话中自由切换 Auto/React/DAG 模式，无需开新对话。每条助手消息在 `metadata.mode` 中记录实际执行模式。历史消息逐条检测并使用正确的渲染引擎（优先读 metadata，回退至 SSE 事件指纹识别）。移除侧栏模式图标。每轮显示模式标签。
* **三级模型角色 (F5)**：General、Fast、Reasoning 三个模型层级的独立环境变量配置。Fast 模型不再继承主模型的推理相关设置。新增环境变量：`FAST_LLM_API_KEY`、`FAST_LLM_BASE_URL`、`FAST_LLM_TEMPERATURE`、`REASONING_LLM_MODEL`、`REASONING_LLM_API_KEY`、`REASONING_LLM_BASE_URL`、`REASONING_LLM_TEMPERATURE`、`REASONING_LLM_CONTEXT_SIZE`、`REASONING_LLM_MAX_OUTPUT_TOKENS`、`REASONING_LLM_EFFORT`、`REASONING_LLM_BUDGET`
* **结构化步骤输出 (F2)**：`StepOutput` 数据类替代 DAG 步骤中的纯字符串结果，为后续支持步骤间的结构化数据和产物传递奠定基础
* **工具缓存 (F3)**：单次 DAG 执行内的相同工具调用会被缓存，避免冗余 API 调用。按 key 的异步锁防止惊群效应。通过 `DAG_TOOL_CACHE` 配置（默认开启）
* **逐步骤验证 (F4)**：可选的基于 LLM 的质量检查，在每个 DAG 步骤完成后执行。验证失败时携带反馈重试一次。通过 `DAG_STEP_VERIFICATION` 配置（默认关闭）
* **自动路由 (F1)**：Fast LLM 将查询分类为 ReAct 或 DAG 执行模式。新增 `/api/auto` 端点及 SSE 路由事件。前端三态模式切换（自动/标准/规划器）。通过 `AUTO_ROUTING` 配置（默认开启）
* **设计理念文档**：新增 `docs/design-philosophy.mdx`，记录架构决策——为何 Handoff、Backflow、Teams 和 agent_hint 被有意排除在 DAG 执行之外

* **连接器凭据加密 + 用户级覆盖** — Bearer Token、API Key、Basic 认证等敏感字段现已从 `auth_config` 中提取，单独存储至新的 `connector_credentials` 表，支持通过 `CREDENTIAL_ENCRYPTION_KEY` 进行 Fernet 加密。所有 API 响应中的敏感字段均已自动屏蔽。新增端点 `GET/PUT/DELETE /api/connectors/{id}/my-credentials` 允许用户存储个人凭据覆盖。连接器新增 `allow_fallback` 字段，控制当用户无个人凭据时是否回退到连接器所有者的默认凭据。Chat 工具加载时优先使用用户个人凭据，`allow_fallback=true` 时自动回退到默认凭据。迁移文件 `s9u1w3y5a678` 自动将现有凭据从 `auth_config` 迁移至新表。

<div id="fixed">
  ### 已修复

* Fast 模型（`get_fast_llm()`）不再继承主模型配置中的 `LLM_API_KEY`、`LLM_BASE_URL`、`LLM_TEMPERATURE` 及推理相关设置
* 后端 Pydantic schema 现已接受 `"auto"` 模式用于会话和 Agent 执行模式，默认值从 `"react"` 改为 `"auto"`
* 前端 SSE 路由事件字段名与后端对齐（`chosen_mode`→`mode`、`reason`→`reasoning`）——此前自动路由无论分类器结果如何都静默回退到 ReAct
* Agent 设置表单默认执行模式从 `"react"` 改为 `"auto"` 以匹配后端 schema 默认值

<div id="changed">
  ### 变更
</div>

* 模式切换标签现已包含引擎名称："标准 (ReAct)"、"规划 (DAG)"，以增加清晰度
* **骨架屏升级** — 所有列表/网格页面在初始加载时现在显示与布局匹配的骨架占位符，替代原先居中的转圈 Spinner。涉及页面：`/agents`（3列 `AgentCard` × 6）、`/chats`（`ListRow` × 8）、`/kb`（3列 `KbCard` × 6）、`/kb/[id]`（含页头 + 工具栏 + `TableRow` × 5 的完整结构骨架）、`/connectors` 连接器标签页（3列 `ConnectorCard` × 6）、`/artifacts`（4列 `ArtifactCard` × 8）。`skeleton.tsx` 新增三个复合组件：`ConnectorCard`、`ArtifactCard`、`TableRow`。Tab 切换及次级加载状态（加载更多、搜索、弹窗）保留 Loader2 转圈。

<div id="added">
  ### 新增

* **管理后台：`json_mode_enabled` 每模型开关** — 每个模型配置现在在 Admin → Models → 高级设置 中新增了"JSON 模式"开关（默认开启）。关闭后，可阻止 LiteLLM 注入 assistant prefill 消息（`{"role":"assistant","content":"{"}`），避免 AWS Bedrock 在 Agent 第 2 次迭代时抛出 `ValidationException`。ENV 配置的模型通过新增的 `LLM_JSON_MODE_ENABLED` 环境变量控制。所有 URL 模式启发式判断（`is_bedrock`、`is_anthropic_relay`）已完全移除，能力标志改为显式声明。包含 Alembic 迁移（`q7s9u1w3y456`）。
* **Admin 模型表单 UX 优化** — 重构添加/编辑模型对话框：默认仅显示名称、模型名称和 API Key 三个必填项；其余所有配置（API 地址、提供商、Temperature、最大输出 Token、上下文大小、JSON 模式）折叠在"高级"开关下，减少常规场景的视觉干扰。
* **设置页：组织标签页** — 用户可在设置页直接创建、加入和管理组织。功能包括：创建组织对话框（名称、自动生成标识符、描述、Emoji 图标）、编辑对话框（仅所有者）、成员管理抽屉（所有者/管理员/成员角色，带颜色徽章），以及行内操作（非所有者可退出，所有者可删除）。所有组织操作调用用户侧 `/api/orgs` 端点。完整 i18n 支持（中英文）。
* **SSE 协议 v2 — 实时回答流式传输**：后端 `StreamChunk` 新增 `delta_reasoning` 和 `usage` 字段；LLM 请求包含用量追踪；ReAct `_run_native()` 现在通过 `stream_chat` 进行流式传输。SSE 协议在回答后立即发送 `done`，随后将 `suggestions`、`title` 和 `end` 作为独立事件发送。LLM 执行期间提前释放 DB 会话。SQLite 连接池大小从 5 提升至 20
* **AI Builder 扩展 — 7 个新工具 + `is_builder` 标志**：三个新连接器构建器工具（`GetSettings`、`TestConnection`、`ImportOpenAPI`）和四个新智能体构建器工具（`ListConnectors`、`AddConnector`、`RemoveConnector`、`SetModel`）。智能体新增 `is_builder` 布尔字段及 Alembic 迁移。构建器提示词在会话创建时自动刷新。OpenAPI 导入新增 SSRF 防护
* **SSE v2 前端 — 流式光标、DAG 轮次快照、UI 优化**：前端 hooks 解析 `end` / `suggestions` / `title` 事件。DAG 重新规划显示之前失败轮次为可折叠卡片。CSS 点脉冲流式光标。DAG 布局与步骤状态解耦。侧边栏折叠时新增新建对话按钮。连接器编辑器溢出修复。`font-mono` 替换为 `tabular-nums`
* **AI Builder 概念文档页面**：新增 `docs/concepts/ai-builder.mdx`，含连接器和智能体构建器指南，附中文翻译
* **Builder Dialog — Connector 管理 ReAct 智能体**：将 AI 助手面板从单次 LLM 调用升级为完整 ReAct 循环。"高级构建器"按钮打开全屏对话框，左侧为嵌入式 PlaygroundPage，右侧为实时 Action 预览（3 秒轮询）。六个 Connector Builder 工具（`connector_list_actions`、`connector_create_action`、`connector_update_action`、`connector_delete_action`、`connector_update_settings`、`connector_test_action`）直接操作数据库，Builder Agent 激活时自动注入。`POST /api/builder/session` 按 Connector 缓存复用 Builder Agent，保留对话历史
* **双数据库支持（SQLite + PostgreSQL）**：`DATABASE_URL` 现在同时支持 SQLite（`sqlite+aiosqlite:///...`）和 PostgreSQL（`postgresql+asyncpg://...`）连接字符串。SQLite 仍为零配置默认值；PostgreSQL 推荐用于生产环境和多进程部署。Docker Compose 自动配置 PostgreSQL 服务并附带健康检查。Alembic 迁移可感知数据库方言（json_extract vs `::json->>'key'`）。管理员健康看板显示实际数据库引擎和版本
* **模型配置文档**：新增 `docs/configuration/models.mdx` 页面，记录模型配置参考，包括各提供商的扩展思考/推理设置
* **扩展思考 / 推理支持**：新增 `LLM_REASONING_EFFORT`（low/medium/high）和 `LLM_REASONING_BUDGET_TOKENS` 环境变量，支持具备扩展思考能力的模型（OpenAI o 系列、Gemini 2.5+、Claude）。OpenAI/Gemini 发送 `reasoning_effort` 参数，Anthropic（自动检测）发送 `thinking`。从多个提供商字段名（DeepSeek R1、Claude 代理）中提取 `reasoning_content`，并在 UI 的"思考中"步骤中展示
* **管理后台：工具禁用**：新增管理后台工具面板，支持逐工具启用/禁用切换（存储为 `disabled_builtin_tools` 设置）。禁用的工具在运行时会从聊天工具解析中过滤掉。`/api/tools/catalog` 端点返回 `disabled` 标志，工具页面 UI 显示禁用状态
* **管理后台：MCP 服务器移至连接器页面**：MCP 服务器管理从工具页面移至连接器页面，作为新标签页。`MCPServersSection` 抽取为可复用组件，支持命令式操作引用
* **管理后台：健康看板显示推理强度**：管理员健康检查看板现在显示推理强度标签
* **导出：PDF 格式**：对话现在可以导出为 PDF 文档（需要 `reportlab` + `Markdown` 额外依赖）
* **start.sh：WORKERS 环境变量**：设置 `WORKERS=N` 以在 portal/api 模式下以多 worker 运行 uvicorn

* **Docker Compose 部署**：单镜像 (API + Frontend) 、用于数据持久化的命名卷，以及独立模式的 Next.js 输出模式
* **管理后台：登录历史记录**：认证端点现在会将每次登录尝试 (IP 地址、User-Agent、成功/失败及原因) 记录到 `LoginHistory` 表中，用于安全审计
* **管理后台：文件浏览器**：新增管理后台文件浏览器端点，用于列出和下载用户上传的文件，并支持分页
* **管理后台：增强的智能体/知识库视图**：管理后台智能体列表和知识库视图现已提供 `model_name`、`tools`、`kb_ids` 和 `embedding_model` 字段，提升可见性
* **聊天：敏感词拦截**：ReAct 和 `inject_message` 聊天端点现在都会执行敏感词过滤，拦截包含已配置敏感词的消息
* **SSE：结构化错误处理**：`useSSE` hook 中的 `onError` 回调已从普通字符串升级为 `ApiError` 对象，以提供更丰富的错误上下文 (错误代码、参数)
* **UI：`formatTokens` 工具**：新增用于显示令牌计数的格式化辅助工具，支持基于区域设置的数字格式化
* **UI：`Checkbox` 和 `FadeIn` 组件**：新增 shadcn 风格的复选框组件和淡入动画包装组件

<div id="changed">
  ### 变更
</div>

* **测试：NativeToolFakeLLM 更新为流式 ReAct**：测试用假 LLM 现在生成 `StreamChunk` 序列，以匹配新的基于流式的 `_run_native()` 循环
* **LLM 层**：使用 [LiteLLM](https://github.com/BerriAI/litellm) 替换了直连 `AsyncOpenAI` 客户端，实现通用提供商支持。所有提供商特定的条件逻辑（Anthropic thinking、`max_completion_tokens`、GPT-5 tools 变通方案）已全部移除。Anthropic 模型现在通过 LiteLLM 的原生 API 路由，正确支持 `reasoning_content`（扩展思考）
* **工具页面简化**：工具页面现在仅显示内置工具（不再有标签页切换）；MCP 管理已移至连接器页面
* **Redis 中断代理**：通过 Redis Pub/Sub 实现跨进程中断/注入中继。设置 `REDIS_URL` 时自动启用，否则回退到进程内队列。Docker Compose 现已包含 Redis 服务（自动配置，无需用户操作）
* **导出：默认详情级别**：导出对话框现在默认为摘要模式（不含执行详情），而非完整模式
* **工件：按文件 mtime 排序**：list-all-artifacts 现在使用每个文件的 mtime 进行精确排序（还原了之前使用会话创建时间的权宜做法）
* **聊天：异步图片加载**：`_load_image_data_urls` 现在是异步函数，通过 `asyncio.to_thread` 将文件读取卸载到线程，避免阻塞事件循环
