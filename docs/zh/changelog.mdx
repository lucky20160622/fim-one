此项目的所有重要更改都记录在此。

格式遵循 [Keep a Changelog](https://keepachangelog.com/)。版本对应于 [Roadmap](/roadmap) 里程碑。## [未发布]### 新增

- **管理员：审查日志审计标签页** — 管理员审计页面现在具有分段式"系统日志/审查日志"切换。审查日志面板查询新的 `GET /api/admin/review-log` 端点（过滤条件：组织、资源类型、操作；限制/偏移分页），并显示包含时间、组织名称、资源类型、资源名称、操作徽章（颜色编码：绿色=已批准、红色=已拒绝、琥珀色=已提交/已重新提交、灰色=未发布）和操作者的表格。完整的 i18n（EN + ZH）。
- **平台组织**：内置组织 `platform`（ID：`00000000-0000-0000-0000-000000000001`）— 所有新用户自动加入。此组织替代了"全局"可见性概念，为跨所有用户共享资源提供了中央枢纽。
- **资源订阅**：用户现在可以从组织市场订阅/取消订阅共享资源（agents、connectors、知识库、MCP servers）。新的 `resource_subscriptions` 数据库表跟踪订阅关系。
- **市场 API**：用于浏览和管理订阅的新端点：
  - `GET /api/market` — 浏览当前组织中的可用资源
  - `POST /api/market/subscribe` — 订阅资源
  - `DELETE /api/market/unsubscribe` — 取消订阅资源
  - `GET /api/market/subscriptions` — 列出用户的当前订阅
- **市场页面**：Portal 中的新 `/market` 页面，用于浏览和管理对组织资源的订阅。
- **Agent 自动发现**：agents 上的新 `discoverable` 布尔字段。`discoverable=true` 的 agents 可以通过 `call_agent` 工具由 LLMs 自动发现和调用（单级委派，无递归）。
- **子 Agent 绑定**：agents 上的新 `sub_agent_ids` 字段 — 可调用子 agents 的显式白名单，用于细粒度访问控制。
- **CallAgentTool**：内置工具，将任务委派给专家 agents，具有结构化输入验证和结果处理。
- **MCP Server 凭证**：新的 `mcp_server_credentials` 表为 MCP servers 启用每用户环境变量和 HTTP 标头覆盖。
- **MCP 凭证端点**：`PUT /api/mcp-servers/{id}/my-credentials` 允许用户为共享 MCP servers 存储个人凭证覆盖。
- **MCPServer 上的 `allow_fallback`**：布尔字段控制凭证回退行为 — 当为 `true` 时，缺失的用户凭证回退到所有者默认值。
- **Connector/KB 切换**：资源状态管理的新端点：
  - `POST /api/connectors/{id}/toggle` — 暂停/恢复 connector
  - `POST /api/knowledge-bases/{id}/toggle` — 暂停/恢复知识库
- **独立 KB 对话**：`ChatStreamRequest` 和 `Conversation` 上的新 `kb_ids` 字段 — 用户现在可以直接与知识库聊天，无需将其绑定到 agent。
- **平台组织保护**：平台组织无法删除，用户无法自行离开。确保所有用户保持与组织枢纽的连接。
- **发布审查 UI（前端）**：组织级发布审查系统前端 — 组织编辑对话框中的审查切换、带有类型/状态过滤器和批准/拒绝工作流的 ReviewsSheet、agent 卡片上的发布状态徽章（待处理/已批准/已拒绝）及拒绝原因工具提示、发布对话框中的审查通知（当目标组织需要审查时）、被拒绝资源的重新提交操作、组织管理员/所有者的"审查管理"下拉菜单。所有资源类型（agents、connectors、KBs、MCP servers）获得 `publish_status`、`reviewed_by`、`reviewed_at`、`review_note` 类型字段。完整的 i18n（EN + ZH）。### 已修复

- **切换 `is_active` 时无需重新审核**：在代理、连接器、知识库或 MCP 服务器上切换 `is_active` 不再将 `publish_status` 从 `approved` 恢复为 `pending_review`。`check_edit_revert` 守卫现在仅在实际内容字段更改时触发（不仅仅是 `is_active`）。### 已更改

- **可见性简化**：资源可见性从 `personal | org | global` 简化为 `personal | org`。"全局"资源现在发布到 Platform 组织，保持向后兼容行为，同时简化可见性模型。
- **发布界面统一**：所有 4 种资源类型（Agent、Connector、Knowledge Base、MCP Server）现在仅支持 `scope="org"` 用于发布。不再支持 `scope="global"` 或仅限管理员的全局发布端点。
- **自动 Platform 注册**：新用户在注册时自动添加到 Platform 组织，确保立即访问共享的组织资源。
- **管理面板清理**：删除了"全局 Agent"和"全局 MCP"管理选项卡，因为全局资源现在通过 Platform 组织进行管理。### 已移除
