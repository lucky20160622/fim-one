### 新增

- **工作流级别超时** — 引擎中新增 `workflow_timeout_ms` 参数，超时时自动取消所有待处理/运行中的节点，具有有界异步等待以实现响应式超时检测。
- **使用上次输入重新运行** — "再次运行"按钮使用上一次运行的值预填充输入表单。
- **运行历史状态筛选** — 下拉菜单按已完成/失败/已取消/运行中筛选工作流运行。
- **工作流版本历史** — 蓝图保存时自动版本控制，提供列表/获取/恢复 API 端点以还原到之前的版本。
- **验证端点** — `POST /validate` 用于结构蓝图验证，无需执行。
- **干运行模式** — `POST /run` 使用 `dry_run=true` 返回执行计划的 JSON，不执行。
- **13 个条件分支路由测试** — 全面覆盖分支、合并、嵌套和分类器路由。
- **单节点执行统计面板** — 新统计面板（`node-stats-panel.tsx`）显示单节点成功率、平均耗时和失败计数，按最差优先排序。集成到工作流工具栏。新增 `getNodeStats` API 客户端方法和 `NodeStatEntry` 类型供前端使用。
- **第 3 阶段节点：SubWorkflow 和 ENV** — 两种新节点类型，包含后端执行器、解析器验证和 14 个新测试（总计：304 → 306）：
  - **SubWorkflow**：用于嵌套工作流编排的存根执行器 — 选择目标工作流并映射输入/输出变量。
  - **ENV**：在运行时读取加密的环境变量，支持密钥选择和回退默认值。
- **SubWorkflow 和 ENV 前端组件** — 完整的前端集成：`sub-workflow-node.tsx` 和 `env-node.tsx`，包含配置表单、调色板条目、小地图颜色以及所有类型映射中的节点类型记录条目。
- **两个新工作流模板（共 14 个）** — "安全 API 集成"（ENV → HTTP 带认证头）和"子工作流编排"（LLM → SubWorkflow → 总结）。
- **键盘快捷键对话框** — 按 `?` 在对话框覆盖层中查看所有可用的键盘快捷键。
- **节点输入预览捕获** — 执行事件现在包含前驱节点输出的截断 JSON，用于调试。
- **工作流分析端点** — `GET /analytics` 返回总运行数、状态分布、成功率、p50/p95/p99 耗时百分位数、每日趋势和最常失败的节点。
- **分析仪表板面板** — 统计卡片、状态分布条、纯 CSS 每日图表、百分位数条和失败节点表，支持 7 天/14 天/30 天范围选择器。
- **模板库对话框** — 工作流模板的可视化网格浏览，支持分类筛选。
- **标记边** — 条件和分类器分支名称显示为边上的标签。
- **工作流变量面板** — 定义、编辑和删除工作流级别变量，支持类型/默认值/描述。

### 已添加

- **工作流引擎节点重试** — 按节点重试，具有指数退避。节点可在其数据配置中指定 `retry_count` 和 `retry_delay_ms`。失败的节点最多重试 N 次，延迟上限为 60 秒。发出带有尝试次数和先前错误的 `node_retrying` SSE 事件。在重试等待期间遵守取消事件。
- **工作流变量存储表达式评估** — 通过 simpleeval 进行安全的 Python 表达式评估。`evaluate_expression()` 方法公开存储变量（不包括环境变量），支持常见内置函数（len、str、int、float、bool、abs、min、max、sum、round、sorted）。另外还有 `has()`、`delete()`、`keys()` 辅助方法。
- **工作流第 2 阶段节点：HumanIntervention** — 暂停待批准节点（自动批准存根）。读取 prompt_message（支持 {{variable}} 插值）、assignee、timeout_hours。Sky-500 主题的前端节点，具有提示预览和 @assignee 徽章。配置面板包含提示文本区、受理人输入、超时字段。
- **工作流模板扩展至 12 个** — 四个新模板：Human Approval Pipeline（LLM → HumanIntervention → ConditionBranch → 双 End）、Multi-Step Data Processing（CodeExecution → ConditionBranch → Transform → ListOperation）、Knowledge-Enhanced Agent（QuestionUnderstanding → KnowledgeRetrieval → LLM）、Iterative Review Loop（Loop → LLM → VariableAssign）。
- **工作流窗格上下文菜单** — 在空白画布上右键单击以访问粘贴（剪贴板为空时禁用）、全选、适应视图和自动布局操作。判别联合状态分离节点和窗格上下文菜单。
- **工作流节点上下文菜单** — 右键单击任何节点以访问配置、复制、复制和删除操作。对于 Start/End 单例节点，删除操作被隐藏。菜单在外部单击或滚动时自动关闭。
- **工作流第 2 阶段模板** — 两个新的内置模板（共 8 个）：List Processing Pipeline（Start → ListOperation → Transform → End）和 Enhanced Question Answering（Start → QuestionUnderstanding → LLM → End）。
- **工作流第 2 阶段验证** — 扩展的蓝图验证警告，针对 ListOperation（缺少输入/表达式）、Transform（缺少输入/空操作）、Iterator（缺少列表变量）、Loop（缺少条件）、DocumentExtractor、QuestionUnderstanding（缺少输入）、ParameterExtractor（缺少输入文本/空参数）。修复：排序操作不再错误地要求表达式。
- **工作流节点注释** — 按节点注释功能。单击任何节点配置面板中的"Note"以添加自由格式注释，显示为画布上节点下方的斜体文本。自动扩展文本区，存储在 `data.note` 中。
- **工作流吸附网格** — 16px 吸附网格，与点状背景图案匹配，用于精确的节点对齐。
- **工作流连接验证** — 除了现有的 Start/End 限制外，还防止自连接和重复边。
- **工作流缩放感知拖放** — 节点调色板拖放使用 `screenToFlowPosition()` 在任何缩放级别下正确放置。
- **工作流 SSE 多行数据** — SSE 解析器现在处理多行 `data:` 字段并忽略 keepalive 注释行。
- **工作流键盘快捷键** — 复制 (Cmd+C)、粘贴 (Cmd+V)、复制 (Cmd+D)、全选 (Cmd+A)、Escape 取消选择。复制/粘贴生成新 ID 并偏移位置；Start/End 节点被排除在复制之外以维持单例约束。
- **工作流内联描述编辑** — 单击工具栏中的描述行以打开弹出文本区，用于编辑工作流描述而无需离开页面。
- **工作流 ConditionBranch LLM 模式** — 适当的基于 LLM 的条件评估，替代了天真的"匹配第一个"存根。从条件标签构建分类提示，调用快速 LLM，并使用不区分大小写 + 模糊匹配。
- **反应式边标签** — 条件/分类器分支的边标签现在使用 `useNodesData()` 在配置面板中编辑条件或类时进行实时更新。
- **工作流统计面板** — 基于工作表的统计显示，显示总运行次数、已完成/失败/已取消计数、带有可视进度条的成功率、平均持续时间和相对格式化的最后运行时间。从工具栏中的 BarChart3 图标触发。
- **第 2 阶段工作流节点：DocumentExtractor 和 QuestionUnderstanding** — 两个更多节点类型：
  - **DocumentExtractor**：文档文本处理的存根执行器 — 支持 text/base64 输入类型和 4 种提取模式（full_text、pages（带 page_range）、metadata（带 word/char/line/page 计数）、tables（带 markdown 表检测））。URL 输入延迟到后期实现。
  - **QuestionUnderstanding**：基于 LLM 的问题预处理，具有 4 种模式 — rewrite（清晰度）、expand（上下文 + 子问题）、classify（intent/topic/confidence JSON）、decompose（子问题数组）。使用快速 LLM，支持可选的自定义系统提示覆盖。
  - 完整前端：节点组件、配置面板、调色板条目（QU 的 AI 类别、DE 的 Data 类别）、变量选择器支持。
- **第 2 阶段工作流节点：ListOperation 和 Transform** — 两个数据处理节点类型：
  - **ListOperation**：8 个列表操作（filter、map、sort、slice、flatten、unique、reverse、length），支持 filter/map/sort 的 simpleeval 表达式。支持 JSON 字符串输入自动解析。
  - **Transform**：基于管道的数据转换，具有 6 种操作类型（json_path、type_cast、format、regex_extract、string_op、math_op）。操作按顺序应用 — 一个的输出馈入下一个。
  - 完整前端：节点组件、带有操作特定编辑器的配置面板、调色板条目、变量选择器支持。
- **第 2 阶段工作流节点：Loop 和 ParameterExtractor** — 两个更多高级节点类型：
  - **Loop**：While 条件循环，具有 simpleeval 表达式评估、可配置的循环变量名称和 max_iterations 安全限制。前端包含条件输入、变量名称和迭代限制配置。
  - **ParameterExtractor**：从文本进行基于 LLM 的结构化参数提取。定义类型化参数（name/type/description/required），提供输入文本引用，并可选地添加提取说明。返回解析的 JSON。前端包含参数列表编辑器和提取提示文本区。
- **第 2 阶段工作流节点：Iterator 和 VariableAggregator** — 两个新的高级节点类型：
  - **Iterator**：使用可配置的 item/index 变量名称和 max_iterations 安全限制迭代列表。支持原生列表、JSON 字符串解析和模板变量引用。
  - **VariableAggregator**：通过四种模式合并来自多个上游分支的输出 — 收集到列表、连接字符串（自定义分隔符）、合并字典或取第一个非空值。
  - 完整前端集成：节点组件、调色板条目、配置面板表单、小地图颜色、变量选择器支持。后端执行器在引擎中注册。
- **工作流模板扩展** — 添加了两个新的内置模板（共 6 个）：Data Processing Pipeline（Start → CodeExecution → TemplateTransform → End）和 Agent with Knowledge Retrieval（Start → KnowledgeRetrieval → Agent → End）。修复了新模板上缺失的 `icon` 字段。为"intermediate"和"advanced"模板类别添加了缺失的 i18n 键。
- **工作流 ENV 变量对话框** — 按工作流加密环境变量管理 UI。键值编辑器，具有掩码显示、重复键检测和内联验证。连接到 `GET/PUT /api/workflows/{id}/env` 端点。可通过工具栏下拉菜单访问。
- **工作流自动保存** — 5 秒去抖动自动保存计时器，支持立即 Cmd+S 覆盖。保存状态指示器显示"Saving..."、"Unsaved changes"或"Saved Xs ago"，30 秒自动刷新。
- **工作流列表搜索和过滤** — 按名称/描述的客户端搜索、状态过滤按钮（All/Draft/Active）、排序下拉菜单（Recently updated/created/Name A-Z）和无结果的空状态。
- **工作流彩色编码小地图** — 小地图节点颜色现在与每个节点类型的类别颜色匹配（绿色=Start、红色=End、蓝色=LLM、橙色=Condition 等）。
- **智能体技能系统** — 按需加载智能体说明的技能：`Skill` 模型（name、description、content、script），具有完整 CRUD、发布审查和可见性生命周期；`read_skill(name)` 内置工具用于渐进式披露（~80% 令牌减少）；技能存根注入到系统提示中（~15 tokens/skill）；每个智能体的 `compact_instructions` 字段用于自定义 ContextGuard 压缩策略。
- **技能 UI** — 技能列表页面、全页编辑器（content/script/settings 选项卡）、SkillCard 组件、侧边栏导航、智能体编辑器技能选择器，带有紧凑说明文本区。
- **工作流发布流** — 工作流的发布/取消发布/重新提交端点，与现有智能体发布模式匹配。Organization 模型上的新 `review_workflows` 标志（带 Alembic 迁移）以启用组织级工作流审查门控。发布操作连接到工作流卡和工具栏 UI。
- **工作流蓝图系统** — 用于设计和执行多步自动化蓝图的可视工作流编辑器：
  - **12 个节点类型**：Start、End、LLM、ConditionBranch、QuestionClassifier、Agent、KnowledgeRetrieval、Connector、HTTPRequest、VariableAssign、TemplateTransform、CodeExecution — 每个都有自定义 React Flow 节点组件、类型特定图标和专用配置面板表单。
  - **执行引擎**：`WorkflowEngine`，具有用于节点执行顺序的拓扑排序、基于 `asyncio.Semaphore` 的并发控制，以及基于 `active_handles` 激活/停用下游边的条件/分类器分支。
  - **SSE 流**：通过服务器发送事件的实时工作流执行进度 — 发出 `run_started`、`node_started`、`node_completed`、`node_failed`、`node_skipped`、`run_completed` 和 `run_failed` 事件。长运行工作流的 SSE keepalive；客户端断开连接检测触发取消。
  - **变量存储**：线程安全的 `VariableStore`，具有 `{{node_id.output}}` 插值、平面名称回退解析、用于加密环境变量的 `env.*` 命名空间、`snapshot_safe()`（排除机密）和 `list_available_variables()`（用于前端选择器）。
  - **边标签**：`WorkflowEdgeDef` 携带 `source_handle` 和 `target_handle` 用于条件/分类器分支，启用标记边，直观指示每个连接代表哪个分支路径。
  - **变量选择器组合框**：可搜索的组合框（基于 `Command` 的弹出菜单），用于从上游节点插入 `{{node_id.output}}` 引用 — 按节点类型分组，带有图标，集成到所有提示/模板/body 字段。
  - **运行历史工作表**：显示过去工作流执行运行的侧工作表，包含状态、持续时间、输入/输出数据和按节点结果。通过 `GET /api/workflows/{id}/runs` 分页。
  - **工作流复制**：`POST /api/workflows/{id}/duplicate` 端点和 UI 操作以克隆现有工作流（复制蓝图、schema、描述；重置状态为草稿）。
