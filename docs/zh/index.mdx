## FIM One 是什么？

FIM One 是一个与提供商无关的 Python 框架，用于构建与现有系统协作的 AI 智能体。与要求你复制逻辑的工作流构建器不同，FIM One 主动桥接你的系统——读取数据库、调用 API、推送通知——所有这些都通过统一的 AI 界面完成。

核心洞察：**三种交付模式，一个智能体核心**。

## 三种交付模式

| 模式 | 说明 | 交付方式 | 使用场景 |
|------|-----------|----------|----------|
| **独立部署** | 通用 AI 助手 — 搜索、代码、知识库 | 门户 | 聊天、代码执行、知识库问答 |
| **Copilot** | 嵌入式 AI — 与用户在现有 UI 中并行工作 | iframe / widget / embed | ERP 网页 UI 中的"财务 Copilot" |
| **Hub** | 中央跨系统编排 — 所有系统互联 | 门户 / API | 智能体查询 ERP、检查 OA、通过 Lark 通知 |

## Hub 架构

Hub 是核心差异化因素——一个中央门户，所有系统在这里与 AI 相遇：

```mermaid
flowchart LR
  ERP --> Hub
  Database --> Hub
  Lark --> Hub
  Hub --> CRM
  Hub --> OA
  Hub["FIM One Hub<br/>(AI orchestration)"] --> CustomAPI["Custom API"]
```

每个连接器都是一个标准化的桥接。智能体不知道也不关心它是在与 SAP 还是自定义 PostgreSQL 数据库通信。你的数据保留在你的系统中；FIM One 提供了跨系统编排的 AI 层。
