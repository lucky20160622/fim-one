<div id="option-a-docker-recommended">
  ## 选项 A：Docker (推荐)
</div>

无需在本地安装 Python 或 Node.js——所有内容都在容器内完成构建。

```bash
git clone https://github.com/fim-ai/fim-agent.git
cd fim-agent

# 配置 — 仅需设置 LLM_API_KEY
cp example.env .env
# 编辑 .env：设置 LLM_API_KEY（可选：LLM_BASE_URL、LLM_MODEL）

# 构建并运行（首次部署，或拉取新代码后）
docker compose up --build -d
```

打开 http://localhost:3000——首次启动时，系统会引导你创建管理员账户。就这么简单。

完成初始构建后，后续启动只需要：

```bash
docker compose up -d          # 启动（如镜像未变更则跳过重新构建）
docker compose down           # 停止
docker compose logs -f        # 查看日志
```

数据会持久化到 Docker 命名卷 (`fim-data`、`fim-uploads`) 中，容器重启后仍会保留。

**注意：** Docker 模式不支持热重载。代码变更后需要重新构建镜像 (`docker compose up --build -d`) 。如果需要在开发过程中实时重载，请使用下面的 **选项 B**。

<div id="option-b-local-development">
  ## 选项 B：本地开发
</div>

先决条件：Python 3.11+、[uv](https://docs.astral.sh/uv/)、Node.js 18+、pnpm。

```bash
git clone https://github.com/fim-ai/fim-agent.git
cd fim-agent

cp example.env .env
# 编辑 .env：设置 LLM_API_KEY

# 安装依赖
uv sync --all-extras
cd frontend && pnpm install && cd ..

# 启动（支持热重载）
./start.sh
```

| 命令               | 启动内容                                         | URL                                       |
| ---------------- | -------------------------------------------- | ----------------------------------------- |
| `./start.sh`     | Next.js + FastAPI                            | http://localhost:3000 (UI) + :8000 (API)  |
| `./start.sh dev` | 同上，但启用热重载 (Python `--reload` + Next.js HMR)  | 同上                                        |
| `./start.sh api` | 仅启动 FastAPI (无头模式，用于集成或测试)                   | http://localhost:8000/api                 |

<div id="configuration">
  ## 配置
</div>

FIM Agent 支持**任何与 OpenAI 兼容的 LLM 提供商**——包括 OpenAI、DeepSeek、Anthropic、Qwen、Ollama、vLLM 等。

| 提供商              | `LLM_API_KEY` | `LLM_BASE_URL`                 | `LLM_MODEL`         |
| ---------------- | ------------- | ------------------------------ | ------------------- |
| **OpenAI**       | `sk-...`      | *(默认)*                         | `gpt-4o`            |
| **DeepSeek**     | `sk-...`      | `https://api.deepseek.com/v1`  | `deepseek-chat`     |
| **Anthropic**    | `sk-ant-...`  | `https://api.anthropic.com/v1` | `claude-sonnet-4-6` |
| **Ollama** (本地)  | `ollama`      | `http://localhost:11434/v1`    | `qwen2.5:14b`       |

**[Jina AI](https://jina.ai/)** 可启用网页搜索/抓取、嵌入以及完整的 RAG 流程 (提供免费层) 。

最小 `.env` 配置：

```bash
LLM_API_KEY=sk-your-key
# LLM_BASE_URL=https://api.openai.com/v1   # 默认值 — 如需使用其他提供商请修改
# LLM_MODEL=gpt-4o                         # 默认值 — 如需使用其他模型请修改

JINA_API_KEY=jina_...                       # 启用网页工具 + RAG
```

有关所有配置选项的完整列表，请参阅[环境变量](/zh/configuration/environment-variables)参考文档。

<div id="production-deployment">
  ## 生产部署
</div>

### Docker（推荐）

`docker compose up -d` 一键启动所有服务——**无需手动配置任何基础设施**：

| 服务 | 用途 | 配置方式 |
|------|------|---------|
| **fim-agent** | API + 前端 | `.env`（LLM 密钥等） |
| **Redis** | 跨进程中断中继 | Compose 自动配置 |

```bash
docker compose up --build -d   # 首次 / 代码变更后
docker compose up -d           # 后续启动
docker compose logs -f         # 查看日志
docker compose down            # 停止所有服务
```

#### 多进程扩展

默认情况下，API 使用单个工作进程运行。如需处理更多并发用户，可在 `.env` 中增加进程数：

```bash
WORKERS=4   # Uvicorn 工作进程数
```

多进程要求：
- **PostgreSQL** — SQLite 为单写模式，不支持并发写入。需设置 `DATABASE_URL` 为 PostgreSQL 连接字符串。
- **Redis** — Docker Compose 已内置（自动配置），负责跨进程中断/注入消息中继。

使用 `WORKERS=1`（默认值）时，不需要 Redis 或 PostgreSQL——SQLite 完全够用。

#### Nginx 反向代理

如需 HTTPS 和自定义域名，请在前端配置 Nginx 反向代理：

```
用户 → Nginx (443/HTTPS) → localhost:3000
```

API 在内部运行于 8000 端口——Next.js 自动代理 `/api/*` 请求。只需暴露 3000 端口。

#### 代码执行沙箱

如果使用代码执行沙箱（`CODE_EXEC_BACKEND=docker`），请挂载 Docker 套接字：

```yaml
# docker-compose.yml
volumes:
  - /var/run/docker.sock:/var/run/docker.sock
```

### 脚本部署（裸机）

对于裸机服务器或自定义进程管理器，可直接使用 `./start.sh`：

```bash
./start.sh           # 生产模式
./start.sh portal    # 同上（显式指定）
./start.sh api       # 仅 API（无头模式）
```

此模式下不自动包含 Redis，系统默认运行在单进程、进程内模式——适合低流量部署。

如需在本地启用多进程 + Redis：

```bash
# 启动 Redis 实例（Docker 或系统包）
docker run -d --name redis -p 6379:6379 redis:7-alpine

# 在 .env 中添加
REDIS_URL=redis://localhost:6379/0
WORKERS=4
DATABASE_URL=postgresql+asyncpg://user:pass@localhost/fim_agent
```
