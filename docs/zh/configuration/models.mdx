FIM Agent 是**提供商无关**的——任何兼容 OpenAI 的端点都可以使用。本页帮助你为自己的使用场景选择最佳模型组合。配置详情请参阅[环境变量](/zh/configuration/environment-variables)。

<div id="how-fim-agent-uses-models">
  ## FIM Agent 如何使用模型
</div>

FIM Agent 有两个模型槽位：

| 槽位 | 环境变量 | 用途 |
|---|---|---|
| **主 LLM** | `LLM_MODEL` | 规划、分析、ReAct 智能体、复杂推理 |
| **快速 LLM** | `FAST_LLM_MODEL` | DAG 步骤执行、上下文压缩（成本更低、速度更快） |

如果未设置 `FAST_LLM_MODEL`，则回退到 `LLM_MODEL`。生产部署中，使用两个不同模型可获得最佳的成本/质量平衡。

### OpenAI

最成熟的选择。OpenAI 模型拥有最好的原生函数调用（tool-calling）支持，这直接影响智能体的可靠性。GPT-5 系列（2025年8月起）是相对于 GPT-4 的重大代际飞跃。

**推荐模型：**
- 主模型：`gpt-5.4`（最新旗舰，2026年3月——内置 computer use）或 `o3`（最佳推理准确度）
- 快速模型：`gpt-5-mini`（$0.25/$2.00 每 MTok）或 `gpt-5-nano`（最便宜，$0.05/$0.40 每 MTok）
- 旧版：`gpt-4.1`（API 中仍可用，1M 上下文，适合编码）——已于 2026年2月从 ChatGPT 退役

**推理：** 设置 `LLM_REASONING_EFFORT=medium` 即可——o 系列和 GPT-5.x 模型原生支持。o 系列需要 `max_completion_tokens` 而非 `max_tokens`，FIM Agent 会自动处理。

| 模型 | 输入 $/MTok | 输出 $/MTok | 上下文 |
|---|---|---|---|
| `gpt-5.4` | $2.50 | $15.00 | 272K |
| `o3` | $2.00 | $8.00 | 200K |
| `o4-mini` | $1.10 | $4.40 | 200K |
| `gpt-5-mini` | $0.25 | $2.00 | — |
| `gpt-5-nano` | $0.05 | $0.40 | — |

```bash
# .env — OpenAI（生产环境，启用推理）
LLM_API_KEY=sk-...
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-5.4
FAST_LLM_MODEL=gpt-5-nano
LLM_REASONING_EFFORT=medium
```

```bash
# .env — OpenAI（经济推理）
LLM_API_KEY=sk-...
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=o3
FAST_LLM_MODEL=gpt-5-nano
LLM_REASONING_EFFORT=medium
```

### Anthropic (Claude)

Claude 在细腻推理和复杂多步骤任务方面表现出色。FIM Agent 通过 Anthropic 的 OpenAI 兼容端点连接。当前最新一代为 Claude 4.6（2026年2月）。

**推荐模型：**
- 主模型：`claude-sonnet-4-6`（能力与成本的最佳平衡——$3/$15 每 MTok）
- 快速模型：`claude-haiku-4-5`（快速且便宜——$1/$5 每 MTok）
- 旗舰：`claude-opus-4-6`（最强能力，128K 最大输出——$5/$25 每 MTok）

**Base URL：** `https://api.anthropic.com/v1/`

所有当前 Claude 模型都支持扩展思考，拥有 200K 上下文窗口（1M beta 中）。

<Warning>
**兼容端点不会返回推理内容。** 设置 `LLM_REASONING_EFFORT` 后，模型*确实会*进行内部思考，但思考文本不会包含在响应中——UI 的"思考中"步骤会出现但内容为空。这是 Anthropic 兼容层的限制。如需完整的推理可见性，请使用能返回 `reasoning_content` 的第三方代理。详见[扩展思考](/zh/configuration/environment-variables#扩展思考reasoning)。
</Warning>

```bash
# .env — Anthropic Claude
LLM_API_KEY=sk-ant-...
LLM_BASE_URL=https://api.anthropic.com/v1/
LLM_MODEL=claude-sonnet-4-6
FAST_LLM_MODEL=claude-haiku-4-5
```

<div id="local-models-ollama">
  ### 本地模型 (Ollama)
</div>

在自己的硬件上完全离线运行——无需 API 密钥。Ollama 开箱即用地暴露了兼容 OpenAI 的端点。开源模型格局已发生巨大变化——Qwen 3.5、Llama 4 和 GPT-OSS（OpenAI 首批开源权重模型）均已可用。

**Base URL：** `http://localhost:11434/v1`

**按显存推荐模型：**

| 显存 | 主 LLM | 快速 LLM | 备注 |
|---|---|---|---|
| 8 GB | `qwen3.5:9b` / `gemma3:4b` | `qwen3.5:4b` | Qwen 3.5 9B 是该档位最佳选择 |
| 16 GB | `gpt-oss:20b` / `deepseek-r1:14b` | `qwen3.5:9b` | GPT-OSS 20B 针对智能体优化 |
| 24 GB | `qwen3:32b` / `deepseek-r1:32b` | `qwen3.5:9b` | Qwen 3 32B 工具调用最佳 |
| 48 GB+ | `llama3.3:70b` / `gpt-oss:120b` | `qwen3.5:14b` | 接近前沿质量 |

**工具调用最佳选择：** Qwen 3/3.5（32B+）、GLM-4.7、GPT-OSS、Mistral——这些模型都有专门的函数调用训练。14B+ 参数是可靠工具调用的最低要求；32B+ 强烈推荐。

<Warning>
**本地模型的工具调用质量差异很大。** 并非所有模型都能可靠地生成有效的函数调用。在生产环境使用前，请先用智能体工作流测试你选择的模型。一般规则：14B 为最低要求，32B+ 推荐用于智能体任务。
</Warning>

```bash

LLM_API_KEY=ollama
LLM_BASE_URL=http://localhost:11434/v1
LLM_MODEL=gpt-oss:20b
FAST_LLM_MODEL=qwen3.5:9b
LLM_CONTEXT_SIZE=32768
LLM_MAX_OUTPUT_TOKENS=8192
```

```bash
# .env — Ollama（智能体优化，24GB 显存）
LLM_API_KEY=ollama
LLM_BASE_URL=http://localhost:11434/v1
LLM_MODEL=qwen3:32b
FAST_LLM_MODEL=qwen3.5:9b

LLM_MAX_OUTPUT_TOKENS=8192
```

<div id="configuration-strategy">
  ## 配置策略
</div>

### 主模型 vs 快速模型：何时拆分

- **拆分使用**：当主模型昂贵或慢时（如 `gpt-5.4` + `gpt-5-nano`）。DAG 模式会运行多个并行步骤——使用更便宜的快速模型可节省大量成本。
- **使用同一模型**：当模型本身就便宜时（如两个都用 `deepseek-chat`）。管理两个模型的额外开销不值得。

### 何时启用推理

- **启用**：复杂的分析任务、多步骤规划、需要仔细判断的任务
- **禁用**（默认）：常规任务、简单问答、成本敏感的部署
- 推理通常会使每次请求的成本增加 2-5 倍——`medium` 级别是个好的起点

### 上下文窗口设置

将 `LLM_CONTEXT_SIZE` 设置为与模型的实际窗口匹配：

| 模型 | 上下文窗口 |
|---|---|
| GPT-5.4 | 272K |
| o3 / o4-mini | 200K |
| Claude Sonnet 4.6 | 200K（1M beta） |
| Gemini 2.5 Pro | 1M |
| Gemini 3.1 Pro | 1M |
| DeepSeek V3.2 | 128K |
| Qwen 3.5 Plus | 1M |
| 本地模型 (Ollama) | 4K–128K（视模型而定） |

对于本地模型，请显式设置 `LLM_CONTEXT_SIZE` 和 `LLM_MAX_OUTPUT_TOKENS`——默认值假设的是云端规模的上下文窗口，本地模型无法支持。
