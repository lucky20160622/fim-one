Die gesamte Konfiguration erfolgt über `.env`. Kopieren Sie `example.env` und füllen Sie Ihre Werte aus:

```bash
cp example.env .env
```## Konfigurationsebenen

Jede Integration hat eine Konfigurationsebene, die ihre Wichtigkeit angibt:

| Level | Bedeutung | Verhalten wenn nicht konfiguriert |
|-------|---------|------------------------------|
| **Required** | Kernabhängigkeit des Systems | System gibt einen Fehler aus — Chat und primäre Funktionen funktionieren nicht |
| **Recommended** | Bedeutender Feature-Enabler | Elegante Verschlechterung — das Feature ist sichtbar nicht verfügbar, aber das System läuft |
| **Optional** | Verbesserungsfähigkeit | Transparente Verschlechterung — System funktioniert einwandfrei, Fähigkeit ist einfach nicht vorhanden |

> **Hinweis**: Von Admin konfigurierte Modelle (Admin → Models-Seite) können LLM-Umgebungsvariablen ersetzen. Die Integritätsprüfung berücksichtigt beide Quellen.

---## Frontend (Nur lokale Entwicklung)

Das Frontend hat eine separate Env-Datei **nur für lokale Entwicklung**: `frontend/.env.local`.

> **Diese Datei wird NICHT in Docker verwendet.** Im Docker-Container leitet Next.js `/api/*` intern an das Python-Backend weiter (Port 8000 ist container-intern), daher ist keine Frontend-Env-Datei erforderlich.

Für lokale Entwicklung funktionieren die Standardwerte sofort — Sie müssen `frontend/.env.local` **nicht** erstellen, es sei denn, Ihr Backend läuft auf einem nicht-standardmäßigen Port.

Falls Sie überschreiben müssen, erstellen Sie `frontend/.env.local` manuell:

```bash
echo 'NEXT_PUBLIC_API_URL=http://localhost:9000' > frontend/.env.local
```

| Variable | Standard | Beschreibung |
|---|---|---|
| `NEXT_PUBLIC_API_URL` | `http://localhost:8000` *(automatisch)* | Backend-URL, die der **Browser** für direkte API-Aufrufe verwendet (OAuth-Umleitungen, Streaming). Wird automatisch von `window.location` erkannt, falls nicht gesetzt — überschreiben Sie diese nur, wenn Ihr Backend lokal auf einem nicht-standardmäßigen Port läuft. |

> **Hinweis zur Build-Zeit**: `NEXT_PUBLIC_*`-Variablen werden zur `pnpm build`-Zeit in das JS-Bundle eingebettet. Das Ändern zur Laufzeit (z. B. über die root `.env`) hat keine Auswirkung — deshalb befinden sie sich nur in `frontend/.env.local` für lokale Entwicklung.## LLM (Erforderlich)

Die Integritätsprüfung berücksichtigt beide Quellen.### Extended Thinking (Reasoning)

When `LLM_REASONING_EFFORT` is set, FIM One enables the model's extended thinking capability so the internal chain-of-thought is surfaced in the UI "thinking" step. FIM One uses [LiteLLM](https://github.com/BerriAI/litellm) to translate the reasoning effort parameter into each provider's native format automatically.#### Unterstützte Anbieter

| Anbieter | `LLM_BASE_URL` | Funktionsweise | Reasoning-Inhalt zurückgegeben? |
|---|---|---|---|
| **OpenAI** (o1 / o3 / o4-mini) | `https://api.openai.com/v1` | `reasoning_effort` nativ gesendet | Ja |
| **Anthropic** (Claude 3.7+) | `https://api.anthropic.com/v1/` | LiteLLM leitet über native Anthropic API mit `thinking` Parameter weiter | Ja |
| **Google Gemini** (2.5+) | `https://generativelanguage.googleapis.com/v1beta/openai/` | `reasoning_effort` auf Kompatibilitäts-Endpunkt gesendet | Ja |

LiteLLM erkennt den Anbieter automatisch aus `LLM_BASE_URL` und ordnet ihn dem korrekten API-Format zu. Unbekannte URLs werden als OpenAI-kompatibel behandelt.#### Wichtige Einschränkungen

<Warning>
**Third-Party-Proxys / Custom Endpoints sind nicht garantiert.**
Wenn Ihr `LLM_BASE_URL` auf einen Third-Party-API-Proxy verweist (z. B. OpenRouter, one-api, Custom Gateway), wird LiteLLM versuchen, basierend auf der URL korrekt weiterzuleiten. Wenn Ihr Proxy jedoch ein nicht standardisiertes Format erwartet, funktioniert das Reasoning möglicherweise nicht wie erwartet. Konsultieren Sie die Dokumentation des Proxys für das erwartete Parameterformat.
</Warning>#### Temperatureinschränkungen bei Reasoning

Einige Provider haben Temperatureinschränkungen, wenn Reasoning aktiv ist:

- **Anthropic**: Erfordert `temperature=1`, wenn Extended Thinking aktiviert ist. Bei Verwendung von Anthropic mit Extended Thinking **müssen** Sie `LLM_TEMPERATURE=1` setzen — Anthropic lehnt andere Werte ab, wenn Thinking aktiviert ist.
- **OpenAI GPT-5.x**: Unterstützt nur `temperature=1` jederzeit. LiteLLMs `drop_params`-Filterung behandelt dies automatisch — nicht unterstützte Temperaturwerte werden stillschweigend gelöscht. Für GPT-5.x ist keine Benutzeraktion erforderlich.#### Wie `LLM_REASONING_BUDGET_TOKENS` funktioniert

Diese Variable ist **hauptsächlich für den Anthropic-Pfad relevant**. Wenn gesetzt, überschreibt sie das automatisch berechnete Budget und wird als `budget_tokens` im `thinking`-Parameter über LiteLLM gesendet. Wenn nicht gesetzt, wird das Budget aus `LLM_MAX_OUTPUT_TOKENS` x Effort-Verhältnis abgeleitet:

| `LLM_REASONING_EFFORT` | Budget-Verhältnis | Beispiel (max_tokens = 64000) |
|---|---|---|
| `low` | 20% | 12.800 Token |
| `medium` | 50% | 32.000 Token |
| `high` | 80% | 51.200 Token |

Das Mindestbudget beträgt 1.024 Token (Anthropics hartes Minimum).

Für OpenAI und Gemini verwaltet der Provider die Token-Zuweisung intern basierend auf der `reasoning_effort`-Stufe — `LLM_REASONING_BUDGET_TOKENS` hat keine Auswirkung.## Agent Execution

---## Analytics (Optional)

Alle Analytics-Provider sind optional. Legen Sie eine beliebige Kombination fest — alle aktiven Provider werden gleichzeitig geladen. Lassen Sie alle leer, um Analytics vollständig zu deaktivieren (empfohlen für lokale Entwicklung).

| Variable | Erforderlich | Standard | Beschreibung |
|---|---|---|---|
| `NEXT_PUBLIC_GA_MEASUREMENT_ID` | Nein | — | Google Analytics 4 Mess-ID (z. B. `G-XXXXXXXXXX`). Erhalten Sie Ihre unter [analytics.google.com](https://analytics.google.com) |
| `NEXT_PUBLIC_UMAMI_SCRIPT_URL` | Nein | — | Umami Analytics-Skript-URL (z. B. `https://your-umami.com/script.js`). Selbst gehostet, datenschutzfreundliche Alternative — [umami.is](https://umami.is) |
| `NEXT_PUBLIC_UMAMI_WEBSITE_ID` | Nein | — | Umami Website-ID. Erforderlich, wenn `NEXT_PUBLIC_UMAMI_SCRIPT_URL` gesetzt ist |
| `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` | Nein | — | Plausible Analytics Domain (z. B. `yourdomain.com`). Leichtgewichtig, datenschutzfreundlich — [plausible.io](https://plausible.io) |
| `NEXT_PUBLIC_PLAUSIBLE_SCRIPT_URL` | Nein | `https://plausible.io/js/script.js` | Benutzerdefinierte Plausible-Skript-URL für selbst gehostete Instanzen |

> Alle `NEXT_PUBLIC_*` Analytics-Variablen sind **Build-Zeit** — Änderungen erfordern einen Frontend-Rebuild, um wirksam zu werden.
