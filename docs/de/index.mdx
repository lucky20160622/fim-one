</Tip>## Was ist FIM One?

FIM One ist ein anbieterunabhängiges Python-Framework zum Erstellen von KI-Agenten, die mit Ihren bestehenden Systemen funktionieren. Im Gegensatz zu Workflow-Buildern, die Sie auffordern, Logik zu duplizieren, verbindet FIM One Ihre Systeme proaktiv – indem es Datenbanken liest, APIs aufruft, Benachrichtigungen sendet – alles über eine einheitliche KI-Schnittstelle.

Der Kerngedanke: **drei Liefermodi, ein Agent-Kern**.## Drei Liefermodi

| Modus | Was es ist | Lieferung | Anwendungsfall |
|------|-----------|----------|----------|
| **Standalone** | Allzweck-KI-Assistent — Suche, Code, Wissensdatenbank | Portal | Chat, Code-Ausführung, Wissensdatenbank-Q&A |
| **Copilot** | KI eingebettet in ein Host-System — funktioniert neben Benutzern in ihrer bestehenden UI | iframe / Widget / Embed | "Finance Copilot" in Ihrer ERP-Web-UI |
| **Hub** | Zentrale systemübergreifende Orchestrierung — alle Ihre Systeme verbunden | Portal / API | Agent fragt ERP ab, prüft OA, benachrichtigt über Lark |## Die Hub-Architektur

Der Hub ist der zentrale Differenzierungsfaktor — ein zentrales Portal, in dem alle Ihre Systeme auf AI treffen:

```mermaid
flowchart LR
  ERP --> Hub
  Database --> Hub
  Lark --> Hub
  Hub --> CRM
  Hub --> OA
  Hub["FIM One Hub<br/>(AI orchestration)"] --> CustomAPI["Custom API"]
```

Jeder Connector ist eine standardisierte Brücke. Der Agent weiß nicht und kümmert sich nicht darum, ob er mit SAP oder einer benutzerdefinierten PostgreSQL-Datenbank spricht. Ihre Daten bleiben in Ihren Systemen; FIM One bietet die AI-Schicht, die sie orchestriert.## Erste Schritte
