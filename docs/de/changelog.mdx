Alle bemerkenswerten Änderungen an diesem Projekt sind hier dokumentiert.

Das Format folgt [Keep a Changelog](https://keepachangelog.com/). Versionen entsprechen [Roadmap](/roadmap) Meilensteinen.## [Unreleased]### Hinzugefügt

- **Admin: Review Log Audit-Tab** — Die Admin-Audit-Seite hat jetzt einen segmentierten Toggle „System Log / Review Log". Das Review Log-Panel fragt den neuen `GET /api/admin/review-log`-Endpunkt ab (Filter: org, resource type, action; limit/offset pagination) und zeigt eine Tabelle mit Zeit, Org-Name, Ressourcentyp, Ressourcenname, Action-Badge (farbcodiert: grün=genehmigt, rot=abgelehnt, bernsteinfarben=eingereicht/erneut eingereicht, gedimmt=unveröffentlicht) und Operator an. Vollständige i18n (EN + ZH).
- **Platform Organization**: Integrierte Org `platform` (ID: `00000000-0000-0000-0000-000000000001`) — alle neuen Benutzer treten automatisch bei. Diese Organisation ersetzt das Konzept der „globalen" Sichtbarkeit und bietet einen zentralen Hub zum Teilen von Ressourcen über alle Benutzer hinweg.
- **Resource Subscriptions**: Benutzer können sich jetzt von gemeinsam genutzten Ressourcen (Agents, Connectors, Knowledge Bases, MCP-Server) aus Org Markets abonnieren/abmelden. Neue `resource_subscriptions`-Datenbanktabelle verfolgt Abonnementbeziehungen.
- **Market API**: Neue Endpunkte zum Durchsuchen und Verwalten von Abonnements:
  - `GET /api/market` — verfügbare Ressourcen in der aktuellen Org durchsuchen
  - `POST /api/market/subscribe` — eine Ressource abonnieren
  - `DELETE /api/market/unsubscribe` — eine Ressource abmelden
  - `GET /api/market/subscriptions` — aktuelle Abonnements des Benutzers auflisten
- **Market Page**: Neue `/market`-Seite im Portal zum Durchsuchen und Verwalten von Abonnements für Org-Ressourcen.
- **Agent Auto-discovery**: Neues `discoverable`-Boolesches Feld auf Agents. Agents mit `discoverable=true` können automatisch erkannt und von LLMs über das `call_agent`-Tool aufgerufen werden (Delegation auf einer Ebene, keine Rekursion).
- **Sub-agent Binding**: Neues `sub_agent_ids`-Feld auf Agents — explizite Whitelist aufrufbarer Sub-Agents für granulare Zugriffskontrolle.
- **CallAgentTool**: Integriertes Tool, das Aufgaben an spezialisierte Agents mit strukturierter Eingabevalidierung und Ergebnisbehandlung delegiert.
- **MCP Server Credentials**: Neue `mcp_server_credentials`-Tabelle ermöglicht Überschreibungen von Umgebungsvariablen und HTTP-Headern pro Benutzer für MCP-Server.
- **MCP Credentials Endpoint**: `PUT /api/mcp-servers/{id}/my-credentials` ermöglicht Benutzern, persönliche Credential-Überschreibungen für gemeinsam genutzte MCP-Server zu speichern.
- **`allow_fallback` auf MCPServer**: Boolesches Feld, das das Fallback-Verhalten für Credentials steuert — wenn `true`, fallen fehlende Benutzer-Credentials auf Standardwerte des Eigentümers zurück.
- **Connector/KB Toggle**: Neue Endpunkte für Ressourcenzustandsverwaltung:
  - `POST /api/connectors/{id}/toggle` — Connector unterbrechen/fortsetzen
  - `POST /api/knowledge-bases/{id}/toggle` — Knowledge Base unterbrechen/fortsetzen
- **Standalone KB Conversations**: Neues `kb_ids`-Feld auf `ChatStreamRequest` und `Conversation` — Benutzer können jetzt direkt mit Knowledge Bases chatten, ohne sie an einen Agent zu binden.
- **Platform Org Protection**: Die Platform Org kann nicht gelöscht werden, und Benutzer können sie nicht selbst verlassen. Stellt sicher, dass alle Benutzer die Konnektivität zur Organisations-Hub beibehalten.
- **Publish Review UI (Frontend)**: Org-Level Publish Review System Frontend — Review-Toggle im Org-Bearbeitungsdialog, ReviewsSheet mit Typ-/Status-Filtern und Genehmigung-/Ablehnungs-Workflow, Publish-Status-Badges (ausstehend/genehmigt/abgelehnt) auf Agent-Karten mit Tooltip für Ablehnungsnotizen, Review-Hinweis im Publish-Dialog, wenn die Ziel-Org eine Überprüfung erfordert, Aktion zum erneuten Einreichen für abgelehnte Ressourcen, Dropdown „Review Management" für Org-Admin/Eigentümer. Alle Ressourcentypen (Agents, Connectors, KBs, MCP-Server) erhalten `publish_status`, `reviewed_by`, `reviewed_at`, `review_note`-Typfelder. Vollständige i18n (EN + ZH).### Behoben

- **Keine erneute Überprüfung beim `is_active`-Umschalten**: Das Umschalten von `is_active` bei Agents, Connectors, Knowledge Bases oder MCP-Servern setzt `publish_status` nicht mehr von `approved` auf `pending_review` zurück. Die `check_edit_revert`-Schutzvorrichtung wird jetzt nur noch ausgelöst, wenn sich tatsächliche Inhaltsfelder ändern (nicht nur `is_active`).### Geändert

- **Sichtbarkeit vereinfacht**: Die Ressourcensichtbarkeit wurde von `personal | org | global` auf `personal | org` reduziert. "Globale" Ressourcen werden nun in der Platform-Organisation veröffentlicht, wobei das rückwärtskompatible Verhalten beibehalten wird und das Sichtbarkeitsmodell vereinfacht wird.
- **Veröffentlichungsschnittstelle vereinheitlicht**: Alle 4 Ressourcentypen (Agent, Connector, Knowledge Base, MCP Server) unterstützen nun nur noch `scope="org"` zum Veröffentlichen. Keine `scope="global"` oder Admin-exklusiven globalen Veröffentlichungsendpunkte mehr.
- **Automatische Platform-Registrierung**: Neue Benutzer werden bei der Registrierung automatisch zur Platform-Organisation hinzugefügt, um sofortigen Zugriff auf gemeinsame Organisationsressourcen zu gewährleisten.
- **Admin-Panel-Bereinigung**: Die Admin-Registerkarten "Global Agents" und "Global MCP" wurden entfernt, da globale Ressourcen nun über die Platform-Organisation verwaltet werden.### Entfernt

- Globale Agent-Management-Endpunkte (`/api/admin/global-agents/*`) entfernt
