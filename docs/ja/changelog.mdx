このプロジェクトへの全ての注目すべき変更はここに記録されています。

形式は [Keep a Changelog](https://keepachangelog.com/) に従っています。バージョンは [ロードマップ](/roadmap) のマイルストーンに対応しています。

## [未リリース]

### 追加

- **モデルごとのネイティブ関数呼び出しトグル** — 新しい `tool_choice_enabled` 設定（ENV グローバル + 管理者によるモデルごと）は、構造化出力抽出のための強制 `tool_choice` の試行を制御します。強制ツール選択を拒否するモデル（例：思考モードモデル）は Level 1 をスキップして JSON Mode に直接進むことができ、構造化呼び出しごとの約 10 秒のペナルティを排除します。Settings → Models → Advanced で設定可能です。
- **構造化出力互換性テーブル** — モデルドキュメント内に表示され、どのプロバイダーがネイティブ関数呼び出し、JSON モード、および計画信頼性評価をサポートしているかを示します。
- **ファイル添付ダウンロード** — チャットメッセージ内のファイル添付カードがクリック可能になり、認証済みフェッチを介して元のファイルをダウンロードできます。ダウンロードアイコンインジケーターとホバー状態が発見性のために追加されました。
- **インテリジェント文書処理（Vision-Aware）ロードマップアイテム** — モデルのビジョン機能に基づいた適応的な文書処理のための計画機能を追加しました（3 つのティア：vision-first、hybrid、text-only）。

### Changed

- **Temperature hint moved to tooltip** — モデルフォームダイアログの温度推論ヒントは、スライダーの下の静的段落ではなく、ラベルの横にコンパクトなツールチップアイコン（HelpCircle）として表示されるようになり、視覚的な混乱が軽減されました。
- **RAG grounding pipeline** — 3段階のパイプライン：検索、引用抽出、信頼度スコアリング。信頼度スコアは、上位3つの証拠ユニットの平均検索スコアです。
- **API Keys banners updated to "Active"** — 管理者およびユーザー設定のAPI Keysページに、緑色の「API Keys Active」バナー（以前の「Coming Soon」通知に代わるもの）が表示され、API認証ドキュメントへのリンクが含まれるようになりました。
- **Multi-worker documentation clarified** — `REDIS_URL`および`WORKERS`環境変数の説明が更新され、停止、注入、PostgreSQL要件、ローカル開発ガイダンスをカバーする警告チェックリストが追加されました。

### 修正

- **ロケール ドキュメント内の重複 OpenAPI エンドポイント** — `docs.json` 内のすべての非英語ロケール API タブから冗長な `Endpoints` OpenAPI グループを削除しました。英語セクションは単一のソースを保持し、ロケール セクションはそれを複製しなくなります。
- **構造化出力の劣化チェーン** — 3 レベルのフォールバック (ネイティブ FC → JSON モード → プレーン テキスト抽出) は、モデルが不正な形式の JSON を生成した場合、すべてのレベルを適切に通過するようになりました。以前は Level 2 で `ValueError` で短絡していました。
- **`json_mode_enabled` DB 値が無視される** — Admin (モデル グループ) 経由で構成されたモデルは、`LLM_JSON_MODE_ENABLED` 環境変数に常にフォールバックするのではなく、モデルごとの `json_mode_enabled` 設定を正しく使用するようになりました。
- **DAG プランニング失敗メッセージ** — プランニング失敗は、生のパイプライン エラーの代わりに、モデル切り替えを提案するユーザーフレンドリーな二言語メッセージを表示するようになりました。
- **DAG ステップ カード レイアウト** — `StepProgressCard` を再設計し、タスク テキストがステップ バッジの下に折り返されるようにしました。単一行で切り詰められるのではなく、ステップ ノード内のツール バッジは、`tools_used` が存在しないが `tool_hint` が存在する場合にのみ表示されるようになり、冗長な表示を回避します。
- **MCP サーバー所有者の allow_fallback バイパス** — サーバー所有者は `allow_fallback=False` 制限によってブロックされなくなりました。以前は、`allow_fallback=False` が設定されていた場合、MCP サーバーの作成者でさえスキップされる可能性がありました。現在は、認証情報を持たない所有者以外のユーザーのみが除外されます。
- **モデル プロバイダー API キー検証** — OpenAI 互換でないプロバイダーを作成する場合、インライン検証エラー、必須アスタリスク インジケーター、および適切な `aria-invalid` アクセシビリティ属性を備えた API キーが必要になります。

## [v0.8] - 2026-03-20

### 追加

- **ReAct合成でのエージェント指令** — `ReActAgent`の新しい`agent_directive`パラメータは、エージェントのコア目的（例：「中国語を英語に翻訳」）を合成システムプロンプトに注入し、最終回答がエージェントのアイデンティティを尊重し、デフォルトの会話応答に陥らないようにします。ReActおよびDAGチャットエンドポイントの両方に配線されています。

- **マーケットプレイス再設計フェーズ1 — ソリューション+コンポーネント二層モデル** — フラットな6タイプのマーケットタブを、構造化された2レベルナビゲーションに置き換えました：トップレベルカテゴリトグル（ソリューション：エージェント/スキル/ワークフロー、コンポーネント：コネクタ/MCPサーバー）とタイプサブタブ。グローバルマーケットとユーザーの組織を切り替えるスコープセレクタドロップダウンを追加しました。マーケットスコープからKBを削除（組織のみ）。組織とマーケット共有を統一：両方が明示的なサブスクリプションを必要とするようになりました（組織の自動表示はなくなりました）。

- **スマートファイルコンテンツインジェクション** — 小さなアップロードファイル（`<32K`文字）は自動的に取得されてLLMコンテキストにインライン化されます。大きなファイルはメタデータサマリーとオンデマンド読み取り用のツールヒントを受け取ります。
- **`read_uploaded_file`エージェントツール** — デュアルモードファイル読み取りツール：シーケンシャルアクセス用のページネーション読み取り（オフセット/リミット）と、ターゲットコンテンツ抽出用のgrepのような検索（正規表現サポート付きクエリ）。
- **`GET /api/files/{file_id}/content`エンドポイント** — アップロードされたファイルから抽出された完全なテキストにアクセスするためのページネーション付きテキストコンテンツAPI。
- **`.content`サイドカーファイル** — 完全に抽出されたテキストがアップロードと一緒にサイドカーファイルとして保存され、再抽出なしで永続的なコンテンツアクセスが可能になります。
- **ファイルAPIレスポンスの`content_length`フィールド** — ファイルアップロードおよびリストエンドポイントに、テキストコンテンツサイズの可視性のための`content_length`が含まれるようになりました。
- **`fileApi.getContent()`フロントエンドクライアントメソッド** — フロントエンドからページネーション付きファイルコンテンツを取得するための新しいAPIクライアントメソッド。

- **ワークフローエンジンノード再試行** — 指数バックオフを伴うノード単位の再試行。ノードはデータ設定で`retry_count`と`retry_delay_ms`を指定できます。失敗したノードは最大N回まで遅延付きで再試行されます。試行番号と前のエラーを含む`node_retrying` SSEイベントを発行します。再試行待機中のキャンセルイベントを尊重します。
- **ワークフロー変数ストア式評価** — simpleeval経由の安全なPython式評価。`evaluate_expression()`メソッドはストア変数（envを除く）と一般的な組み込み関数（len、str、int、float、bool、abs、min、max、sum、round、sorted）を公開します。さらに`has()`、`delete()`、`keys()`ヘルパーメソッドがあります。
- **ワークフローフェーズ2ノード：HumanIntervention** — 承認待機ノード（自動承認スタブ）。prompt_message（{{variable}}補間付き）、assignee、timeout_hoursを読み取ります。Sky-500テーマのフロントエンドノード（プロンプトプレビューと@assigneeバッジ付き）。プロンプトテキストエリア、assignee入力、タイムアウトフィールド付きの設定パネル。
- **ワークフローテンプレート12に拡張** — 4つの新しいテンプレート：人間承認パイプライン（LLM → HumanIntervention → ConditionBranch → デュアルEnd）、マルチステップデータ処理（CodeExecution → ConditionBranch → Transform → ListOperation）、知識強化エージェント（QuestionUnderstanding → KnowledgeRetrieval → LLM）、反復レビューループ（Loop → LLM → VariableAssign）。
- **ワークフローペーンコンテキストメニュー** — 空のキャンバスを右クリックして、貼り付け（クリップボードが空の場合は無効）、すべて選択、ビューに合わせる、自動レイアウトアクションにアクセスします。判別共用体状態がノードとペーンコンテキストメニューを分離します。
- **ワークフローノードコンテキストメニュー** — 任意のノードを右クリックして、設定、コピー、複製、削除アクションにアクセスします。削除アクションはStart/Endシングルトンノードでは非表示です。メニューは外側のクリックまたはスクロール時に自動的に閉じます。
- **ワークフローフェーズ2テンプレート** — 2つの新しい組み込みテンプレート（合計8個）：リスト処理パイプライン（Start → ListOperation → Transform → End）と強化された質問応答（Start → QuestionUnderstanding → LLM → End）。
- **ワークフローフェーズ2検証** — ListOperation（入力/式が不足）、Transform（入力/操作が不足）、Iterator（リスト変数が不足）、Loop（条件が不足）、DocumentExtractor、QuestionUnderstanding（入力が不足）、ParameterExtractor（入力テキスト/パラメータが不足）の拡張ブループリント検証警告。修正：ソート操作は式を不正に要求しなくなりました。
- **ワークフローノートノート** — ノード単位の注釈機能。任意のノードの設定パネルで「Note」をクリックして、キャンバス上のノードの下にイタリック体で表示される自由形式のノートを追加します。自動拡張テキストエリア（`data.note`ストレージ）。
- **ワークフロースナップツーグリッド** — ドット背景パターンに合わせた16pxスナップグリッド（正確なノード配置用）。
- **ワークフロー接続検証** — 既存のStart/End制限に加えて、自己接続と重複エッジを防止します。
- **ワークフロー拡大縮小対応ドロップ** — ノードパレットドラッグアンドドロップは、任意のズームレベルで正しい配置のために`screenToFlowPosition()`を使用します。
- **ワークフロー SSE複数行データ** — SSEパーサーは複数行の`data:`フィールドを処理し、キープアライブコメント行を無視するようになりました。
- **ワークフローキーボードショートカット** — コピー（Cmd+C）、貼り付け（Cmd+V）、複製（Cmd+D）、すべて選択（Cmd+A）、Escapeで選択解除。コピー/貼り付けは新しいIDを生成し、位置をオフセットします。Start/Endノードはシングルトン制約を維持するためにコピーから除外されます。
- **ワークフローインライン説明編集** — ツールバーの説明行をクリックして、ワークフローから移動することなくワークフロー説明を編集するためのポップオーバーテキストエリアを開きます。
- **ワークフロー ConditionBranch LLMモード** — ナイーブな「最初にマッチ」スタブに代わる適切なLLMベースの条件評価。条件ラベルから分類プロンプトを構築し、高速LLMを呼び出し、大文字小文字を区別しないファジーマッチングを使用します。
- **リアクティブエッジラベル** — 条件/分類器ブランチのエッジラベルは、設定パネルで条件またはクラスが編集されるときにリアルタイム更新のために`useNodesData()`を使用するようになりました。
- **ワークフロー統計パネル** — シートベースの統計表示（総実行数、完了/失敗/キャンセル数、ビジュアルプログレスバー付き成功率、平均期間、相対形式付き最終実行時刻）。ツールバーのBarChart3アイコンからトリガーされます。
- **フェーズ2ワークフローノード：DocumentExtractor & QuestionUnderstanding** — さらに2つのノードタイプ：
  - **DocumentExtractor**：ドキュメントテキスト処理用のスタブエグゼキューター — テキスト/base64入力タイプと4つの抽出モード（full_text、page_range付きページ、word/char/line/pageカウント付きメタデータ、マークダウンテーブル検出付きテーブル）をサポート。URL入力は後で実装予定。
  - **QuestionUnderstanding**：4つのモード付きLLMベースの質問前処理 — 書き直し（明確性）、展開（コンテキスト+サブ質問）、分類（インテント/トピック/信頼度JSON）、分解（サブ質問配列）。オプションのカスタムシステムプロンプトオーバーライド付きの高速LLMを使用します。
  - 完全なフロントエンド：ノードコンポーネント、設定パネル、パレットエントリ（QU用AI カテゴリ、DE用データカテゴリ）、変数ピッカーサポート。
- **フェーズ2ワークフローノード：ListOperation & Transform** — 2つのデータ処理ノードタイプ：
  - **ListOperation**：8つのリスト操作（filter、map、sort、slice、flatten、unique、reverse、length）（filter/map/sort用simpleeval式付き）。JSON文字列入力自動解析をサポート。
  - **Transform**：6つの操作タイプ（json_path、type_cast、format、regex_extract、string_op、math_op）を備えたパイプラインベースのデータ変換。操作は順序付きで適用されます — 1つの出力が次の入力になります。
  - 完全なフロントエンド：ノードコンポーネント、操作固有のエディター付き設定パネル、パレットエントリ、変数ピッカーサポート。
- **フェーズ2ワークフローノード：Loop & ParameterExtractor** — さらに2つの高度なノードタイプ：
  - **Loop**：simpleeval式評価、設定可能なループ変数名、max_iterations安全制限を備えたwhile条件ループ。条件入力、変数名、反復制限設定付きフロントエンド。
  - **ParameterExtractor**：テキストからのLLMベースの構造化パラメータ抽出。型付きパラメータ（名前/タイプ/説明/必須）を定義し、入力テキスト参照を提供し、オプションで抽出指示を追加します。解析されたJSONを返します。パラメータリストエディターと抽出プロンプトテキストエリア付きフロントエンド。
- **フェーズ2ワークフローノード：Iterator & VariableAggregator** — 2つの新しい高度なノードタイプ：
  - **Iterator**：設定可能なアイテム/インデックス変数名とmax_iterations安全制限を備えたリストの反復。ネイティブリスト、JSON文字列解析、テンプレート変数参照をサポート。
  - **VariableAggregator**：4つのモード経由で複数の上流ブランチからの出力をマージ — リストに収集、文字列を連結（カスタム区切り文字）、辞書をマージ、または最初の空でない値を取得。
  - 完全なフロントエンド統合：ノードコンポーネント、パレットエントリ、設定パネルフォーム、ミニマップ色、変数ピッカーサポート。バックエンドエグゼキューターはエンジンに登録されます。
- **ワークフローテンプレート拡張** — 2つの新しい組み込みテンプレート（合計6個）を追加しました：データ処理パイプライン（Start → CodeExecution → TemplateTransform → End）とナレッジ検索付きエージェント（Start → KnowledgeRetrieval → Agent → End）。新しいテンプレートの欠落している`icon`フィールドを修正しました。「intermediate」および「advanced」テンプレートカテゴリの欠落しているi18nキーを追加しました。
- **ワークフロー環境変数ダイアログ** — ワークフロー単位の暗号化環境変数管理UI。キー値エディター（マスク表示）、重複キー

### 変更

- **サブスクリプションベースの可視性モデル** — `build_visibility_filter()` が3段階（自分 → 組織メンバー → サブスクライバー）から2段階（自分 → サブスクライバー）に簡略化されました。データマイグレーションは既存の全組織メンバーに対して `ResourceSubscription` 行を自動作成し、アクセスを保持します。ダッシュボードのコネクタ数とKB統計にはサブスクライブされたリソースが含まれるようになりました。
- **Market API スコープとカテゴリパラメータ** — `browse_market()` は `scope`（market または org:{id}）と `category`（solutions または components）クエリパラメータを受け入れるようになりました。`subscribe_resource()` は非マーケット組織のサブスクリプションに対して組織メンバーシップを検証するようになりました。

- **ツールキャッシュホワイトリストリファクタリング** — `ToolCache` のハードコードされたブラックリストをホワイトリストアプローチに置き換えました。`Tool` プロトコルに `cacheable` プロパティを追加（デフォルト `False`）；`cacheable=True` と明示的にマークされたツールのみがキャッシュされます。11個の読み取り専用ツール（web_search、kb_retrieve、calculator など）がキャッシュ可能としてマークされ、すべての副作用ツール（コード実行、メール、コネクタ）はデフォルトでキャッシュされません。新しいコネクタツールは追加の設定なしで安全です。
- **DAG エグゼキューター カスケード失敗** — 失敗またはタイムアウトしたステップは依存するステップをカスケードブロックするようになりました。`failed_ids` トラッキングが推移的伝播（A→B→C チェーン）で追加されました。依存ステップは自動的に説明的なメッセージとともに失敗としてマークされ、汚染されたコンテキストで実行されることはありません。
- **DAG ステップ検証ステータス修正** — ステップ検証の再試行失敗は、`"completed"` のままにするのではなく、ステップを `"failed"` として正しくマークするようになりました。失敗した検証は新しいカスケード失敗メカニズムを介してダウンストリームステップにカスケードします。
- **DAG プランナー ツール説明** — プランナーは名前だけでなく、各ツールに対して `{name, description}` を受け取るようになり、より正確なステップ対ツールマッピングが可能になり、幻覚的な `tool_hint` 値が減少します。
- **DAG 再プランナー 完全履歴** — 再計画コンテキストは最後のものだけでなく、すべての前のラウンドからの結果を蓄積するようになりました。古いラウンドはより積極的に切り詰められ（200文字/ステップ）、最新のラウンドは500文字/ステップを保持します。
- **stream_answer 合成品質** — 最終回答を合成する際のツール観察切り詰めが2000から8000文字に増加され、より多くの構造化データ（JSONテーブル、コード出力）が保持されます。
- **14個のエンジン定数をパラメータ化** — ReAct エージェント、DAG プランナー、コンテキストガード、ワークスペース、およびシステムモジュール全体でハードコードされた数値定数を環境変数に抽出しました。すべての新しい環境変数は `example.env` と `docs/configuration/environment-variables.mdx` に記載されています。

- **エビデンス信頼度 UI 拡張** — グラウンディングパイプラインのフロントエンド表示に3つの改善：（1）不十分なエビデンス観察は生のテキストではなく i18n 琥珀色警告カードとしてレンダリングされるようになりました、（2）インライン `[N]` 引用バッジはホバーポップオーバーでソース名、引用抜粋、関連性スコアを表示するようになりました（新しい EvidenceContext 経由）、（3）エビデンスソースが矛盾する場合、参照セクションに競合警告バナーが表示され、展開可能な並べて比較ができます。
