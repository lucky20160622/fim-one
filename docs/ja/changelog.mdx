このプロジェクトへの全ての注目すべき変更はここに記録されています。

形式は [Keep a Changelog](https://keepachangelog.com/) に従っています。バージョンは [ロードマップ](/roadmap) のマイルストーンに対応しています。

## [未リリース]

### 追加

- **ワークフロー レベルのタイムアウト** — エンジンの新しい `workflow_timeout_ms` パラメータが、超過時にすべての保留中/実行中のノードをキャンセルし、レスポンシブなタイムアウト検出のための制限付き非同期待機を行います。
- **最後の入力で再実行** — 「再度実行」ボタンが、前回の実行の値で入力フォームを事前入力します。
- **実行履歴ステータス フィルタ** — ワークフロー実行を完了/失敗/キャンセル/実行中でフィルタするドロップダウン。
- **ワークフロー バージョン履歴** — ブループリント保存時の自動バージョニング、および以前のバージョンに戻すためのリスト/取得/復元 API エンドポイント。
- **検証エンドポイント** — 実行なしで構造的なブループリント検証を行う `POST /validate`。
- **ドライラン モード** — `dry_run=true` を指定した `POST /run` が、実行せずに実行計画を JSON として返します。
- **13 の条件分岐ルーティング テスト** — 分岐、マージ、ネスト、および分類器ルーティングの包括的なカバレッジ。
- **ノード単位の実行統計パネル** — ノード単位の成功率、平均期間、失敗数を最悪順で表示する新しい統計パネル (`node-stats-panel.tsx`)。ワークフロー ツールバーに統合されています。フロントエンド使用のための新しい `getNodeStats` API クライアント メソッドと `NodeStatEntry` 型。
- **フェーズ 3 ノード: SubWorkflow と ENV** — バックエンド エグゼキューター、パーサー検証、および 14 の新しいテスト (合計: 304 → 306) を備えた 2 つの新しいノード タイプ:
  - **SubWorkflow**: ネストされたワークフロー オーケストレーション用のスタブ エグゼキューター — ターゲット ワークフローを選択し、入出力変数をマップします。
  - **ENV**: ランタイムに暗号化された環境変数を読み取り、キー選択とフォールバック デフォルトを使用します。
- **SubWorkflow と ENV フロントエンド コンポーネント** — 完全なフロントエンド統合: `sub-workflow-node.tsx` と `env-node.tsx` (設定フォーム、パレット エントリ、ミニマップ色、すべてのタイプ マップ全体のノード タイプ レコード エントリを含む)。
- **2 つの新しいワークフロー テンプレート (合計 14)** — 「セキュア API 統合」(ENV → HTTP 認証ヘッダー付き) と「サブワークフロー オーケストレーション」(LLM → SubWorkflow → 要約)。
- **キーボード ショートカット ダイアログ** — `?` を押して、ダイアログ オーバーレイで利用可能なすべてのキーボード ショートカットを表示します。
- **ノード入力プレビュー キャプチャ** — 実行イベントに、デバッグ用の先行ノード出力の切り詰められた JSON が含まれるようになりました。
- **ワークフロー分析エンドポイント** — `GET /analytics` は、総実行数、ステータス分布、成功率、p50/p95/p99 期間パーセンタイル、日次トレンド、および最も失敗したノードを返します。
- **分析ダッシュボード パネル** — 統計カード、ステータス分布バー、純 CSS 日次チャート、パーセンタイル バー、および 7d/14d/30d 範囲セレクター付きの失敗ノード テーブル。
- **テンプレート ギャラリー ダイアログ** — カテゴリ フィルター付きのワークフロー テンプレートの視覚的なグリッド ブラウジング。
- **ラベル付きエッジ** — 条件および分類器ブランチ名がエッジ上にピルとして表示されます。
- **ワークフロー変数パネル** — ワークフロー レベルの変数を型/デフォルト/説明で定義、編集、削除します。

### Changed

- **elkjs を軽量 BFS 自動レイアウトに置き換え** — elkjs（430 kB）をカスタム BFS アルゴリズムに置き換え。`/workflows/[id]` バンドルが 473 kB から 43 kB に削減。
- **ワークフロー一覧のソートオプションを拡張** — 最新、最古、名前 A-Z、名前 Z-A、最近更新。
- **ワークフロー テストスイートを 306 テストに拡張** — SubWorkflow と ENV ノード実行器用に Phase 3 テスト 14 個を追加。ノードごとの実行統計テストを追加。合計: 275 → 306。
- **ワークフロー テストスイートを 275 テストに拡張** — 高度なエンジンテスト 23 個（指数バックオフ付き再試行、再試行中のキャンセル、式評価、変数ストア ヘルパー）を別のテストファイルに追加。合計: 252 → 275。
- **ワークフロー テストスイートを 252 テストに拡張** — HumanIntervention テスト 8 個（自動承認、補間、割り当て先/タイムアウト、デフォルト、検証警告）と新しいテンプレートテスト 4 個（人間による承認パイプライン、マルチステップデータ処理、知識強化エージェント、反復的なレビューループ）を追加。合計: 240 → 252。
- **ワークフロー テストスイートを 240 テストに拡張** — Phase 2 検証テスト 15 個（ListOperation、Transform、Iterator、Loop、DocumentExtractor、QuestionUnderstanding、ParameterExtractor 警告用）と Phase 2 テンプレートテスト 5 個を追加。合計: 220 → 240。
- **ワークフロー テストスイートを 220 テストに拡張** — DocumentExtractor 用テスト 20 個（full_text/pages/metadata/tables モード、base64 デコード、page_range 解析、URL スタブエラー、空入力、ダッシュ区切り文字用 12 テスト）と QuestionUnderstanding 用テスト 8 個（LLM モッキング付き rewrite/classify/decompose/expand モード、カスタムシステムプロンプト、変数補間、入力欠落）を追加。合計: 200 → 220。
- **ワークフロー テストスイートを 200 テストに拡張** — ListOperation 用テスト 50 個（filter、map、sort、slice、flatten、unique、reverse、length、JSON 文字列入力、無効な型、欠落した設定、index-in-scope、デフォルト出力変数用 21 テスト）と Transform 用テスト 29 個（json_path、すべての型の type_cast、フォーマットテンプレート、regex_extract、すべてのバリアントの string_op、すべてのバリアントの math_op、5 ステップパイプラインチェーン、ゼロ除算、ワイルドカードパス、未知の操作）を追加。合計: 150 → 200。
- **ワークフロー テストスイートを 150 テストに拡張** — Loop 用テスト 12 個（基本カウント、max_iterations 安全性、ゼロ反復、空条件、カスタム変数名、変数補間用 6 テスト）と ParameterExtractor 用テスト 6 個（基本抽出、入力欠落、空パラメータ、無効な JSON、変数補間、抽出プロンプト含有）を追加。合計: 138 → 150。
- **ワークフロー テストスイートを 138 テストに拡張** — Phase 2 Iterator 用テスト 20 個（基本リスト、JSON 解析、max_iterations、空リスト、欠落した設定、None-to-empty、テンプレート参照、非リスト拒否、メタデータ用 9 テスト）と VariableAggregator 用テスト 11 個（list/concat/merge/first_non_empty モード、空入力のエッジケース、None スキップ、未知モード）を追加。合計: 118 → 138。
- **ワークフロー テストスイートを 118 テストに拡張** — ConditionBranch LLM モード用テスト 7 個（完全一致、大文字小文字を区別しない、ファジー、不一致フォールバック、DEFAULT レスポンス、候補なし、変数補間）を追加。合計: 111 → 118。
- **ワークフロー実行モード ビジュアルフィードバック大幅改善** — ボーダーベースのアプローチに代わるリングベースのノードステータススタイリング、実行中ノード用のピング効果付きアニメーション ステータスドット、クリーンな状態分離のための `useMemo` 派生の `displayNodes`/`displayEdges`、完了から実行中へのノード間のエッジアニメーション。
- **ワークフロー テストスイートを 105 テストに拡張** — テンプレート検証テスト、インポート/エクスポート ラウンドトリップ、変数ストア高度なメソッド、デッドロック検出、マルチ条件分岐、質問分類器、包括的なブループリント検証を追加。合計: 68 → 97 → 105。
- **ワークフロー テストスイートを 68 テストに拡張** — eval 名前空間フラット化（ドット記法キー解決、短いエイリアス生成、フラット名を使用した ConditionBranch/VariableAssign 用 17 テスト）とブループリント検証警告（切断されたノード、到達不可能なエンド、空の条件/プロンプト/コード用 7 テスト）をカバーする 24 個の新しいテストを追加。合計ワークフロー テスト数: 34 → 68。
- **ワークフロー エディター大幅改善（Dify スタイル）** — Dify にインスパイアされたワークフロー エディターの完全なビジュアル再設計:
  - 上から下へのレイアウトに代わる水平（左から右）レイアウト。すべての 12 ノードタイプが Left/Right ハンドルを使用
  - 左カラーバー ノード設計、コンパクト 220px 幅、小さいアイコン、実行ステータスバッジ
  - エッジ上にノードを追加: "+" 中点ボタンと既存エッジを分割するノードタイプピッカーポップアップ付きカスタム `AddNodeEdge`
  - より豊富な設定パネル: LLM ノード用モデルセレクタードロップダウン、条件用オペレータードロップダウン（==、!=、>、<、contains など）、セクションヘッダー、すべての出力ノード用 `InsertVariableHint` と `OutputVariableField`
  - 折りたたみ可能なノードパレット（アイコンのみモード）、カテゴリグループ化。既に配置されている Start/End ノードは淡色表示
  - ドロップ時の単一 Start/End 検証。`maxZoom:1` 付き `fitViewOptions` で小さいグラフでの過度なズームを防止

### 追加

- **ワークフロー エンジン ノード リトライ** — ノード単位のリトライと指数バックオフ。ノードはデータ設定で `retry_count` と `retry_delay_ms` を指定できます。失敗したノードは最大 N 回リトライされ、遅延は 60 秒に制限されます。試行番号と前のエラーを含む `node_retrying` SSE イベントを発行します。リトライ待機中のキャンセル イベントを尊重します。
- **ワークフロー変数ストア式評価** — simpleeval を使用した安全な Python 式評価。`evaluate_expression()` メソッドはストア変数（env を除く）と一般的なビルトイン（len、str、int、float、bool、abs、min、max、sum、round、sorted）を公開します。さらに `has()`、`delete()`、`keys()` ヘルパー メソッドがあります。
- **ワークフロー フェーズ 2 ノード: HumanIntervention** — 承認待機ノード（自動承認スタブ）。prompt_message（{{variable}} 補間付き）、assignee、timeout_hours を読み取ります。Sky-500 テーマのフロントエンド ノード（プロンプト プレビューと @assignee バッジ付き）。プロンプト テキストエリア、assignee 入力、タイムアウト フィールド付きの設定パネル。
- **ワークフロー テンプレートを 12 に拡張** — 4 つの新しいテンプレート: Human Approval Pipeline（LLM → HumanIntervention → ConditionBranch → dual End）、Multi-Step Data Processing（CodeExecution → ConditionBranch → Transform → ListOperation）、Knowledge-Enhanced Agent（QuestionUnderstanding → KnowledgeRetrieval → LLM）、Iterative Review Loop（Loop → LLM → VariableAssign）。
- **ワークフロー ペイン コンテキスト メニュー** — 空のキャンバスを右クリックして、Paste（クリップボードが空の場合は無効）、Select All、Fit View、Auto Layout アクションにアクセスします。判別された共用体の状態がノード コンテキスト メニューとペイン コンテキスト メニューを分離します。
- **ワークフロー ノード コンテキスト メニュー** — 任意のノードを右クリックして、Configure、Copy、Duplicate、Delete アクションにアクセスします。Delete アクションは Start/End シングルトン ノードでは非表示です。メニューは外側のクリックまたはスクロール時に自動的に閉じます。
- **ワークフロー フェーズ 2 テンプレート** — 2 つの新しい組み込みテンプレート（合計 8 個）: List Processing Pipeline（Start → ListOperation → Transform → End）と Enhanced Question Answering（Start → QuestionUnderstanding → LLM → End）。
- **ワークフロー フェーズ 2 検証** — ListOperation（入力/式がない）、Transform（入力がない/操作が空）、Iterator（リスト変数がない）、Loop（条件がない）、DocumentExtractor、QuestionUnderstanding（入力がない）、ParameterExtractor（入力テキスト/パラメータが空）に対する拡張ブループリント検証警告。修正: ソート操作は式を必須としなくなりました。
- **ワークフロー ノート** — ノード単位の注釈機能。任意のノードの設定パネルで「Note」をクリックして、フリーフォーム ノートを追加します。ノートはキャンバス上のノードの下にイタリック テキストとして表示されます。自動拡張テキストエリア（`data.note` ストレージ）。
- **ワークフロー グリッドにスナップ** — ドット背景パターンに一致する 16px スナップ グリッド（ノードの正確な配置用）。
- **ワークフロー 接続検証** — 既存の Start/End 制限に加えて、自己接続と重複エッジを防止します。
- **ワークフロー ズーム対応ドロップ** — ノード パレット ドラッグ アンド ドロップは `screenToFlowPosition()` を使用して、任意のズーム レベルで正しい配置を実現します。
- **ワークフロー SSE マルチラインデータ** — SSE パーサーはマルチラインの `data:` フィールドを処理し、キープアライブ コメント行を無視します。
- **ワークフロー キーボード ショートカット** — Copy（Cmd+C）、Paste（Cmd+V）、Duplicate（Cmd+D）、Select All（Cmd+A）、Escape でデセレクト。Copy/Paste は新しい ID を生成し、位置をオフセットします。Start/End ノードはシングルトン制約を維持するためにコピーから除外されます。
- **ワークフロー インライン説明編集** — ツールバーの説明行をクリックして、ポップオーバー テキストエリアを開き、ワークフロー説明を編集します（ページ移動なし）。
- **ワークフロー ConditionBranch LLM モード** — ナイーブな「最初にマッチ」スタブを置き換える適切な LLM ベースの条件評価。条件ラベルから分類プロンプトを構築し、高速 LLM を呼び出し、大文字小文字を区別しないファジー マッチングを使用します。
- **リアクティブ エッジ ラベル** — 条件/分類器ブランチのエッジ ラベルは `useNodesData()` を使用して、設定パネルで条件またはクラスが編集されたときにリアルタイム更新されます。
- **ワークフロー 統計パネル** — シート ベースの統計表示（総実行数、完了/失敗/キャンセル数、ビジュアル プログレス バー付きの成功率、平均期間、相対形式の最後の実行時刻）。ツールバーの BarChart3 アイコンからトリガーされます。
- **フェーズ 2 ワークフロー ノード: DocumentExtractor & QuestionUnderstanding** — 2 つのノード タイプ:
  - **DocumentExtractor**: ドキュメント テキスト処理用のスタブ エグゼキューター — テキスト/base64 入力タイプと 4 つの抽出モード（full_text、page_range 付きページ、単語/文字/行/ページ数付きメタデータ、マークダウン テーブル検出付きテーブル）をサポートします。URL 入力は後で実装予定です。
  - **QuestionUnderstanding**: 4 つのモード付き LLM ベースの質問前処理 — rewrite（明確性）、expand（コンテキスト + サブ質問）、classify（intent/topic/confidence JSON）、decompose（サブ質問配列）。オプションのカスタム システム プロンプト オーバーライド付きの高速 LLM を使用します。
  - 完全なフロントエンド: ノード コンポーネント、設定パネル、パレット エントリ（QU 用 AI カテゴリ、DE 用 Data カテゴリ）、変数ピッカー サポート。
- **フェーズ 2 ワークフロー ノード: ListOperation & Transform** — 2 つのデータ処理ノード タイプ:
  - **ListOperation**: 8 つのリスト操作（filter、map、sort、slice、flatten、unique、reverse、length）と filter/map/sort 用の simpleeval 式。JSON 文字列入力の自動解析をサポートします。
  - **Transform**: 6 つの操作タイプ（json_path、type_cast、format、regex_extract、string_op、math_op）を使用したパイプライン ベースのデータ変換。操作は順序に適用されます — 1 つの出力が次の入力になります。
  - 完全なフロントエンド: ノード コンポーネント、操作固有のエディター付き設定パネル、パレット エントリ、変数ピッカー サポート。
- **フェーズ 2 ワークフロー ノード: Loop & ParameterExtractor** — 2 つのより高度なノード タイプ:
  - **Loop**: simpleeval 式評価、設定可能なループ変数名、最大反復回数安全制限付きの While 条件ループ。条件入力、変数名、反復制限設定付きのフロントエンド。
  - **ParameterExtractor**: テキストから型付きパラメータを抽出する LLM ベースの構造化パラメータ抽出。型付きパラメータ（名前/型/説明/必須）を定義し、入力テキスト参照を提供し、オプションで抽出指示を追加します。解析済み JSON を返します。パラメータ リスト エディターと抽出プロンプト テキストエリア付きのフロントエンド。
- **フェーズ 2 ワークフロー ノード: Iterator & VariableAggregator** — 2 つの新しい高度なノード タイプ:
  - **Iterator**: 設定可能なアイテム/インデックス変数名と最大反復回数安全制限付きでリストを反復処理します。ネイティブ リスト、JSON 文字列解析、テンプレート変数参照をサポートします。
  - **VariableAggregator**: 4 つのモード経由で複数のアップストリーム ブランチからの出力をマージ — リストに収集、文字列を連結（カスタム区切り文字）、辞書をマージ、最初の空でない値を取得。
  - 完全なフロントエンド統合: ノード コンポーネント、パレット エントリ、設定パネル フォーム、ミニマップ色、変数ピッカー サポート。バックエンド エグゼキューターはエンジンに登録されます。
- **ワークフロー テンプレート拡張** — 2 つの新しい組み込みテンプレートを追加（合計 6 個）: Data Processing Pipeline（Start → CodeExecution → TemplateTransform → End）と Agent with Knowledge Retrieval（Start → KnowledgeRetrieval → Agent → End）。新しいテンプレートの欠落している `icon` フィールドを修正。「intermediate」および「advanced」テンプレート カテゴリの欠落している i18n キーを追加。
- **ワークフロー ENV 変数ダイアログ** — ワークフロー単位の暗号化環境変数管理 UI。マスク表示、重複キー検出、インライン検証付きのキー値エディター。`GET/PUT /api/workflows/{id}/env` エンドポイントに接続。ツールバー ドロップダウンからアクセス可能。
- **ワークフロー 自動保存** — 5 秒のデバウンス自動保存タイマー（即座の Cmd+S オーバーライド付き）。保存ステータス インジケーター「Saving...」、「Unsaved changes」、または「Saved Xs ago」を表示（30 秒自動更新）。
- **ワークフロー リスト検索とフィルター** — 名前/説明によるクライアント側検索、ステータス フィルター ボタン（All/Draft/Active）、ソート ドロップダウン（Recently updated/created/Name A-Z）、結果なしの空状態。
- **ワークフロー カラーコード ミニマップ** — ミニマップ ノード色は各ノード タイプのカテゴリ色に一致（緑=Start、赤=End、青=LLM、オレンジ=Condition など）。

### 修正

- **ワークフロー eval 名前空間のフラット化**: ConditionBranch と VariableAssign が短い変数名（例：ストアに `input.score` がある場合の `score`）の解決に失敗する問題を修正しました。`_flatten_eval_names()` ヘルパーを追加して、eval 名前空間に短いエイリアスを作成し、フルパス参照なしでドット記法キーの解決を可能にしました。

- **ワークフロー SSE イベント名の不一致**: SSE イベント名を修正しました — `node_status` を `node_started`/`node_completed`/`node_failed`/`node_skipped` に名前変更して、フロントエンド SSE ハンドラーの期待値と一致させました。

- **`is_active` トグルで再レビューなし**: エージェント、コネクタ、ナレッジベース、または MCP サーバーの `is_active` をトグルしても、`publish_status` が `approved` から `pending_review` に戻ることはなくなりました。`check_edit_revert` ガードは、実際のコンテンツフィールドが変更された場合にのみ発火するようになりました（`is_active` だけではなく）。
- **条件分岐のカスケードスキップ**: スキップされた条件/分類器ノードは、発信エッジを正しく非アクティブ化し、ダウンストリームノードが予期せず実行されるのを防ぐようになりました。

- **SSE: 構造化エラーハンドリング**: `useSSE` フック内の `onError` コールバックはプレーン文字列から `ApiError` オブジェクトにアップグレードされ、より豊富なエラーコンテキスト (エラーコード、引数) を提供
- **UI: `formatTokens` ユーティリティ**: トークンカウントをロケール対応の数値フォーマットで表示するための新しいフォーマットヘルパー
- **UI: `Checkbox` および `FadeIn` コンポーネント**: 新しい shadcn スタイルのチェックボックスおよびフェードインアニメーションラッパーコンポーネント

### 変更内容
- **ドキュメント: ASCII → Mermaid ダイアグラム**: すべてのボックス描画 ASCII ダイアグラムがネイティブ Mermaid チャートに変換されました（6 ページにわたる 12 個のダイアグラム）。レンダリングとインタラクティビティが向上しました
- **ドキュメント: ダークモード テーマ**: Mintlify カラー設定を改善 — ダークモードでの暖かいアンバーアクセント、グラデーション背景装飾、プロジェクトテーマに合わせた適切な背景色

### 修正
- **エクスポート: RFC 5987 ファイル名サポート**: Content-Disposition ヘッダーでプレーン `filename` より UTF-8 エンコードされた `filename*` を優先し、会話をエクスポートする際の CJK ファイル名の文字化けを修正
- **Docker i18n 破損**: `messages/` ディレクトリをスタンドアロン出力にコピーし、`outputFileTracingIncludes` を追加して、next-intl がランタイムにロケール JSON ファイルを検出できるようにした
- **Docker スタートアップ競合状態**: Next.js 起動前にエントリーポイントで API 準備完了待機ループ (Python ソケットポーリング) を追加し、`e.map is not a function` SSR エラーを防止
- **アーティファクト: 会話の created_at でソート**: ファイル mtime ではなく会話作成時刻でアーティファクトをソート (mtime は Docker ボリュームで信頼性が低い)
- **OAuth: window.location ポートから本番 API URL を自動検出**: ブラウザ自動検出が本番環境のカスタムポート (3000 だけでなく) で正しく機能するようになった
- **Docker: DooD サンドボックスボリュームマウントパスの不一致**: `CODE_EXEC_BACKEND=docker` (Docker-outside-of-Docker) でコンテナ内で実行する場合、サンドボックススクリプトはホスト Docker デーモンに見えないコンテナ内部パスに書き込まれ、空の `-v` マウントと「No such file or directory」エラーが発生していた。サンドボックスディレクトリを `data/sandbox/` に移動し、`_resolve_host_path()` によるパス変換を行う `DOCKER_HOST_DATA_DIR` 環境変数を追加し、空文字列環境変数のエッジケースを処理することで修正
