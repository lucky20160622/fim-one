</Tip>## FIM One란 무엇인가요?

FIM One은 기존 시스템과 함께 작동하는 AI 에이전트를 구축하기 위한 공급자 중립적 Python 프레임워크입니다. 로직을 복제하도록 요청하는 워크플로우 빌더와 달리, FIM One은 통합 AI 인터페이스를 통해 데이터베이스를 읽고, API를 호출하고, 알림을 푸시하는 등 시스템을 사전에 연결합니다.

핵심 통찰: **세 가지 전달 모드, 하나의 에이전트 코어**.## 세 가지 배포 모드

| 모드 | 설명 | 배포 | 사용 사례 |
|------|-----------|----------|----------|
| **Standalone** | 범용 AI 어시스턴트 — 검색, 코드, 지식 기반 | Portal | 채팅, 코드 실행, 지식 기반 Q&A |
| **Copilot** | 호스트 시스템에 내장된 AI — 기존 UI에서 사용자와 함께 작동 | iframe / widget / embed | ERP 웹 UI의 "Finance Copilot" |
| **Hub** | 중앙 집중식 크로스 시스템 오케스트레이션 — 모든 시스템 연결 | Portal / API | Agent가 ERP를 쿼리하고, OA를 확인하고, Lark를 통해 알림 |## Hub 아키텍처

Hub는 핵심 차별화 요소입니다 — 모든 시스템이 AI와 만나는 중앙 포털입니다:

```mermaid
flowchart LR
  ERP --> Hub
  Database --> Hub
  Lark --> Hub
  Hub --> CRM
  Hub --> OA
  Hub["FIM One Hub<br/>(AI orchestration)"] --> CustomAPI["Custom API"]
```

각 connector는 표준화된 브릿지입니다. agent는 SAP와 통신하는지 또는 사용자 정의 PostgreSQL 데이터베이스와 통신하는지 알거나 신경 쓰지 않습니다. 데이터는 시스템에 유지되며, FIM One은 이들을 조율하는 AI 계층을 제공합니다.## 시작하기
