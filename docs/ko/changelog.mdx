이 프로젝트의 모든 주목할 만한 변경 사항이 여기에 기록되어 있습니다.

형식은 [Keep a Changelog](https://keepachangelog.com/)를 따릅니다. 버전은 [로드맵](/roadmap) 마일스톤에 해당합니다.

## [미출시]

### 추가됨

- **워크플로우 테스트-노드 엔드포인트** — `POST /api/workflows/{id}/test-node`으로 모의 VariableStore를 사용한 격리된 단일 노드 실행, 30초 타임아웃, START/END 거부. 프론트엔드 `TestNodeDialog` 시트에 키-값 변수 입력, 환경 변수 섹션, JSON 결과 뷰어. 노드 구성 패널(테스트 버튼) 및 편집기 컨텍스트 메뉴에 통합. 23개 테스트.
- **워크플로우 버전 비교 대화상자** — 두 블루프린트 버전의 나란히 비교로 추가/제거/수정된 노드 및 엣지 표시. 색상 코딩된 표시기(초록색=추가됨, 빨강색=제거됨, 황색=수정됨). 버전 기록 시트에 통합되며 버전당 "비교" 버튼 포함.
- **35개 엔드-투-엔드 라이프사이클 테스트** — 생성, 편집, 검증, 실행, 버전 관리, 복원, 삭제 흐름을 포함한 포괄적인 워크플로우 라이프사이클 테스트.
- **목록 및 카드의 워크플로우 실행 통계** — 백엔드에서 GROUP BY 서브쿼리를 통해 실행 횟수 및 성공률을 일괄 가져오기; 프론트엔드는 색상 코딩된 성공률 표시기가 있는 워크플로우 카드에 통계 표시.
- **워크플로우 스케줄러 데몬** — 60초마다 폴링하여 기한이 된 크론 기반 워크플로우를 찾는 백그라운드 비동기 서비스. Croniter 타임존 지원, 세마포어 기반 동시성 제어, `last_scheduled_at` 추적, 예약된 실행 결과에 대한 웹훅 전달. 14개 스케줄러 데몬 테스트.
- **워크플로우 가져오기 충돌 해결기** — 워크플로우 가져오기 중 미해결 에이전트/커넥터/KB/MCP 참조 감지. 가시성 필터링을 사용한 일괄 DB 쿼리; 프론트엔드는 누락된 외부 종속성에 대한 토스트 경고 표시. 17개 가져오기 충돌 해결기 테스트.
- **워크플로우 삭제-실행 및 실행-초기화 엔드포인트** — `DELETE /api/workflows/{id}/runs/{run_id}`로 개별 실행 삭제; `DELETE /api/workflows/{id}/runs`로 모든 완료된 실행 초기화. 두 작업 모두에 대한 프론트엔드 확인 대화상자.
- **56개 워크플로우 검증 경고 테스트** — Phase 2 노드 검증 경고의 포괄적 범위.
- **137개 워크플로우 Phase 2 노드 실행기 테스트** — 모든 Phase 2 노드 유형 실행기에 대한 전체 테스트 범위.
- **예약된 트리거** — 워크플로우당 크론 구성(타임존 선택, 기본 입력, 다음 실행 시간 계산 포함). 사전 설정된 크론 버튼(5분마다 / 시간마다 / 매일 / 매주 / 매월)이 있는 프론트엔드 대화상자.
- **공개 API 트리거 엔드포인트** — 워크플로우당 API 키(`wf_` 접두사)로 사용자 인증 없이 외부 실행 가능, 속도 제한 포함.
- **API 키 관리 대화상자** — 워크플로우 API 키 생성, 재생성, 취소(복사 버튼 포함), 트리거 URL 표시, cURL/JavaScript 사용 예제.
- **실시간 실행 로그 뷰어** — 실행 패널의 시간순 SSE 이벤트 스트림(타임스탬프, 색상 코딩된 이벤트 유형 배지, 이벤트 유형별 필터 토글 포함).
- **일괄 실행 대화상자** — 여러 입력 집합(JSON 편집기)으로 워크플로우 실행, 구성 가능한 병렬 처리(1-10), 항목별 결과 축소 가능, JSON 내보내기.
- **30개 예약된 트리거 테스트** — 크론 검증, 스키마 처리, 타임존 지원.
- **14개 일괄 실행 테스트** — 스키마 검증 및 동시 엔진 실행.
- **워크플로우 수준 타임아웃** — 엔진의 새로운 `workflow_timeout_ms` 매개변수는 초과 시 모든 대기 중/실행 중인 노드를 취소하며, 반응형 타임아웃 감지를 위한 제한된 비동기 대기.
- **마지막 입력으로 다시 실행** — "다시 실행" 버튼은 이전 실행의 값으로 입력 양식을 미리 채웁니다.
- **실행 기록 상태 필터** — 완료됨/실패함/취소됨/실행 중으로 워크플로우 실행을 필터링하는 드롭다운.
- **워크플로우 버전 기록** — 블루프린트 저장 시 자동 버전 관리(이전 버전으로 되돌리기 위한 목록/가져오기/복원 API 엔드포인트 포함).
- **검증 엔드포인트** — `POST /validate`로 실행 없이 구조적 블루프린트 검증.
- **드라이-런 모드** — `POST /run`에 `dry_run=true`를 사용하면 실행 계획을 JSON으로 반환하되 실행하지 않음.
- **13개 조건 분기 라우팅 테스트** — 분기, 병합, 중첩, 분류기 라우팅에 대한 포괄적 범위.
- **노드별 실행 통계 패널** — 노드별 성공률, 평균 지속 시간, 실패 횟수를 표시하는 새로운 통계 패널(`node-stats-panel.tsx`)(최악 우선 정렬). 워크플로우 도구 모음에 통합. 프론트엔드 소비를 위한 새로운 `getNodeStats` API 클라이언트 메서드 및 `NodeStatEntry` 유형.
- **Phase 3 노드: SubWorkflow 및 ENV** — 백엔드 실행기, 파서 검증, 14개 새로운 테스트(총: 304 → 306)를 포함한 두 가지 새로운 노드 유형:
  - **SubWorkflow**: 중첩된 워크플로우 오케스트레이션을 위한 스텁 실행기 — 대상 워크플로우를 선택하고 입력/출력 변수를 매핑합니다.
  - **ENV**: 런타임에 암호화된 환경 변수를 읽으며 키 선택 및 폴백 기본값 포함.
- **SubWorkflow 및 ENV 프론트엔드 컴포넌트** — 전체 프론트엔드 통합: `sub-workflow-node.tsx` 및 `env-node.tsx`(구성 양식, 팔레트 항목, 미니맵 색상, 모든 유형 맵의 노드 유형 레코드 항목 포함).
- **두 개의 새로운 워크플로우 템플릿(총 14개)** — "보안 API 통합"(ENV → HTTP with 인증 헤더) 및 "Sub-Workflow 오케스트레이션"(LLM → SubWorkflow → 요약).
- **키보드 단축키 대화상자** — `?`를 눌러 대화상자 오버레이에서 사용 가능한 모든 키보드 단축키 보기.
- **노드 입력 미리보기 캡처** — 실행 이벤트는 이제 디버깅을 위해 선행 출력의 잘린 JSON을 포함합니다.
- **워크플로우 분석 엔드포인트** — `GET /analytics`는 총 실행, 상태 분포, 성공률, p50/p95/p99 지속 시간 백분위수, 일일 추세, 가장 실패한 노드를 반환합니다.
- **분석 대시보드 패널** — 통계 카드, 상태 분포 막대, 순수 CSS 일일 차트, 백분위수 막대, 7일/14일/30일 범위 선택기가 있는 실패한 노드 테이블.
- **템플릿 갤러리 대화상자** — 카테고리 필터 및 원클릭 생성이 있는 워크플로우 템플릿의 시각적 그리드 브라우징.
- **레이블이 지정된 엣지** — 조건 및 분류기 분기 이름이 엣지의 약자로 표시됨.
- **워크플로우 변수 패널** — 유형/기본값/설명이 있는 워크플로우 수준 변수 정의, 편집, 삭제.
- **일괄 실행 엔드포인트** — `POST /batch-run`은 최대 100개의 입력 집합으로 워크플로우를 동시에 실행하며, 구성 가능한 병렬 처리(1–10).
- **실행 비교 대화상자** — 노드별 상태, 지속 시간, 입력 차이를 표시하는 두 실행의 나란히 비교.
- **웹훅 구성 대화상자** — 인라인 검증 및 연결 테스트를 사용하여 웹훅 URL 설정, 테스트, 초기화.
- **캔버스 노드 검색** — `Cmd+F`로 유형, ID, 콘텐츠별 노드 검색(강조 표시된 일치 항목 및 화살표 키 탐색 포함).
- **노드 주석/주석** — 구성 패널의 축소 가능한 메모 텍스트 영역(캔버스 표시기 아이콘 포함).

### 변경사항

- **NodeTest 스키마 이름 변경** — `TestNodeRequest`/`TestNodeResponse`를 `NodeTestRequest`/`NodeTestResponse`로 이름 변경하여 pytest 수집 경고 방지.

- **실행 패널 Results/Logs 탭** — 실행 패널이 이제 탭 레이아웃(Results + Logs)을 사용하여 실행 출력 및 이벤트 스트림을 더 잘 정리.
- **elkjs를 경량 BFS 자동 레이아웃으로 교체** — elkjs(430 kB)를 커스텀 BFS 알고리즘으로 교체; `/workflows/[id]` 번들이 473 kB에서 43 kB로 감소.
- **워크플로우 목록 정렬 옵션 확대** — 최신순, 오래된순, 이름 A-Z, 이름 Z-A, 최근 업데이트순.
- **워크플로우 테스트 스위트 306개 테스트로 확대** — SubWorkflow 및 ENV 노드 실행기를 위한 14개 Phase 3 테스트 추가. 노드별 실행 통계 테스트 추가. 총: 275 → 306.
- **워크플로우 테스트 스위트 275개 테스트로 확대** — 고급 엔진 테스트 23개 추가(지수 백오프를 사용한 재시도, 재시도 중 취소, 표현식 평가, 변수 저장소 헬퍼) 별도 테스트 파일에 추가. 총: 252 → 275.
- **워크플로우 테스트 스위트 252개 테스트로 확대** — HumanIntervention 테스트 8개(자동 승인, 보간, 담당자/타임아웃, 기본값, 유효성 검사 경고) 및 새 템플릿 테스트 4개(인간 승인 파이프라인, 다단계 데이터 처리, 지식 강화 에이전트, 반복적 검토 루프) 추가. 총: 240 → 252.
- **워크플로우 테스트 스위트 240개 테스트로 확대** — Phase 2 유효성 검사 테스트 15개(ListOperation, Transform, Iterator, Loop, DocumentExtractor, QuestionUnderstanding, ParameterExtractor 경고용) 및 Phase 2 템플릿 테스트 5개(새 템플릿용) 추가. 총: 220 → 240.
- **워크플로우 테스트 스위트 220개 테스트로 확대** — DocumentExtractor용 20개 테스트(12개: full_text/pages/metadata/tables 모드, base64 디코드, page_range 파싱, URL 스텁 오류, 빈 입력, 대시 구분 기호) 및 QuestionUnderstanding용 8개 테스트(LLM 모킹을 사용한 rewrite/classify/decompose/expand 모드, 커스텀 시스템 프롬프트, 변수 보간, 누락된 입력) 추가. 총: 200 → 220.
- **워크플로우 테스트 스위트 200개 테스트로 확대** — ListOperation용 50개 테스트(21개: filter, map, sort, slice, flatten, unique, reverse, length, JSON 문자열 입력, 잘못된 타입, 누락된 구성, 인덱스 범위 내, 기본 출력 변수) 및 Transform용 29개 테스트(json_path, 모든 타입의 type_cast, 형식 템플릿, regex_extract, 모든 변형의 string_op, 모든 변형의 math_op, 5단계 파이프라인 체이닝, 0으로 나누기, 와일드카드 경로, 알 수 없는 작업) 추가. 총: 150 → 200.
- **워크플로우 테스트 스위트 150개 테스트로 확대** — Loop용 12개 테스트(6개: 기본 카운팅, max_iterations 안전성, 0 반복, 빈 조건, 커스텀 변수 이름, 변수 보간) 및 ParameterExtractor용 6개 테스트(기본 추출, 누락된 입력, 빈 매개변수, 잘못된 JSON, 변수 보간, 추출 프롬프트 포함) 추가. 총: 138 → 150.
- **워크플로우 테스트 스위트 138개 테스트로 확대** — Phase 2 Iterator용 20개 테스트(9개: 기본 목록, JSON 파싱, max_iterations, 빈 목록, 누락된 구성, None-to-empty, 템플릿 참조, 비목록 거부, 메타데이터) 및 VariableAggregator용 11개 테스트(list/concat/merge/first_non_empty 모드, 빈 입력의 엣지 케이스, None 건너뛰기, 알 수 없는 모드) 추가. 총: 118 → 138.

- **워크플로우 편집기 전면 개편(Dify 스타일)** — Dify에서 영감을 받은 워크플로우 편집기의 완전한 시각적 재설계:

  - 왼쪽 색상 막대 노드 디자인, 컴팩트한 220px 너비, 작은 아이콘, 실행 상태 배지
  - 엣지에 노드 추가: "+" 중점 버튼과 기존 엣지를 분할하는 노드 타입 선택기 팝업이 있는 커스텀 `AddNodeEdge`
  - 더 풍부한 구성 패널: LLM 노드용 모델 선택기 드롭다운, 조건용 연산자 드롭다운(==, !=, >, <, contains 등), 섹션 헤더, 모든 출력 노드용 `InsertVariableHint` 및 `OutputVariableField`

  - 드롭 시 단일 Start/End 유효성 검사; 작은 그래프에서 과도한 줌을 방지하기 위해 `maxZoom:1`을 사용한 `fitViewOptions`

### 추가됨

- **워크플로우 엔진 노드 재시도** — 노드별 재시도 및 지수 백오프. 노드는 데이터 구성에서 `retry_count` 및 `retry_delay_ms`를 지정할 수 있습니다. 실패한 노드는 최대 N회 재시도되며 지연은 60초로 제한됩니다. 시도 번호 및 이전 오류와 함께 `node_retrying` SSE 이벤트를 발생시킵니다. 재시도 대기 중 취소 이벤트를 준수합니다.
- **워크플로우 변수 저장소 표현식 평가** — simpleeval을 통한 안전한 Python 표현식 평가. `evaluate_expression()` 메서드는 저장소 변수(env 제외)를 일반 내장 함수(len, str, int, float, bool, abs, min, max, sum, round, sorted)와 함께 노출합니다. 또한 `has()`, `delete()`, `keys()` 헬퍼 메서드를 포함합니다.
- **워크플로우 Phase 2 노드: HumanIntervention** — 승인 대기 노드(자동 승인 스텁). prompt_message({{variable}} 보간 포함), assignee, timeout_hours를 읽습니다. Sky-500 테마 프론트엔드 노드(프롬프트 미리보기 및 @assignee 배지 포함). 프롬프트 텍스트 영역, assignee 입력, 타임아웃 필드가 있는 구성 패널.
- **워크플로우 템플릿 12개로 확대** — 4개의 새로운 템플릿: Human Approval Pipeline(LLM → HumanIntervention → ConditionBranch → 이중 End), Multi-Step Data Processing(CodeExecution → ConditionBranch → Transform → ListOperation), Knowledge-Enhanced Agent(QuestionUnderstanding → KnowledgeRetrieval → LLM), Iterative Review Loop(Loop → LLM → VariableAssign).
- **워크플로우 창 컨텍스트 메뉴** — 빈 캔버스에서 마우스 우클릭하여 Paste(클립보드가 비어있을 때 비활성화), Select All, Fit View, Auto Layout 작업에 접근합니다. 판별된 합집합 상태는 노드 vs 창 컨텍스트 메뉴를 분리합니다.
- **워크플로우 노드 컨텍스트 메뉴** — 모든 노드를 마우스 우클릭하여 Configure, Copy, Duplicate, Delete 작업에 접근합니다. Delete 작업은 Start/End 싱글톤 노드에 대해 숨겨집니다. 메뉴는 외부 클릭 또는 스크롤 시 자동으로 닫힙니다.
- **워크플로우 Phase 2 템플릿** — 2개의 새로운 기본 제공 템플릿(총 8개): List Processing Pipeline(Start → ListOperation → Transform → End) 및 Enhanced Question Answering(Start → QuestionUnderstanding → LLM → End).
- **워크플로우 Phase 2 검증** — ListOperation(누락된 입력/표현식), Transform(누락된 입력/빈 작업), Iterator(누락된 리스트 변수), Loop(누락된 조건), DocumentExtractor, QuestionUnderstanding(누락된 입력), ParameterExtractor(누락된 입력 텍스트/빈 매개변수)에 대한 확장된 청사진 검증 경고. 수정: 정렬 작업은 더 이상 표현식을 잘못 요구하지 않습니다.
- **워크플로우 노드 메모** — 노드별 주석 기능. 모든 노드의 구성 패널에서 "Note"를 클릭하여 캔버스의 노드 아래에 기울임꼴 텍스트로 표시되는 자유형 메모를 추가합니다. `data.note` 저장소가 있는 자동 확장 텍스트 영역.
- **워크플로우 격자 스냅** — 정확한 노드 정렬을 위해 점 배경 패턴과 일치하는 16px 스냅 격자.
- **워크플로우 연결 검증** — 기존 Start/End 제한 외에도 자체 연결 및 중복 간선을 방지합니다.
- **워크플로우 줌 인식 드롭** — 노드 팔레트 드래그 앤 드롭은 모든 줌 레벨에서 올바른 배치를 위해 `screenToFlowPosition()`을 사용합니다.
- **워크플로우 SSE 다중 라인 데이터** — SSE 파서는 이제 다중 라인 `data:` 필드를 처리하고 keepalive 주석 라인을 무시합니다.
- **워크플로우 키보드 단축키** — Copy(Cmd+C), Paste(Cmd+V), Duplicate(Cmd+D), Select All(Cmd+A), Escape로 선택 해제. 복사/붙여넣기는 새 ID를 생성하고 위치를 오프셋합니다. Start/End 노드는 싱글톤 제약을 유지하기 위해 복사에서 제외됩니다.
- **워크플로우 인라인 설명 편집** — 도구 모음의 설명 라인을 클릭하여 팝오버 텍스트 영역을 열어 워크플로우를 떠나지 않고 설명을 편집합니다.
- **워크플로우 ConditionBranch LLM 모드** — 순진한 "첫 번째 일치" 스텁을 대체하는 적절한 LLM 기반 조건 평가. 조건 레이블에서 분류 프롬프트를 구성하고, 빠른 LLM을 호출하며, 대소문자 구분 없음 + 퍼지 매칭을 사용합니다.
- **반응형 간선 레이블** — 조건/분류자 분기의 간선 레이블은 이제 구성 패널에서 조건 또는 클래스를 편집할 때 실시간 업데이트를 위해 `useNodesData()`를 사용합니다.
- **워크플로우 통계 패널** — 총 실행, 완료/실패/취소 횟수, 시각적 진행률 표시줄이 있는 성공률, 평균 지속 시간, 상대 형식의 마지막 실행 시간을 표시하는 시트 기반 통계 표시. 도구 모음의 BarChart3 아이콘에서 트리거됩니다.
- **Phase 2 워크플로우 노드: DocumentExtractor & QuestionUnderstanding** — 2개의 추가 노드 유형:
  - **DocumentExtractor**: 문서 텍스트 처리를 위한 스텁 실행자 — text/base64 입력 유형 및 4가지 추출 모드(full_text, page_range가 있는 pages, word/char/line/page 개수가 있는 metadata, 마크다운 테이블 감지가 있는 tables)를 지원합니다. URL 입력은 나중에 구현하기 위해 연기됩니다.
  - **QuestionUnderstanding**: 4가지 모드가 있는 LLM 기반 질문 전처리 — rewrite(명확성), expand(컨텍스트 + 부분 질문), classify(의도/주제/신뢰도 JSON), decompose(부분 질문 배열). 선택적 사용자 정의 시스템 프롬프트 재정의가 있는 빠른 LLM을 사용합니다.
  - 전체 프론트엔드: 노드 구성 요소, 구성 패널, 팔레트 항목(QU의 경우 AI 범주, DE의 경우 Data 범주), 변수 선택기 지원.
- **Phase 2 워크플로우 노드: ListOperation & Transform** — 2개의 데이터 처리 노드 유형:
  - **ListOperation**: 8가지 리스트 작업(filter, map, sort, slice, flatten, unique, reverse, length) 및 filter/map/sort에 대한 simpleeval 표현식. JSON 문자열 입력 자동 파싱을 지원합니다.
  - **Transform**: 6가지 작업 유형(json_path, type_cast, format, regex_extract, string_op, math_op)이 있는 파이프라인 기반 데이터 변환. 작업이 순차적으로 적용됩니다 — 하나의 출력이 다음 입력으로 공급됩니다.
  - 전체 프론트엔드: 노드 구성 요소, 작업별 편집기가 있는 구성 패널, 팔레트 항목, 변수 선택기 지원.
- **Phase 2 워크플로우 노드: Loop & ParameterExtractor** — 2개의 추가 고급 노드 유형:
  - **Loop**: simpleeval 표현식 평가, 구성 가능한 루프 변수 이름, max_iterations 안전 제한이 있는 while 조건 루프. 조건 입력, 변수 이름, 반복 제한 구성이 있는 프론트엔드.
  - **ParameterExtractor**: 텍스트에서 LLM 기반 구조화된 매개변수 추출. 입력된 매개변수(이름/유형/설명/필수) 정의, 입력 텍스트 참조 제공, 선택적으로 추출 지침 추가. 구문 분석된 JSON을 반환합니다. 매개변수 목록 편집기 및 추출 프롬프트 텍스트 영역이 있는 프론트엔드.
- **Phase 2 워크플로우 노드: Iterator & VariableAggregator** — 2개의 새로운 고급 노드 유형:
  - **Iterator**: 구성 가능한 항목/인덱스 변수 이름 및 max_iterations 안전 제한이 있는 리스트 반복. 기본 리스트, JSON 문자열 파싱, 템플릿 변수 참조를 지원합니다.
  - **VariableAggregator**: 4가지 모드를 통해 여러 업스트림 분기의 출력 병합 — 리스트로 수집, 문자열 연결(사용자 정의 구분자), 딕셔너리 병합, 또는 첫 번째 비어있지 않은 값 선택.
  - 전체 프론트엔드 통합: 노드 구성 요소, 팔레트 항목, 구성 패널 양식, 미니맵 색상, 변수 선택기 지원. 백엔드 실행자는 엔진에 등록됩니다.
- **워크플로우 템플릿 확대** — 2개의 새로운 기본 제공 템플릿 추가(총 6개): Data Processing Pipeline(Start → CodeExecution → TemplateTransform → End) 및 Agent with Knowledge Retrieval(Start → KnowledgeRetrieval → Agent → End). 새 템플릿에서 누락된 `icon` 필드 수정. "intermediate" 및 "advanced" 템플릿 범주에 대한 누락된 i18n 키 추가.

- **워크플로우 청사진 시스템** — 다단계 자동화 청사진을 설계하고 실행하기 위한 시각적 워크플로우 편집기:
  - **12가지 노드 유형**: Start, End, LLM, ConditionBranch, QuestionClassifier, Agent, KnowledgeRetrieval, Connector, HTTPRequest, VariableAssign, TemplateTransform, CodeExecution — 각각 사용자 정의 React Flow 노드 구성 요소, 유형별 아이콘, 전용 구성 패널 양식이 있습니다.
  - **실행 엔진**: 노드 실행 순서에 대한 위상 정렬, `asyncio.Semaphore` 기반 동시성 제어, `active_handles`를 기반으로 다운스트림 간선을 활성화/비활성화하는 조건/분류자 분기가 있는 `WorkflowEngine`.
  - **SSE 스트리밍**: Server-Sent Events를 통한 실시간 워크플로우 실행 진행 — `run_started`, `node_started`, `node_completed`, `node_failed`, `node_skipped`, `run_completed`, `run_failed` 이벤트를 발생시킵니다. 장시간 실행 워크플로우에 대한 SSE keepalive; 클라이언트 연결 해제 감지는 취소를 트리거합니다.
  - **변수 저장소**: `{{node_id.output}}` 보간, 플랫 이름 폴백 해석, 암호화된 환경 변수에 대한 `env.*` 네임스페이스, `snapshot_safe()`(비밀 제외), 프론트엔드 선택기에 대한 `list_available_variables()`가 있는 스레드 안전 `VariableStore`.
  - **간선 레이블**: `WorkflowEdgeDef`는 조건/분류자 분기에 대한 `source_handle` 및 `target_handle`을 전달하여 각 연결이 나타내는 분기 경로를 시각적으로 나타내는 레이블이 지정된 간선을 활성화합니다.
  - **변수 선택기 콤보박스**: 업스트림 노드에서 `{{node_id.output}}` 참조를 삽입하기 위한 검색 가능한 콤보박스(`Command` 기반 팝오버) — 노드 유형별로 그룹화되고 아이콘이 있으며, 모든 프롬프트/템플릿/본문 필드에 통합됩니다.
  - **실행 기록 시트**: 상태, 지속 시간, 입력/출력 데이터, 노드별 결과를 표시하는 과거 워크플로우 실행 실행을 표시하는 측면 시트. `GET /api/workflows/{id}/runs`를 통해 페이지 매김됩니다.
  - **워크플로우 복제**: `POST /api/workflows/{id}/duplicate` 엔드포인트 및 UI 작업으로 기존 워크플로우를 복제합니다(청사진, 스키마, 설명 복사; 상태를 초안으로 재설정).
  - **청사진 검증**: `POST /api/workflows/validate` 엔드포인트 — 저장하지 않고 청사진을 검증합니다(Start/End 노드 존재, 중복 ID, 알 수 없는 노드 유형, 간선 참조, 사이클 감지 확인). 성공 시 노드/간선 개수를 반환하거나 특정 검증 오류를 반환합니다.
  - **오류 전략**: 노드별 `error_strategy` 필드 및 3가지 모드 — `STOP_WORKFLOW`(기본값: 모든 나머지 노드 중지), `CONTINUE`(실패로 표시하되 다운스트림이 진행되도록 허용), `FAIL_BRANCH`(실패한 노드를 통해서만 도달 가능한 모든 노드 건너뛰기).
  - **노드별 타임아웃**: 노드별 구성 가능한 `timeout_ms`(기본값 30초) 및 `asyncio.wait_for`를 통해 적용됨; 타임아웃은 설명적 오류 메시지가 있는 구조화된 `FAILED` 결과를 생성합니다.
  - **노드 구성 패널**: 유형별 구성 양식 — LLM 노드는 시스템 프롬프트, 모델 선택기, 모델 계층(fast/main) 토글, 온도 슬라이더, 최대 토큰을 가집니다. ConditionBranch는 연산자 드롭다운(==, !=, >, <, contains 등) 및 표현식/LLM 모드를 가집니다. HTTPRequest는 메서드 선택기, URL, 헤더 편집기(키-값 쌍 추가/제거), 변수 삽입이 있는 본문을 가집니다. CodeExecution는 언어 선택기(Python/JavaScript) 및 구문 적절한 힌트를 가집니다. 모든 출력 노드는 `OutputVariableField` 및 `InsertVariableBar`를 포함합니다.
  - **간선에 노드 추가**: "+" 중점 버튼이 있는 사용자 정의 `AddNodeEdge` 구성 요소 — 클릭하면 노드 유형 선택기 팝업을 열어 새 노드를 인라인으로 삽입하고 기존 간선을 두 개의 연결로 분할합니다.

### 수정됨

- **워크플로우 eval 네임스페이스 평탄화**: ConditionBranch와 VariableAssign이 짧은 변수 이름(예: 저장소에 `input.score`가 있을 때 `score`)을 해석하지 못하는 문제를 수정했습니다. eval 네임스페이스에서 짧은 별칭을 생성하는 `_flatten_eval_names()` 헬퍼를 추가하여 전체 경로 참조 없이 점 표기법 키 해석을 가능하게 했습니다.
- **워크플로우 `run_started` SSE 이벤트**: `run_started` 이벤트는 SSE 프로토콜에 문서화되었지만 실제로 발생하지 않았습니다. 이제 오케스트레이터 루프가 시작되기 전에 발생하므로 프론트엔드 소비자가 실행 시작 신호를 받습니다.
- **워크플로우 SSE 이벤트 이름 불일치**: SSE 이벤트 이름을 수정했습니다 — `node_status`를 `node_started`/`node_completed`/`node_failed`/`node_skipped`로 이름을 변경하여 프론트엔드 SSE 핸들러 예상과 일치하도록 했습니다.
- **워크플로우 `NodeRunResult.duration` 필드 이름**: `duration`을 `duration_ms`로 수정하여 Pydantic 스키마 및 프론트엔드 파싱과 일치하도록 했습니다.
- **워크플로우 내보내기/가져오기 API 응답 래핑**: 워크플로우 내보내기 및 가져오기 엔드포인트의 응답 봉투 래핑을 수정했습니다.
- **`is_active` 토글 시 재검토 없음**: 에이전트, 커넥터, 지식 베이스 또는 MCP 서버에서 `is_active`를 토글해도 더 이상 `publish_status`가 `approved`에서 `pending_review`로 되돌아가지 않습니다. `check_edit_revert` 가드는 이제 실제 콘텐츠 필드가 변경될 때만 작동합니다(`is_active`만 변경되는 경우 제외).

## [v0.5] - 2026-02-28

### 추가됨
- **Full RAG pipeline**: Jina embedding + LanceDB vector store + FTS + RRF hybrid retrieval + reranker
- **Grounded Generation**: Evidence-anchored RAG with `[N]` citations, conflict detection, explainable confidence scores
- **KB document management**: Chunk-level CRUD, text search, failed doc retry, auto-migrating vector store schema
- **ContextGuard + Pinned Messages**: Token budget manager; pinned messages protected from compaction

### Changed
- **Multi-turn memory**: DbMemory in `core/memory/db.py`, CompactUtils in `core/memory/compact.py`
- **LLM Compact**: `CompactUtils.llm_compact()` summarizes old turns via fast LLM
- **DAG Re-Planning**: Loop up to `MAX_REPLAN_ROUNDS=3`; re-plans when `analysis.achieved=False` and `confidence < 0.5`

### 수정됨
- **토큰 오버플로우 처리**: 자동 LLM 기반 요약이 토큰 예산을 준수합니다
