이 프로젝트의 모든 주목할 만한 변경 사항이 여기에 문서화되어 있습니다.

형식은 [Keep a Changelog](https://keepachangelog.com/)를 따릅니다. 버전은 [Roadmap](/roadmap) 마일스톤에 해당합니다.## [미출시]### 추가됨

- **Admin: Review Log 감사 탭** — Admin 감사 페이지에 이제 분할된 "System Log / Review Log" 토글이 있습니다. Review Log 패널은 새로운 `GET /api/admin/review-log` 엔드포인트를 쿼리하고(필터: org, resource type, action; limit/offset 페이지네이션), 시간, org 이름, resource type, resource 이름, action 배지(색상 코드: 녹색=승인됨, 빨강=거부됨, 황색=제출됨/재제출됨, 회색=미발행됨), 운영자를 표시하는 테이블을 표시합니다. 전체 i18n (EN + ZH).
- **Platform Organization**: 기본 제공 org `platform` (ID: `00000000-0000-0000-0000-000000000001`) — 모든 신규 사용자가 자동으로 참여합니다. 이 조직은 "global" 가시성 개념을 대체하고 모든 사용자 간에 리소스를 공유하기 위한 중앙 허브를 제공합니다.
- **Resource Subscriptions**: 사용자는 이제 org Markets에서 공유 리소스(agents, connectors, knowledge bases, MCP servers)를 구독/구독 취소할 수 있습니다. 새로운 `resource_subscriptions` 데이터베이스 테이블이 구독 관계를 추적합니다.
- **Market API**: 구독 검색 및 관리를 위한 새로운 엔드포인트:
  - `GET /api/market` — 현재 org에서 사용 가능한 리소스 검색
  - `POST /api/market/subscribe` — 리소스 구독
  - `DELETE /api/market/unsubscribe` — 리소스 구독 취소
  - `GET /api/market/subscriptions` — 사용자의 현재 구독 목록
- **Market Page**: Portal의 새로운 `/market` 페이지로 org 리소스에 대한 구독을 검색하고 관리합니다.
- **Agent Auto-discovery**: agents의 새로운 `discoverable` 부울 필드입니다. `discoverable=true`인 agents는 `call_agent` 도구를 통해 LLMs에 의해 자동으로 발견되고 호출될 수 있습니다(1단계 위임, 재귀 없음).
- **Sub-agent Binding**: agents의 새로운 `sub_agent_ids` 필드 — 세분화된 액세스 제어를 위한 호출 가능한 sub-agents의 명시적 화이트리스트입니다.
- **CallAgentTool**: 구조화된 입력 검증 및 결과 처리를 통해 specialist agents에게 작업을 위임하는 기본 제공 도구입니다.
- **MCP Server Credentials**: 새로운 `mcp_server_credentials` 테이블은 MCP servers에 대한 사용자별 환경 변수 및 HTTP 헤더 재정의를 활성화합니다.
- **MCP Credentials Endpoint**: `PUT /api/mcp-servers/{id}/my-credentials`를 통해 사용자는 공유 MCP servers에 대한 개인 자격증명 재정의를 저장할 수 있습니다.
- **MCPServer의 `allow_fallback`**: 자격증명 폴백 동작을 제어하는 부울 필드 — `true`일 때, 누락된 사용자 자격증명은 소유자 기본값으로 폴백됩니다.
- **Connector/KB Toggle**: 리소스 상태 관리를 위한 새로운 엔드포인트:
  - `POST /api/connectors/{id}/toggle` — connector 일시 중단/재개
  - `POST /api/knowledge-bases/{id}/toggle` — knowledge base 일시 중단/재개
- **Standalone KB Conversations**: `ChatStreamRequest` 및 `Conversation`의 새로운 `kb_ids` 필드 — 사용자는 이제 agent에 바인딩하지 않고도 knowledge bases와 직접 채팅할 수 있습니다.
- **Platform Org Protection**: Platform org는 삭제할 수 없으며, 사용자는 이를 자체 탈퇴할 수 없습니다. 모든 사용자가 조직 허브에 대한 연결을 유지하도록 보장합니다.
- **Publish Review UI (Frontend)**: Org 수준 발행 검토 시스템 프론트엔드 — org 편집 대화상자의 검토 토글, 유형/상태 필터 및 승인/거부 워크플로우가 있는 ReviewsSheet, agent 카드의 발행 상태 배지(대기 중/승인됨/거부됨) 및 거부 사유에 대한 툴팁, 대상 org가 검토를 요구할 때 발행 대화상자의 검토 공지, 거부된 리소스에 대한 재제출 작업, org admin/owners를 위한 "Review Management" 드롭다운입니다. 모든 리소스 유형(agents, connectors, KBs, MCP servers)은 `publish_status`, `reviewed_by`, `reviewed_at`, `review_note` 유형 필드를 획득합니다. 전체 i18n (EN + ZH).### 수정됨

- **`is_active` 토글 시 재검토 없음**: 에이전트, 커넥터, 지식 베이스 또는 MCP 서버에서 `is_active`를 토글해도 더 이상 `publish_status`가 `approved`에서 `pending_review`로 되돌아가지 않습니다. `check_edit_revert` 가드는 이제 실제 콘텐츠 필드가 변경될 때만 작동합니다(`is_active`만 변경되는 경우는 제외).### 변경 사항

- **가시성 단순화**: 리소스 가시성이 `personal | org | global`에서 `personal | org`로 축소되었습니다. "Global" 리소스는 이제 Platform org에 게시되며, 역호환성을 유지하면서 가시성 모델을 단순화합니다.
- **게시 인터페이스 통합**: 모든 4가지 리소스 유형(Agent, Connector, Knowledge Base, MCP Server)은 이제 게시를 위해 `scope="org"`만 지원합니다. `scope="global"` 또는 관리자 전용 글로벌 게시 엔드포인트는 더 이상 없습니다.
- **자동 Platform 등록**: 새 사용자는 등록 시 자동으로 Platform org에 추가되어 공유 조직 리소스에 즉시 액세스할 수 있습니다.
- **관리자 패널 정리**: "Global Agents" 및 "Global MCP" 관리자 탭이 제거되었습니다. 글로벌 리소스는 이제 Platform org를 통해 관리됩니다.### 제거됨

- Agent 및 MCPServer 모델과 모든 API 응답에서 `is_global` 필드 제거
- Agent 및 MCPServer에서 `cloned_from_agent_id` 및 `cloned_from_mcp_server_id` 필드 제거
- 전역 agent 관리 엔드포인트(`/api/admin/global-agents/*`) 제거
- 전역 MCP server 엔드포인트(`/api/admin/mcp-servers/*`) 제거
- 리소스, API 계약 및 프론트엔드 로직에서 모든 `visibility="global"` 개념 참조 제거## [v0.7.5] - 2026-03-12### 추가됨
