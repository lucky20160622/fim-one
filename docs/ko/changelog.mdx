이 프로젝트의 모든 주목할 만한 변경 사항이 여기에 기록되어 있습니다.

형식은 [Keep a Changelog](https://keepachangelog.com/)를 따릅니다. 버전은 [로드맵](/roadmap) 마일스톤에 해당합니다.

## [미출시]

### 변경 사항

- **워크플로우 테스트 스위트 118개 테스트로 확장** — ConditionBranch LLM 모드에 대한 7개 테스트 추가 (정확한 일치, 대소문자 구분 없음, 퍼지 매칭, 일치 없음 폴백, DEFAULT 응답, 후보 없음, 변수 보간). 총계: 111 → 118.
- **워크플로우 실행 모드 시각적 피드백 대폐신** — 테두리 기반 접근 방식을 대체하는 링 기반 노드 상태 스타일링, 실행 중인 노드에 대한 핑 효과가 있는 애니메이션 상태 점, 깔끔한 상태 분리를 위한 `useMemo` 파생 `displayNodes`/`displayEdges`, 완료된 노드에서 실행 중인 노드로의 엣지 애니메이션.
- **워크플로우 테스트 스위트 105개 테스트로 확장** — 템플릿 검증 테스트, 가져오기/내보내기 왕복, 변수 저장소 고급 메서드, 교착 상태 감지, 다중 조건 분기, 질문 분류기, 포괄적인 블루프린트 검증 추가. 총계: 68 → 97 → 105.
- **워크플로우 테스트 스위트 68개 테스트로 확장** — eval 네임스페이스 평탄화(점 표기법 키 해석, 짧은 별칭 생성, 평탄화된 이름을 사용한 ConditionBranch/VariableAssign에 대한 17개 테스트) 및 블루프린트 검증 경고(연결되지 않은 노드, 도달 불가능한 끝, 빈 조건/프롬프트/코드에 대한 7개 테스트)를 다루는 24개의 새로운 테스트 추가. 총 워크플로우 테스트 수: 34 → 68.
- **워크플로우 편집기 대폐신(Dify 스타일)** — Dify에서 영감을 받은 워크플로우 편집기의 완전한 시각적 재설계:
  - 위에서 아래로의 레이아웃을 대체하는 수평(왼쪽에서 오른쪽) 레이아웃; 모든 12개 노드 유형이 Left/Right 핸들 사용
  - 왼쪽 색상 막대 노드 디자인, 컴팩트한 220px 너비, 더 작은 아이콘, 실행 상태 배지
  - 엣지에 노드 추가: "+" 중점 버튼과 기존 엣지를 분할하는 노드 유형 선택기 팝업이 있는 커스텀 `AddNodeEdge`
  - 더 풍부한 구성 패널: LLM 노드용 모델 선택기 드롭다운, 조건용 연산자 드롭다운(==, !=, >, <, contains 등), 섹션 헤더, 모든 출력 노드용 `InsertVariableHint` 및 `OutputVariableField`
  - 축소 가능한 노드 팔레트(아이콘 전용 모드), 카테고리 그룹화; 이미 배치된 경우 Start/End 노드 흐릿하게 표시
  - 드롭 시 단일 Start/End 검증; 작은 그래프에서 과도한 줌을 방지하기 위해 `maxZoom:1`이 있는 `fitViewOptions`

### 추가됨

- **워크플로우 노드 노트** — 노드별 주석 기능. 노드의 설정 패널에서 "Note"를 클릭하여 자유 형식 노트를 추가하면 캔버스의 노드 아래에 이탤릭 텍스트로 표시됩니다. 자동 확장 텍스트 영역과 `data.note` 저장소.
- **워크플로우 스냅-투-그리드** — 16px 스냅 그리드로 점 배경 패턴과 일치하여 정확한 노드 정렬 제공.
- **워크플로우 연결 검증** — 기존 시작/종료 제한 외에도 자체 연결 및 중복 간선 방지.
- **워크플로우 줌 인식 드롭** — 노드 팔레트 드래그 앤 드롭은 `screenToFlowPosition()`을 사용하여 모든 줌 레벨에서 올바른 배치 제공.
- **워크플로우 SSE 멀티라인 데이터** — SSE 파서는 이제 멀티라인 `data:` 필드를 처리하고 keepalive 주석 줄을 무시합니다.
- **워크플로우 키보드 단축키** — 복사(Cmd+C), 붙여넣기(Cmd+V), 복제(Cmd+D), 모두 선택(Cmd+A), Escape로 선택 해제. 복사/붙여넣기는 새 ID를 생성하고 위치를 오프셋하며, 시작/종료 노드는 싱글톤 제약을 유지하기 위해 복사에서 제외됩니다.
- **워크플로우 인라인 설명 편집** — 도구 모음의 설명 줄을 클릭하여 팝오버 텍스트 영역을 열고 워크플로우 설명을 편집할 수 있습니다.
- **워크플로우 ConditionBranch LLM 모드** — 순진한 "첫 번째 일치" 스텁을 대체하는 적절한 LLM 기반 조건 평가. 조건 레이블에서 분류 프롬프트를 구축하고, 빠른 LLM을 호출하며, 대소문자 구분 없음 + 퍼지 매칭을 사용합니다.
- **반응형 간선 레이블** — 조건/분류기 분기의 간선 레이블은 이제 `useNodesData()`를 사용하여 설정 패널에서 조건 또는 클래스를 편집할 때 실시간 업데이트.
- **워크플로우 ENV 변수 대화** — 워크플로우별 암호화된 환경 변수 관리 UI. 마스크된 표시, 중복 키 감지, 인라인 검증이 있는 키-값 편집기. `GET/PUT /api/workflows/{id}/env` 엔드포인트에 연결. 도구 모음 드롭다운을 통해 접근 가능.
- **워크플로우 자동 저장** — 5초 디바운스 자동 저장 타이머와 즉시 Cmd+S 오버라이드. 저장 상태 표시기는 "저장 중...", "저장되지 않은 변경 사항" 또는 "Xs 전에 저장됨"을 표시하며 30초 자동 새로고침.
- **워크플로우 목록 검색 및 필터** — 이름/설명으로 클라이언트 측 검색, 상태 필터 버튼(모두/초안/활성), 정렬 드롭다운(최근 업데이트/생성/이름 A-Z), 결과 없음에 대한 빈 상태.
- **워크플로우 색상 코드 미니맵** — 미니맵 노드 색상은 이제 각 노드 유형의 카테고리 색상과 일치합니다(녹색=시작, 빨강=종료, 파랑=LLM, 주황=조건 등).
- **에이전트 스킬 시스템** — 에이전트 지침을 위한 온디맨드 스킬 로딩: `Skill` 모델(이름, 설명, 콘텐츠, 스크립트)과 전체 CRUD, 게시 검토, 가시성 라이프사이클; `read_skill(name)` 내장 도구로 점진적 공개(~80% 토큰 감소); 시스템 프롬프트에 주입된 스킬 스텁(~15 토큰/스킬); 에이전트별 `compact_instructions` 필드로 사용자 정의 ContextGuard 압축 전략.
- **스킬 UI** — 스킬 목록 페이지, 전체 페이지 편집기(콘텐츠/스크립트/설정 탭), SkillCard 컴포넌트, 사이드바 네비게이션, 컴팩트 지침 텍스트 영역이 있는 에이전트 편집기 스킬 선택기.
- **워크플로우 게시 흐름** — 워크플로우의 게시/게시 취소/재제출 엔드포인트로 기존 에이전트 게시 패턴과 일치. Organization 모델의 새로운 `review_workflows` 플래그(Alembic 마이그레이션 포함)로 조직 수준 워크플로우 검토 게이팅 활성화. 게시 작업이 워크플로우 카드 및 도구 모음 UI에 연결됨.
- **워크플로우 블루프린트 시스템** — 다단계 자동화 블루프린트를 설계하고 실행하기 위한 시각적 워크플로우 편집기:
  - **12 노드 유형**: 시작, 종료, LLM, ConditionBranch, QuestionClassifier, 에이전트, KnowledgeRetrieval, 커넥터, HTTPRequest, VariableAssign, TemplateTransform, CodeExecution — 각각 사용자 정의 React Flow 노드 컴포넌트, 유형별 아이콘, 전용 설정 패널 양식.
  - **실행 엔진**: 노드 실행 순서를 위한 위상 정렬, `asyncio.Semaphore` 기반 동시성 제어, `active_handles`를 기반으로 다운스트림 간선을 활성화/비활성화하는 조건/분류기 분기를 포함한 `WorkflowEngine`.
  - **SSE 스트리밍**: Server-Sent Events를 통한 실시간 워크플로우 실행 진행 상황 — `run_started`, `node_started`, `node_completed`, `node_failed`, `node_skipped`, `run_completed`, `run_failed` 이벤트 발출. 장기 실행 워크플로우를 위한 SSE keepalive; 클라이언트 연결 해제 감지로 취소 트리거.
  - **변수 저장소**: `{{node_id.output}}` 보간, 플랫 이름 폴백 해석, 암호화된 환경 변수를 위한 `env.*` 네임스페이스, `snapshot_safe()`(비밀 제외), 프론트엔드 선택기를 위한 `list_available_variables()`를 포함한 스레드 안전 `VariableStore`.
  - **간선 레이블**: `WorkflowEdgeDef`는 조건/분류기 분기를 위해 `source_handle` 및 `target_handle`을 전달하여 각 연결이 나타내는 분기 경로를 시각적으로 나타내는 레이블이 지정된 간선 활성화.
  - **변수 선택기 콤보박스**: 업스트림 노드에서 `{{node_id.output}}` 참조를 삽입하기 위한 검색 가능한 콤보박스(`Command` 기반 팝오버) — 노드 유형별로 그룹화되고 아이콘 포함, 모든 프롬프트/템플릿/본문 필드에 통합.
  - **실행 기록 시트**: 상태, 기간, 입력/출력 데이터, 노드별 결과를 표시하는 과거 워크플로우 실행 실행을 표시하는 사이드 시트. `GET /api/workflows/{id}/runs`를 통한 페이지 매김.
  - **워크플로우 복제**: `POST /api/workflows/{id}/duplicate` 엔드포인트 및 UI 작업으로 기존 워크플로우 복제(블루프린트, 스키마, 설명 복사; 상태를 초안으로 재설정).
  - **블루프린트 검증**: `POST /api/workflows/validate` 엔드포인트 — 저장하지 않고 블루프린트 검증(시작/종료 노드 존재, 중복 ID, 알 수 없는 노드 유형, 간선 참조, 사이클 감지 확인). 성공 시 노드/간선 수를 반환하거나 특정 검증 오류 반환.
  - **오류 전략**: 노드별 `error_strategy` 필드로 세 가지 모드 — `STOP_WORKFLOW`(기본값: 모든 나머지 노드 중단), `CONTINUE`(실패로 표시하지만 다운스트림 진행 허용), `FAIL_BRANCH`(실패한 노드를 통해서만 도달 가능한 모든 노드 건너뛰기).
  - **노드별 타임아웃**: 노드별 구성 가능한 `timeout_ms`(기본값 30초)는 `asyncio.wait_for`를 통해 적용됨; 타임아웃은 설명 오류 메시지가 있는 구조화된 `FAILED` 결과 생성.
  - **노드 설정 패널**: 유형별 설정 양식 — LLM 노드는 시스템 프롬프트, 모델 선택기, 모델 계층(빠름/메인) 토글, 온도 슬라이더, 최대 토큰; ConditionBranch는 연산자 드롭다운(==, !=, >, <, contains 등)과 표현식/LLM 모드; HTTPRequest는 메서드 선택기, URL, 헤더 편집기(키-값 쌍 추가/제거), 변수 삽입이 있는 본문; CodeExecution는 언어 선택기(Python/JavaScript)와 구문 적절한 힌트; 모든 출력 노드는 `OutputVariableField` 및 `InsertVariableBar` 포함.
  - **간선에 노드 추가**: "+" 중점 버튼이 있는 사용자 정의 `AddNodeEdge` 컴포넌트 — 클릭하면 노드 유형 선택기 팝업이 열리고 새 노드를 인라인으로 삽입하여 기존 간선을 두 개의 연결로 분할.
  - **자동 레이아웃(ELK.js)**: `elk.direction: RIGHT`를 사용한 ELK.js 계층화 알고리즘으로 수평 좌에서 우로 노드 배열을 위한 `getAutoLayoutedNodes()` 유틸리티. 구성 가능한 노드 간격(노드 간 60px, 계층 간 120px).
  - **포괄적 테스트 스위트**: 파서 검증(시작/종료 존재, 중복 ID, 알 수 없는 유형, 사이클 감지, 오류 전략/타임아웃 파싱), 위상 정렬(선형 및 병렬 분기), `VariableStore`(가져오기/설정, 보간, env 주입, 스냅샷 안전성, 노드 출력), 엔진 실행(시작-종료, 입력 통과, 취소, STOP_WORKFLOW, CONTINUE 전략), 노드 실행기(VariableAssign, 변수가 있는 CodeExecution, 오류 처리)를 다루는 34개 테스트.
  - **백엔드 CRUD + API**: Alembic 마이그레이션이 있는 `Workflow` 및 `WorkflowRun` ORM 모델; Pydantic 스키마; 생성, 읽기, 업데이트, 삭제, 목록, 가져오기/내보내기가 있는 전체 REST API(`/api/workflows/*`); `CREDENTIAL_ENCRYPTION_KEY`를 통한 암호화된 환경 변수; 프론트엔드 설정 패널을 위한 변수 자체 검사 엔드포인트(`GET /api/workflows/{id}/variables`).
  - **보안**: 코드 실행 노드는 OS 수준 격리를 위해 베어 `exec()` 대신 `subprocess` 사용; eval/템플릿 실행 컨텍스트에서 필터링된 환경 변수; 모든 노드 실행기에 걸친 `ExecutionContext` 배선.
  - **프론트엔드**: React Flow v12로 구축된 시각적 편집기 — 카테고리 그룹화(축소 가능한 아이콘 전용 모드)가 있는 드래그 앤 드롭 노드 팔레트, 12개 사용자 정의 노드 컴포넌트, SSE 기반 라이브 실행 상태가 있는 실행 패널, 카드 그리드가 있는 워크플로우 목록 페이지. 전체 i18n 지원(6개 언어).

### 수정됨

- **워크플로우 eval 네임스페이스 평탄화**: ConditionBranch와 VariableAssign이 짧은 변수 이름(예: 저장소에 `input.score`가 있을 때 `score`)을 확인하지 못하는 문제를 수정했습니다. eval 네임스페이스에서 짧은 별칭을 생성하는 `_flatten_eval_names()` 헬퍼를 추가하여 전체 경로 참조 없이 점 표기법 키 확인을 가능하게 했습니다.
- **워크플로우 `run_started` SSE 이벤트**: `run_started` 이벤트는 SSE 프로토콜에 문서화되었지만 실제로 발생하지 않았습니다. 이제 오케스트레이터 루프가 시작되기 전에 발생하므로 프론트엔드 소비자가 실행 시작 신호를 받습니다.
- **워크플로우 SSE 이벤트 이름 불일치**: SSE 이벤트 이름을 수정했습니다 — `node_status`를 `node_started`/`node_completed`/`node_failed`/`node_skipped`로 이름 변경하여 프론트엔드 SSE 핸들러 예상과 일치하도록 했습니다.
- **워크플로우 `NodeRunResult.duration` 필드 이름**: `duration`을 `duration_ms`로 수정하여 Pydantic 스키마 및 프론트엔드 파싱과 일치하도록 했습니다.
- **워크플로우 내보내기/가져오기 API 응답 래핑**: 워크플로우 내보내기 및 가져오기 엔드포인트의 응답 봉투 래핑을 수정했습니다.
- **`is_active` 토글 시 재검토 없음**: 에이전트, 커넥터, 지식 베이스 또는 MCP 서버에서 `is_active`를 토글해도 더 이상 `publish_status`가 `approved`에서 `pending_review`로 되돌아가지 않습니다. `check_edit_revert` 가드는 이제 `is_active`뿐만 아니라 실제 콘텐츠 필드가 변경될 때만 작동합니다.

- **DAG: 축소된 단계로 단계로 스크롤**: 사이드바에서 단계 노드를 클릭하면 이제 축소된 단계 섹션을 자동으로 확장한 후 스크롤
- **DAG: 히스토리 턴이 단계 카드 표시**: 히스토리 턴이 이제 확장 가능한 섹션에서 단계 카드를 렌더링 (이전에는 `hideStepCards`로 숨겨짐)
- **DAG: 재계획 시 오래된 분석**: 1라운드 "목표 달성 안 됨" 분석이 이제 재계획 시작 시 즉시 삭제되고, 2라운드 "분석 중…" 스피너가 올바르게 나타남
- **UI: 채팅 검색 포커스 링**: 포커스 링 누수를 방지하기 위해 검색 입력에 `focus-visible:outline-none` 추가
- **내보내기: PDF 가독성**: 모든 PDF 스타일에서 줄 간격(leading) 및 셀 패딩 증가로 텍스트 가독성 향상
- **내보내기: 이모지 제거 삭제**: DOCX 및 PDF 내보내기 모두에서 최종 답변 렌더링에서 불필요한 `_strip_emoji()` 제거, 원본 콘텐츠 보존

### 수정됨
- **Export: RFC 5987 filename support**: Content-Disposition 헤더에서 일반 `filename` 대신 UTF-8 인코딩된 `filename*`을 선호하도록 하여, 대화 내보내기 시 CJK 파일명이 깨지는 문제 해결

- **Artifacts: sort by conversation created_at**: 파일 mtime 대신 대화 생성 시간으로 아티팩트 정렬(mtime은 Docker 볼륨에서 신뢰할 수 없음)
- **OAuth: auto-detect production API URL from window.location port**: 브라우저 자동 감지가 프로덕션의 커스텀 포트에서 올바르게 작동(3000만 아님)
- **Docker: DooD sandbox volume mount path mismatch**: `CODE_EXEC_BACKEND=docker`(Docker-outside-of-Docker)로 컨테이너 내부에서 실행할 때, 샌드박스 스크립트가 호스트 Docker 데몬에 보이지 않는 컨테이너 내부 경로에 작성되어 빈 `-v` 마운트 및 "No such file or directory" 오류 발생. `data/sandbox/`로 샌드박스 디렉토리 이동, 경로 변환을 위한 `_resolve_host_path()`가 포함된 `DOCKER_HOST_DATA_DIR` 환경 변수 추가, 빈 문자열 환경 변수 엣지 케이스 처리로 수정

## [v0.1] - 2026-02-22

### 추가됨
- **ReActAgent**: Reason → Act → Observe 루프와 도구
- **DAGPlanner**: LLM 생성 의존성 그래프, 동시 실행을 위한 DAGExecutor, 결과 검증을 위한 PlanAnalyzer
- **BaseTool 프레임워크**: 모든 도구를 위한 통합 인터페이스
- **Calculator 도구**: 산술 표현식, 공식 평가
- **Python exec 도구**: 격리된 환경에서의 코드 실행
- **KaTeX 렌더링**: 스트리밍 출력의 수학 표기법
- **JSON 모드 파싱**: 견고한 구조화된 출력 처리
- **Portal UI**: 실시간 스트리밍, 다크/라이트/시스템 테마가 있는 Next.js 웹 인터페이스
- **초기 문서**: 철학, 아키텍처, 로드맵을 다루는 Wiki 페이지
