이 프로젝트의 모든 주목할 만한 변경 사항이 여기에 기록되어 있습니다.

형식은 [Keep a Changelog](https://keepachangelog.com/)를 따릅니다. 버전은 [로드맵](/roadmap) 마일스톤에 해당합니다.

## [미출시]

### 추가됨

- **모델별 네이티브 함수 호출 토글** — 새로운 `tool_choice_enabled` 설정(ENV 전역 + 관리자 모델별)은 구조화된 출력 추출을 위해 강제 `tool_choice`를 시도할지 여부를 제어합니다. 강제 도구 선택을 거부하는 모델(예: 사고 모드 모델)은 레벨 1을 건너뛰고 JSON 모드로 직접 이동하여 구조화된 호출당 약 10초의 페널티를 제거할 수 있습니다. 설정 → 모델 → 고급에서 구성 가능합니다.
- **모델 문서의 구조화된 출력 호환성 표** — 어떤 제공자가 네이티브 함수 호출, JSON 모드 및 계획 안정성 등급을 지원하는지 보여줍니다.
- **파일 첨부 다운로드** — 채팅 메시지의 파일 첨부 카드를 클릭하여 인증된 fetch를 통해 원본 파일을 다운로드할 수 있습니다. 발견성을 위해 다운로드 아이콘 표시기 및 호버 상태가 추가되었습니다.
- **지능형 문서 처리(비전 인식) 로드맵 항목** — 모델 비전 기능에 따른 적응형 문서 처리를 위한 계획된 기능이 추가되었습니다(세 가지 계층: 비전 우선, 하이브리드, 텍스트 전용).
- **DAG 품질 개선(P0): 기본 모델 업그레이드** — 명시적 `model_hint`가 없는 DAG 단계는 이제 빠른 모델 대신 레지스트리의 일반 모델(예: Sonnet)을 사용합니다. `model_hint="fast"`가 있는 단계만 경량 모델을 사용합니다.
- **DAG 품질 개선(P1): 스킬 자동 발견** — `DAGPlanner.plan()`은 `skill_descriptions` 매개변수를 허용합니다. 사용 가능한 스킬(이름 + 설명)이 계획 프롬프트에 주입되어 플래너가 도메인과 일치하는 단계에 대해 `tool_hint="read_skill"`을 제안할 수 있습니다.
- **DAG 품질 개선(P2): 인용 검증자** — 새로운 `citation_verifier.py` 모듈은 정규식을 통해 법률/의료/금융 인용을 추출하고, LLM 판단을 통해 정확성을 검증하며, 수정 피드백과 함께 실패 시 재시도합니다. 실행자 단계 후 흐름에 통합됩니다. `DAG_CITATION_VERIFICATION` 환경 변수로 제어됩니다(기본값: `true`).
- **DAG 품질 개선(P3): 구조화된 콘텐츠 컨텍스트 보존** — `_build_step_context()`는 종속성 결과에서 구조화된 콘텐츠(법률 인용, 마크다운 테이블, 코드 블록)를 감지하고 구성 가능한 절단 승수(`DAG_STRUCTURED_CONTEXT_MULTIPLIER`, 기본값 `3.0`)를 적용하여 DAG 단계 간 중요한 인용 손실을 방지합니다.
- **DAG 품질 개선(P4): 라우터의 도메인 분류** — `RouteDecision`은 이제 `domain_hint`(법률/의료/금융/null)를 포함합니다. 긴밀하게 결합된 도메인 분석은 ReAct 모드를 선호하며, DAG가 선택되면 도메인 컨텍스트가 플래너에 주입되어 `model_hint` 선택을 안내합니다.
- **ReAct의 도메인 모델 에스컬레이션** — 라우터가 전문 도메인(법률/의료/금융)을 감지하면 ReAct 엔드포인트는 `registry.get_by_role("reasoning")`을 통해 일반 모델에서 추론 모델로 에스컬레이션합니다. 웹 검색 전에 작성하고 검색을 통한 인용 검증을 요구하는 필수 지침이 주입됩니다. `read_skill` 도구는 도구 선택에 고정됩니다.

### 변경사항

- **온도 힌트가 툴팁으로 이동** — 모델 폼 대화상자의 온도 추론 힌트가 이제 슬라이더 아래의 정적 단락 대신 레이블 옆의 컴팩트한 툴팁 아이콘(HelpCircle)으로 표시되어 시각적 혼란을 줄입니다.
- **RAG 기반 파이프라인** — 3단계 파이프라인: 검색, 인용 추출 및 신뢰도 점수 매기기. 신뢰도 점수는 상위 3개 증거 단위의 평균 검색 점수입니다.
- **API 키 배너가 "활성"으로 업데이트됨** — 관리자 및 사용자 설정 API 키 페이지에는 이제 API 인증 문서 링크가 포함된 녹색 "API 키 활성" 배너(이전의 "출시 예정" 공지 대체)가 표시됩니다.
- **다중 워커 문서 명확화** — `REDIS_URL` 및 `WORKERS` 환경 변수 설명이 중지, 주입, PostgreSQL 요구사항 및 로컬 개발 지침을 다루는 경고 체크리스트로 업데이트되었습니다.
- **DAG 단계 기본 모델** — `model_hint`이 없는 단계는 이제 일반 모델로 해결됩니다(이전에는 빠른 모델). 빠른 모델은 `model_hint="fast"`가 명시적으로 설정된 경우에만 사용됩니다.
- **라우터 분류 출력** — `RouteDecision`은 이제 기존 `mode` 필드와 함께 `domain_hint`(법률/의료/금융/null)를 출력하여 다운스트림 도메인 인식 모델 선택을 가능하게 합니다.

### 수정됨

- **로케일 문서의 중복 OpenAPI 엔드포인트** — `docs.json`의 모든 비영어 로케일 API 탭에서 중복된 `Endpoints` OpenAPI 그룹을 제거했습니다. 영어 섹션은 단일 소스를 유지하며, 로케일 섹션은 더 이상 중복되지 않습니다.
- **구조화된 출력 저하 체인** — 3단계 폴백(네이티브 FC → JSON 모드 → 일반 텍스트 추출)이 이제 모델이 잘못된 형식의 JSON을 생성할 때 모든 단계를 올바르게 통과합니다. 이전에는 `ValueError`로 2단계에서 단락되었습니다.
- **`json_mode_enabled` DB 값 무시됨** — 관리자(모델 그룹)를 통해 구성된 모델은 이제 `LLM_JSON_MODE_ENABLED` 환경 변수로 항상 폴백하는 대신 모델별 `json_mode_enabled` 설정을 올바르게 사용합니다.
- **DAG 계획 실패 메시지** — 계획 실패 시 이제 원본 파이프라인 오류 대신 모델 전환을 제안하는 사용자 친화적인 이중언어 메시지를 표시합니다.
- **DAG 단계 카드 레이아웃** — `StepProgressCard`를 다시 설계하여 작업 텍스트가 단계 배지 아래에 줄바꿈되도록 하고 한 줄에서 잘리지 않도록 했습니다. 단계 노드의 도구 배지는 `tools_used`가 없지만 `tool_hint`가 있을 때만 표시되어 중복 표시를 방지합니다.
- **MCP 서버 소유자 allow_fallback 우회** — 서버 소유자는 더 이상 `allow_fallback=False` 제한으로 차단되지 않습니다. 이전에는 `allow_fallback=False`가 설정된 경우 MCP 서버의 생성자도 건너뛸 수 있었지만, 이제는 자격 증명이 없는 비소유자만 제외됩니다.
- **모델 공급자 API 키 검증** — openai 호환이 아닌 공급자를 생성할 때 이제 인라인 검증 오류, 필수 별표 표시기 및 적절한 `aria-invalid` 접근성 속성이 있는 API 키가 필요합니다.

## [v0.8] - 2026-03-20

### 추가됨

- **ReAct 합성의 에이전트 지시문** — `ReActAgent`의 새로운 `agent_directive` 파라미터는 에이전트의 핵심 목적(예: "중국어를 영어로 번역")을 합성 시스템 프롬프트에 주입하여 최종 답변이 대화형 응답으로 기본 설정되는 대신 에이전트의 정체성을 존중하도록 보장합니다. ReAct 및 DAG 채팅 엔드포인트 모두에 연결됨.

- **마켓플레이스 재설계 1단계 — 솔루션 + 컴포넌트 2계층 모델** — 평면 6가지 유형 마켓 탭을 구조화된 2단계 네비게이션으로 교체: 최상위 카테고리 토글(솔루션: 에이전트/스킬/워크플로우; 컴포넌트: 커넥터/MCP 서버) 및 유형 하위 탭. 글로벌 마켓과 사용자의 조직 간 전환을 위한 범위 선택기 드롭다운 추가. 마켓 범위에서 KB 제거(조직 전용). 통합된 조직 및 마켓 공유: 둘 다 명시적 구독 필요(더 이상 조직 자동 표시 없음).

- **스마트 파일 콘텐츠 주입** — 작은 업로드된 파일(`<32K` 문자)은 자동으로 가져와 LLM 컨텍스트에 인라인됨; 큰 파일은 메타데이터 요약과 온디맨드 읽기를 위한 도구 힌트 수신.
- **`read_uploaded_file` 에이전트 도구** — 이중 모드 파일 읽기 도구: 순차 접근을 위한 페이지 매김 읽기(오프셋/제한) 및 대상 콘텐츠 추출을 위한 grep 유사 검색(정규식 지원 쿼리).
- **`GET /api/files/{file_id}/content` 엔드포인트** — 업로드된 파일에서 추출된 전체 텍스트에 접근하기 위한 페이지 매김 텍스트 콘텐츠 API.
- **`.content` 사이드카 파일** — 업로드와 함께 사이드카 파일로 저장된 전체 추출 텍스트로 재추출 없이 지속적인 콘텐츠 접근 활성화.
- **파일 API 응답의 `content_length` 필드** — 파일 업로드 및 목록 엔드포인트는 이제 텍스트 콘텐츠 크기 가시성을 위해 `content_length` 포함.
- **`fileApi.getContent()` 프론트엔드 클라이언트 메서드** — 프론트엔드에서 페이지 매김 파일 콘텐츠를 가져오기 위한 새로운 API 클라이언트 메서드.

- **워크플로우 엔진 노드 재시도** — 지수 백오프를 사용한 노드별 재시도. 노드는 데이터 구성에서 `retry_count` 및 `retry_delay_ms`를 지정할 수 있습니다. 실패한 노드는 지연 상한 60초로 N번까지 재시도됩니다. 시도 번호 및 이전 오류와 함께 `node_retrying` SSE 이벤트 발생. 재시도 대기 중 취소 이벤트 존중.
- **워크플로우 변수 저장소 표현식 평가** — simpleeval을 통한 안전한 Python 표현식 평가. `evaluate_expression()` 메서드는 저장소 변수(env 제외)를 공통 내장 함수(len, str, int, float, bool, abs, min, max, sum, round, sorted)와 함께 노출합니다. 또한 `has()`, `delete()`, `keys()` 헬퍼 메서드 포함.
- **워크플로우 2단계 노드: HumanIntervention** — 승인 대기 노드(자동 승인 스텁). prompt_message(`{{variable}}` 보간 포함), assignee, timeout_hours 읽음. 프롬프트 미리보기 및 @assignee 배지가 있는 Sky-500 테마 프론트엔드 노드. 프롬프트 텍스트 영역, assignee 입력, 타임아웃 필드가 있는 구성 패널.
- **워크플로우 템플릿 12개로 확장** — 4개의 새로운 템플릿: 인간 승인 파이프라인(LLM → HumanIntervention → ConditionBranch → 이중 End), 다단계 데이터 처리(CodeExecution → ConditionBranch → Transform → ListOperation), 지식 강화 에이전트(QuestionUnderstanding → KnowledgeRetrieval → LLM), 반복 검토 루프(Loop → LLM → VariableAssign).
- **워크플로우 창 컨텍스트 메뉴** — 빈 캔버스에서 우클릭하여 붙여넣기(클립보드가 비어있을 때 비활성화), 모두 선택, 보기 맞추기, 자동 레이아웃 작업 수행. 판별된 합집합 상태는 노드 vs 창 컨텍스트 메뉴를 분리합니다.
- **워크플로우 노드 컨텍스트 메뉴** — 모든 노드에서 우클릭하여 구성, 복사, 복제, 삭제 작업에 접근. 삭제 작업은 시작/종료 싱글톤 노드에 대해 숨겨집니다. 메뉴는 외부 클릭 또는 스크롤 시 자동 닫힘.
- **워크플로우 2단계 템플릿** — 2개의 새로운 기본 제공 템플릿(총 8개): 목록 처리 파이프라인(Start → ListOperation → Transform → End) 및 향상된 질문 답변(Start → QuestionUnderstanding → LLM → End).
- **워크플로우 2단계 검증** — ListOperation(누락된 입력/표현식), Transform(누락된 입력/빈 작업), Iterator(누락된 목록 변수), Loop(누락된 조건), DocumentExtractor, QuestionUnderstanding(누락된 입력), ParameterExtractor(누락된 입력 텍스트/빈 파라미터)에 대한 확장 블루프린트 검증 경고. 수정: 정렬 작업은 더 이상 표현식을 잘못 요구하지 않습니다.
- **워크플로우 노드 노트** — 노드별 주석 기능. 노드의 구성 패널에서 "노트"를 클릭하여 캔버스의 노드 아래에 이탤릭 텍스트로 표시되는 자유형 노트 추가. `data.note` 저장소가 있는 자동 확장 텍스트 영역.
- **워크플로우 그리드에 맞추기** — 정확한 노드 정렬을 위해 점 배경 패턴과 일치하는 16px 스냅 그리드.
- **워크플로우 연결 검증** — 기존 시작/종료 제한 외에도 자체 연결 및 중복 간선 방지.
- **워크플로우 줌 인식 드롭** — 노드 팔레트 드래그 앤 드롭은 모든 줌 레벨에서 올바른 배치를 위해 `screenToFlowPosition()`을 사용합니다.
- **워크플로우 SSE 다중 라인 데이터** — SSE 파서는 이제 다중 라인 `data:` 필드를 처리하고 keepalive 주석 라인을 무시합니다.
- **워크플로우 키보드 단축키** — 복사(Cmd+C), 붙여넣기(Cmd+V), 복제(Cmd+D), 모두 선택(Cmd+A), Escape로 선택 해제. 복사/붙여넣기는 새 ID를 생성하고 위치를 오프셋합니다; 싱글톤 제약 유지를 위해 시작/종료 노드는 복사에서 제외됩니다.
- **워크플로우 인라인 설명 편집** — 도구 모음의 설명 라인을 클릭하여 워크플로우에서 벗어나지 않고 워크플로우 설명을 편집하기 위해 팝오버 텍스트 영역을 엽니다.
- **워크플로우 ConditionBranch LLM 모드** — 순진한 "첫 번째 일치" 스텁을 대체하는 적절한 LLM 기반 조건 평가. 조건 레이블에서 분류 프롬프트를 구축하고, 빠른 LLM을 호출하며, 대소문자 무시 + 퍼지 매칭을 사용합니다.
- **반응형 간선 레이블** — 조건/분류기 분기에 대한 간선 레이블은 이제 구성 패널에서 조건 또는 클래스가 편집될 때 실시간 업데이트를 위해 `useNodesData()`를 사용합니다.
- **워크플로우 통계 패널** — 총 실행, 완료/실패/취소 횟수, 시각적 진행률 표시줄이 있는 성공률, 평균 지속 시간, 상대 형식의 마지막 실행 시간을 표시하는 시트 기반 통계 표시. 도구 모음의 BarChart3 아이콘에서 트리거됨.
- **2단계 워크플로우 노드: DocumentExtractor & QuestionUnderstanding** — 2개의 추가 노드 유형:
  - **DocumentExtractor**: 문서 텍스트 처리를 위한 스텁 실행기 — 텍스트/base64 입력 유형 및 4가지 추출 모드(full_text, page_range가 있는 페이지, 단어/문자/라인/페이지 개수가 있는 메타데이터, 마크다운 테이블 감지가 있는 테이블) 지원. URL 입력은 나중 구현을 위해 연기됨.
  - **QuestionUnderstanding**: 4가지 모드가 있는 LLM 기반 질문 전처리 — 다시 쓰기(명확성), 확장(컨텍스트 + 하위 질문), 분류(의도/주제/신뢰도 JSON), 분해(하위 질문 배열). 선택적 사용자 정의 시스템 프롬프트 재정의가 있는 빠른 LLM 사용.
  - 전체 프론트엔드: 노드 컴포넌트, 구성 패널, 팔레트 항목(QU의 AI 카테고리, DE의 데이터 카테고리), 변수 선택기 지원.
- **2단계 워크플로우 노드: ListOperation & Transform** — 2개의 데이터 처리 노드 유형:
  - **ListOperation**: 8개의 목록 작업(필터, 맵, 정렬, 슬라이스, 평탄화, 고유, 역순, 길이) 필터/맵/정렬을 위한 simpleeval 표현식 포함. JSON 문자열 입력 자동 파싱 지원.
  - **Transform**: 6가지 작업 유형(json_path, type_cast, format, regex_extract, string_op, math_op)이 있는 파이프라인 기반 데이터 변환. 작업이 순차적으로 적용됨 — 하나의 출력이 다음 입력으로 공급됨.
  - 전체 프론트엔드: 노드 컴포넌트, 작업별 편집기가 있는 구성 패널, 팔레트 항목, 변수 선택기 지원.
- **2단계 워크플로우 노드: Loop & ParameterExtractor** — 2개의 추가 고급 노드 유형:
  - **Loop**: simpleeval 표현식 평가, 구성 가능한 루프 변수 이름, max_iterations 안전 제한이 있는 while 조건 루프. 조건 입력, 변수 이름, 반복 제한 구성이 있는 프론트엔드.
  - **ParameterExtractor**: 텍스트에서 LLM 기반 구조화된 파라미터 추출. 유형 파라미터(이름/유형/설명/필수) 정의, 입력 텍스트 참조 제공, 선택적으로 추출 지침 추가. 파싱된 JSON 반환. 파라미터 목록 편집기 및 추출 프롬프트 텍스트 영역이 있는 프론트엔드.

### 변경 사항

- **구독 기반 가시성 모델** — `build_visibility_filter()`가 3단계(own → org-member → subscribed)에서 2단계(own → subscribed)로 단순화됨. 데이터 마이그레이션이 기존의 모든 조직 구성원에 대해 `ResourceSubscription` 행을 자동으로 생성하여 접근 권한을 유지. 대시보드 커넥터 수 및 KB 통계에 이제 구독 리소스 포함.
- **Market API 범위 및 카테고리 매개변수** — `browse_market()`이 `scope`(market 또는 org:`{id}`) 및 `category`(solutions 또는 components) 쿼리 매개변수를 허용. `subscribe_resource()`가 이제 비-market 조직 구독에 대한 조직 멤버십 검증.

- **도구 캐시 화이트리스트 리팩터** — `ToolCache`의 하드코딩된 블랙리스트를 화이트리스트 방식으로 대체. `Tool` 프로토콜에 `cacheable` 속성 추가(기본값 `False`); `cacheable=True`로 명시적으로 표시된 도구만 캐시됨. 11개의 읽기 전용 도구(web_search, kb_retrieve, calculator 등)가 캐시 가능으로 표시; 모든 부작용 도구(code exec, email, connectors)는 기본적으로 캐시되지 않음. 새로운 커넥터 도구는 추가 구성 없이 안전.
- **DAG 실행기 계단식 실패** — 실패하거나 시간 초과된 단계가 이제 종속 단계를 계단식으로 차단. `failed_ids` 추적 추가(A→B→C 체인의 전이 전파). 종속 단계는 중독된 컨텍스트로 실행되는 대신 설명적 메시지와 함께 자동으로 실패로 표시.
- **DAG 단계 검증기 상태 수정** — 단계 검증 재시도 실패가 이제 `"completed"` 상태로 남겨두는 대신 단계를 `"failed"`로 올바르게 표시. 실패한 검증이 새로운 계단식 실패 메커니즘을 통해 다운스트림 단계로 전파.
- **DAG 플래너 도구 설명** — 플래너가 이제 도구 이름만이 아닌 각 도구에 대해 `(name, description)`을 수신하여 더 정확한 단계-도구 매핑 및 환각된 `tool_hint` 값 감소.
- **DAG 재계획 전체 이력** — 재계획 컨텍스트가 이제 마지막 라운드만이 아닌 모든 이전 라운드의 결과를 누적. 이전 라운드는 더 적극적으로 잘림(단계당 200자)이고 가장 최근 라운드는 500자/단계 유지.
- **stream_answer 합성 품질** — 최종 답변 합성 시 도구 관찰 잘림을 2000에서 8000자로 증가하여 더 많은 구조화된 데이터(JSON 테이블, 코드 출력) 보존.
- **14개 엔진 상수 매개변수화** — ReAct 에이전트, DAG 플래너, 컨텍스트 가드, 워크스페이스 및 시스템 모듈 전반에 걸쳐 하드코딩된 숫자 상수를 환경 변수로 추출. 모든 새로운 환경 변수는 `example.env` 및 `docs/configuration/environment-variables.mdx`에 문서화.

- **증거 신뢰도 UI 개선** — 근거 파이프라인의 프론트엔드 표시에 대한 3가지 개선: (1) 불충분한 증거 관찰이 이제 원본 영어 텍스트 대신 i18n 호박색 경고 카드로 렌더링, (2) 인라인 `[N]` 인용 배지가 새로운 EvidenceContext를 통해 소스 이름, 인용 발췌 및 관련성 점수를 포함한 호버 팝오버 표시, (3) 증거 출처가 서로 모순될 때 참고 섹션에 충돌 경고 배너 표시(확장 가능한 나란히 비교 포함).
- **Shadow Market 조직** — 기본 제공 Platform 조직을 shadow Market 조직으로 변환. 리소스는 이제 멤버십을 통한 자동 가시성(푸시 모델) 대신 마켓플레이스 탐색을 통해 발견되고 명시적으로 구독(풀 모델). 비관리자 사용자는 더 이상 등록 시 자동으로 조직에 가입하지 않음; Market 조직은 사용자가 마켓플레이스 UI를 통해서만 상호작용하는 보이지 않는 백엔드 엔티티로 작동.
- **마켓플레이스 필수 검토와 함께 게시** — Market 조직에 게시하려면 이제 Market 조직 소유자/관리자를 제외하고 항상 검토 필요. 게시 대화가 필수 검토 공지와 함께 "마켓플레이스에 게시" 옵션 표시.
- **Skill 및 Workflow 마켓플레이스 탐색** — Skill 및 Workflow 리소스 유형을 마켓플레이스에 추가하여 모든 5가지 리소스 유형(에이전트, 커넥터, 지식 베이스, MCP 서버, 스킬/워크플로우)에 대한 커뮤니티 발견 및 구독 활성화.
- **Market 조직 게시 엔드포인트 바이패스** — 6개의 리소스 게시 엔드포인트가 이제 Market 조직에 대한 멤버십 확인을 건너뛰어 모든 사용자가 마켓플레이스 멤버가 되지 않고도 리소스를 게시 가능.
- **Platform-to-Market 마이그레이션** — Alembic 마이그레이션이 Platform 조직을 Market으로 이름 변경하고 모든 비소유자 멤버십 제거하여 푸시 기반에서 풀 기반 리소스 공유로의 전환 완료.
- **Workflow 버전 변경 요약** — `compute_blueprint_diff()`가 새 버전 저장 시 blueprint diff에서 인간이 읽을 수 있는 요약을 자동 생성하여 추가/제거/수정된 노드 및 엣지 설명.
- **Workflow 실행 보존 정리** — 구성 가능한 나이 및 개수 제한에 따라 이전 워크플로우 실행을 제거하는 백그라운드 정리 작업. 워크플로우별 보존 재정의. 수동 정리 트리거를 위한 관리자 엔드포인트. 새로운 환경 변수: `WORKFLOW_RUN_MAX_AGE_DAYS`, `WORKFLOW_RUN_MAX_PER_WORKFLOW`, `WORKFLOW_RUN_CLEANUP_INTERVAL_HOURS`.
- **커넥터 서킷 브레이커** — 3상태 서킷 브레이커(closed/open/half-open)와 커넥터별 실패 추적. 5xx 오류 감지 및 실패한 커넥터 임시 비활성화. 서킷 브레이커 상태 검사를 위한 모니터링 엔드포인트.
- **ConnectorMetaTool(점진적 공개 Phase 1-2)** — 단일 메타 도구가 에이전트 프롬프트의 작업별 도구 대체. 시스템 프롬프트가 경량 스텁(커넥터당 ~30 토큰 vs 작업당 ~250 토큰) 수신; 에이전트가 `discover(connector)`를 호출하여 전체 작업 스키마를 필요에 따라 로드하고 `execute`를 호출하여 작업 실행. 하위 호환성을 위한 기능 플래그 `CONNECTOR_TOOL_MODE`.
- **실제 MCP 실행기** — 스텁 MCP 노드 실행기를 워크플로우 실행 내에서 MCP 서버 발견 및 도구 호출을 지원하는 전체 구현으로 대체.
- **에이전트 중간 루프 자기 성찰** — ReAct 루프(JSON 및 네이티브 모드 모두)에서 6개의 도구 호출 반복마다 주입된 경량 목표 확인 프롬프트로 긴 추론 체인에서 목표 표류 방지. 에이전트에게 여전히 진행 중인지, 작업을 반복하는지, 아니면 최종 답변을 생성해야 하는지 묻기.

- **실제 BuiltinTool 실행기** — 스텁 BuiltinTool 노드 실행기를 워크플로우 실행을 위한 `ToolRegistry` 통합을 사용하는 실제 구현으로 대체.
- **의미론적 스키마 주석** — 커넥터 스키마 필드에 대한 16개의 사전 정의된 의미론적 태그(`customer_id`, `email`, `phone`, `amount`, `date` 등). 각 필드는 `semantic_tag`, `description` 및 `pii` 플래그를 전달할 수 있으며, LLM 도구 설명에 표시되어 에이전트가 열 이름에서 추측하지 않고 필드 의도를 이해.
- **커넥터 가져오기/내보내기/포크 UI** — 커넥터를 JSON으로 내보내기, 파일 선택기에서 가져오기, 기존 커넥터 포크(복제 + 사용자 정의)를 위한 프론트엔드 카드 작업.
- **커넥터 가져오기/내보내기 및 포크 엔드포인트** — 백엔드 API: `GET /api/connectors/{id}/export`, `POST /api/connectors/import`, `POST /api/connectors/{id}/fork`(비밀 유출 방지를 위한 삭제된 자격증명 처리).
- **Workflow 템플릿 시스템** — 관리자 CRUD 및 공개 목록/복제 엔드포인트가 있는 `WorkflowTemplate` ORM 모델. 빠른 워크플로우 부트스트래핑을 위해 첫 시작 시 자동 삽입되는 5개의 시드 템플릿.
- **Workflow 인라인 검증 배지** — 캔버스 노드에 실시간 `ValidationBadge`로 노드별 오류 및 경고를 설명적 도구 설명과 함께 표시하여 blueprint 편집 중 즉각적인 시각적 피드백 제공.
- **Workflow 실행 추적 뷰어** — 완료된 실행 검사를 위한 타임라인 기반 추적 뷰어 Sheet. 엔진 `trace_level` 매개변수가 세부 정보 세분성 제어; 단계별 디버깅을 위해 각 노드에서 변수 스냅샷 캡처.
- **Workflow 사용자별 속도 제한 및 전역 시간 초과** — 슬라이딩 윈도우 적용(사용자당 분당 10회 실행, 동시 3회)을 사용하는 `WorkflowRateLimiter`. 모든 워크플로우 실행에 적용되는 기본 10분(600초) 전역 실행 시간 초과.
- **관리자 배치 워크플로우 관리** — 워크플로우에 대한 3개의 배치 관리자 엔드포인트: 배치 삭제, 배치 토글(활성/비활성), 배치 게시(모두 감사 로깅 포함).
- **Workflow 팔레트 개선** — `useRecentNodes` 훅을 통한 최근 사용 노드 섹션, 설명으로 검색 지원, 노드 팔레트의 빈 "결과 없음" 상태.

### 수정됨

- **Template API client endpoint paths** — 워크플로우 템플릿 API 클라이언트를 새로운 `/api/workflow-templates` 경로를 사용하도록 업데이트하여 손상된 템플릿 CRUD 작업을 수정했습니다.
- **Workflow eval namespace flattening**: ConditionBranch와 VariableAssign이 짧은 변수 이름(예: 저장소에 `input.score`가 있을 때 `score`)을 확인하지 못하는 문제를 수정했습니다. eval namespace에 짧은 별칭을 생성하는 `_flatten_eval_names()` 헬퍼를 추가하여 전체 경로 참조 없이 점 표기법 키 확인을 가능하게 했습니다.
- **Workflow `run_started` SSE event**: `run_started` 이벤트는 SSE 프로토콜에 문서화되었지만 실제로 발생하지 않았습니다. 이제 오케스트레이터 루프가 시작되기 전에 발생하므로 프론트엔드 소비자가 실행 시작 신호를 받습니다.
- **Workflow SSE event name mismatch**: SSE 이벤트 이름을 수정했습니다 — `node_status`를 `node_started`/`node_completed`/`node_failed`/`node_skipped`로 이름을 변경하여 프론트엔드 SSE 핸들러 예상과 일치하도록 했습니다.
- **Workflow `NodeRunResult.duration` field name**: `duration`을 `duration_ms`로 수정하여 Pydantic 스키마 및 프론트엔드 파싱과 일치하도록 했습니다.
- **Workflow export/import API response wrapping**: 워크플로우 내보내기 및 가져오기 엔드포인트의 응답 봉투 래핑을 수정했습니다.
- **No re-review on `is_active` toggle**: 에이전트, 커넥터, 지식 베이스 또는 MCP 서버에서 `is_active`를 토글해도 더 이상 `publish_status`를 `approved`에서 `pending_review`로 되돌리지 않습니다. `check_edit_revert` 가드는 이제 실제 콘텐츠 필드가 변경될 때만 작동합니다(`is_active`만 변경되는 경우는 제외).
- **Cascade-skip for condition branches** — 건너뛴 조건/분류자 노드는 이제 나가는 엣지를 올바르게 비활성화하여 다운스트림 노드가 예기치 않게 실행되는 것을 방지합니다.
- **Dependency analyzer `skill_ids` resolution** — `_resolve_agent`는 이제 에이전트가 참조하는 `skill_ids`를 확인합니다. 이전에는 의존성 분석 중에 스킬이 자동으로 무시되었습니다.
- **Case-insensitive workflow node type matching** — 의존성 분석기의 워크플로우 노드 타입 매칭이 이제 대소문자를 구분하지 않습니다. 소문자 타입(예: `"connector"` vs `"CONNECTOR"`)이 있는 블루프린트가 이제 올바르게 확인됩니다.
- **Dependency analyzer test fixtures** — 누락된 DB 픽스처 레코드 및 잘못된 의존성 타입 어설션으로 인한 의존성 분석기 테스트의 7가지 테스트 실패를 수정했습니다.

"고급 빌더" 버튼은 임베드된 PlaygroundPage가 있는 전체 화면 대화상자를 엽니다(왼쪽에 채팅, 오른쪽에 라이브 작업 미리보기). 6개의 커넥터 빌더 도구(`connector_list_actions`, `connector_create_action`, `connector_update_action`, `connector_delete_action`, `connector_update_settings`, `connector_test_action`)는 DB에서 직접 작동하며 빌더 에이전트가 활성화되면 자동으로 주입됩니다. `POST /api/builder/session`은 대화 기록을 유지하는 커넥터별 빌더 에이전트를 생성하거나 재사용합니다.
- **이중 데이터베이스 지원(SQLite + PostgreSQL)**: `DATABASE_URL`은 이제 SQLite(`sqlite+aiosqlite:///...`) 및 PostgreSQL(`postgresql+asyncpg://...`) 연결 문자열을 모두 허용합니다. SQLite는 제로 구성 기본값으로 유지되며, PostgreSQL은 프로덕션 및 다중 워커 배포에 권장됩니다. Docker Compose는 상태 확인을 통해 PostgreSQL 서비스를 자동으로 프로비저닝합니다. Alembic 마이그레이션은 방언 인식입니다(`json_extract` vs `::json->>'key'`). 관리자 상태 대시보드는 실제 데이터베이스 엔진 및 버전을 표시합니다.
- **모델 구성 문서**: 각 공급자에 대한 확장 사고/추론 설정을 포함한 모델 구성 참조를 문서화하는 새로운 `docs/configuration/models.mdx` 페이지.
- **확장 사고/추론 지원**: 확장 사고가 있는 모델(OpenAI o 시리즈, Gemini 2.5+, Claude)에 대해 `LLM_REASONING_EFFORT`(낮음/중간/높음) 및 `LLM_REASONING_BUDGET_TOKENS` 환경 변수를 추가합니다. OpenAI/Gemini의 경우 `reasoning_effort`를 전송하고, Anthropic의 경우 `thinking`을 전송합니다(자동 감지). 여러 공급자 필드 이름(DeepSeek R1, Claude 프록시)에서 `reasoning_content`를 추출하고 UI "사고" 단계에서 표시합니다.
- **관리자: 도구 비활성화**: 도구별 활성화/비활성화 토글이 있는 새로운 관리자 도구 패널(`disabled_builtin_tools` 설정으로 저장됨). 비활성화된 도구는 런타임에 채팅 도구 해석에서 필터링됩니다. `/api/tools/catalog` 엔드포인트는 `disabled` 플래그를 반환하고, 도구 페이지 UI는 비활성화 상태를 표시합니다.
- **관리자: 커넥터 페이지의 MCP 서버**: MCP 서버 관리가 도구 페이지에서 커넥터 페이지로 새로운 탭으로 이동했습니다. `MCPServersSection`이 명령형 작업 참조가 있는 재사용 가능한 컴포넌트로 추출되었습니다.
- **관리자: 상태 대시보드의 추론 노력**: 추론 노력 레이블이 이제 관리자 상태 확인 대시보드에 표시됩니다.
- **내보내기: PDF 형식**: 대화를 이제 PDF 문서로 내보낼 수 있습니다(`reportlab` + `Markdown` 추가 필요).

### 변경사항
- **Tests: NativeToolFakeLLM이 스트림 기반 ReAct로 업데이트됨**: 테스트 fake LLM이 이제 새로운 스트림 기반 `_run_native()` 루프와 일치하는 `StreamChunk` 시퀀스를 생성합니다
- **LLM 레이어**: 직접 `AsyncOpenAI` 클라이언트를 [LiteLLM](https://github.com/BerriAI/litellm)으로 교체하여 범용 제공자 지원을 제공합니다. 모든 제공자별 조건부 처리(Anthropic 사고, `max_completion_tokens`, GPT-5 도구 해결책)가 제거되었습니다. Anthropic 모델은 이제 LiteLLM의 네이티브 API를 통해 라우팅되어 적절한 `reasoning_content`(확장 사고) 지원을 활성화합니다

- **Redis 인터럽트 브로커**: Redis Pub/Sub을 통한 워커 간 인터럽트/주입 릴레이입니다. `REDIS_URL`이 설정되면 자동으로 활성화되고, 그렇지 않으면 프로세스 내 큐로 폴백됩니다. Docker Compose는 이제 Redis 서비스를 포함합니다(자동 구성, 사용자 작업 불필요)
- **내보내기: 기본 상세 수준**: 내보내기 대화상자는 이제 전체 대신 요약(실행 세부사항 없음)으로 기본 설정됩니다
- **아티팩트: 파일 mtime으로 정렬**: 모든 아티팩트 나열은 이제 정확한 순서 지정을 위해 파일별 mtime을 사용합니다(대화 시간 해결책 되돌림)
- **채팅: 비동기 이미지 로딩**: `_load_image_data_urls`는 이제 비동기이며, `asyncio.to_thread`를 통해 파일 읽기를 스레드로 오프로드하여 이벤트 루프를 차단하지 않습니다
- **다중 워커 준비**: SSE 티켓, OAuth 상태 및 바인드 티켓은 이제 메모리 내 딕셔너리 대신 JWT 서명 토큰을 사용합니다. 파일 인덱스 잠금은 프로세스 간 안전을 위해 `fcntl.flock()`을 사용합니다. 유지보수 캐시 TTL은 워커 간 더 빠른 전파를 위해 30초에서 5초로 단축되었습니다
- **상태 대시보드: 인프라 검사**: 관리자 상태 대시보드에 데이터베이스 유형(SQLite/PostgreSQL) 및 Redis 상태가 추가되었습니다
- **스마트 릴레이 라우팅**: URL 경로 패턴(예: `/claude` 또는 `/anthropic` → Anthropic 네이티브, `/gemini` → Google 네이티브)에서 API 프로토콜을 자동으로 감지하여 수동 제공자 구성 없이 타사 릴레이 플랫폼에 대한 적절한 인증 헤더 및 매개변수 변환을 활성화합니다
- **구조화된 출력 저하**: 3단계 추출(Native FC → JSON Mode → 일반 텍스트 + 정규식 폴백)을 포함한 통합 `structured_llm_call()` 유틸리티입니다. DAG 플래너, 계획 분석기 및 도구 선택을 공유 유틸리티를 사용하도록 리팩토링하여 임시 JSON 파싱을 모든 구조화된 LLM 호출 사이트에서 일관된 재시도 및 저하로 대체합니다
